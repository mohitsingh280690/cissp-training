⚠️ Exam Trap: Stricter classification = stricter handling at every stage (transmission, storage, disposal). A common exam trap presents handling procedures that are appropriate for a lower classification applied to a higher classification — and asks you to identify the violation.

2. Real-World Relevance

Data handling failures are a primary vector for data breaches. Unmarked sensitive documents treated as public; employees emailing confidential data via unencrypted channels; USB drives with PII left in parking lots — all handling failures. Clear, enforced handling procedures reduce human error and create accountability chains for sensitive information.

3. Marking and Labelling

Marking: The application of visual or digital indicators to data assets indicating their classification level, enabling users and systems to handle them appropriately.

Physical Document Marking

  • Headers and footers on every page bearing the classification label (e.g., "CONFIDENTIAL – INTERNAL USE ONLY")
  • Cover page classification marking (mandatory for multi-page documents)
  • Individual page marking for documents where pages may be separated (government use case)
  • Marking must be visible and unambiguous — abbreviations should follow organisational standards

Digital / Electronic Marking

  • Metadata tags: Document properties containing classification metadata readable by DLP systems
  • Email sensitivity labels: Microsoft Purview / MIP sensitivity labels (Public, General, Confidential, Highly Confidential) applied to emails and documents
  • Watermarks: Visible classification watermarks embedded in digital documents
  • File naming conventions: Some organisations include classification in file names (e.g., "budget_FY24_CONFIDENTIAL.xlsx")
  • DLP policy tags: Machine-readable labels consumed by Data Loss Prevention tools to enforce handling policies automatically

Physical Equipment / Media Labelling

  • Physical labels on storage devices (USB drives, external HDDs, backup tapes) indicating the highest classification of data they contain
  • Server/cabinet labels indicating classification of hosted data
  • Colour-coded labels for quick visual identification in secure facilities
✅ Exam Tip: The purpose of marking is to allow any person (or automated system) handling the asset to immediately know what protection level it requires without having to read the contents. Unmarked sensitive data is a handling failure even if the contents are inherently sensitive.

5. Shipping and Transport Security

When sensitive data must physically move (printed documents, storage media, hardware), transport security controls must match the classification:

  • Encrypted transport media: USB drives and removable media should be hardware-encrypted; if media containing sensitive data is lost in transit, encryption prevents disclosure
  • Chain of custody documentation: Record of every person or organisation that had possession of the asset, from sender to receiver
  • Tamper-evident packaging: Security envelopes and containers that show if the package has been opened or tampered with in transit
  • Approved couriers: For classified government material — only GSA-approved or agency-cleared couriers; for commercial sensitive data — tracked courier services with insurance
  • Sealed transit containers: Double-envelope system for highly classified material; inner envelope marked with classification, outer envelope unmarked
  • Receipt acknowledgement: Receiver confirms receipt; enables detection of lost-in-transit incidents

6. Physical Storage Requirements

  • Locked cabinets and safes: Printed sensitive material must be stored in locked storage when not in use; classification level determines lock grade
  • Clean desk policy: No sensitive documents left unattended on desk surfaces — a core physical security control against shoulder-surfing and casual access
  • Clear screen policy: Lock screens / screensavers when leaving a workstation; prevents casual viewing of sensitive data on screen
  • Secure printing: Pull printing / follow-me printing — documents not released from printer until user authenticates at the device (prevents documents left uncollected in printer trays)
  • Shredding of drafts: Waste paper containing sensitive data (even draft copies, error printouts) must be shredded — not placed in regular waste bins
  • Visitor management: Sensitive areas require visitor escort; restrict visitor line-of-sight to sensitive screens and documents

7. Digital Storage Requirements

  • Encryption at rest: Required for confidential and above; AES-256 minimum; FDE for endpoint devices
  • Access controls: Classification-appropriate access control mechanisms (file system ACLs, database roles, application-level controls)
  • Approved systems: Sensitive data may only be stored on approved, accredited systems — not personal devices, personal cloud storage, or unmanaged USB drives
  • Data residency: Some regulations require data to remain in specific countries/regions (GDPR for EU data; Chinese data localisation laws); cloud storage must comply with data residency requirements
  • Shadow IT controls: Unapproved cloud syncing (personal Dropbox, Google Drive) can violate storage requirements; DLP and CASB controls enforce approved storage boundaries

8. Media Management

Media management refers to the policies and controls governing physical and logical storage media throughout its lifecycle:

  • Removable media policy: Restrict or disable USB ports and removable drives via endpoint management (Group Policy, MDM); auto-run disabled; only organisation-approved encrypted devices allowed
  • Media inventory: Track all removable media used for sensitive data — who has it, what it contains, where it is stored
  • Media sanitisation before reuse: Any media reused for a different classification level or user must be sanitised (Clear or Purge per NIST 800-88)
  • Cloud storage policy: Define approved cloud services; enforce via CASB (Cloud Access Security Broker); data classification determines which cloud tiers are permissible (e.g., Top Secret data may not be stored in commercial public cloud)
  • Backup media security: Backup tapes/drives require the same protection level as the data they contain; encrypted off-site backups; chain of custody for off-site rotation
  • End-of-life media disposal: Media must be sanitised (Clear/Purge/Destroy per classification) before disposal or decommissioning; never donate, sell, or discard unwiped media
Key Concept — Media Sanitisation Standard: NIST SP 800-88 "Guidelines for Media Sanitization" is the authoritative standard. It defines Clear, Purge, and Destroy methods suited to different media types and classification levels. (Covered in depth on Day 30.)

9. Memory Tricks & Mnemonics

🧠 Marking Purpose:
"If it's not marked, it's not managed." — Unmarked sensitive data is a governance failure. Classification without marking is classification in name only.
🧠 Physical Handling Rules — "Clean, Clear, Lock, Shred":
Clean desk — nothing sensitive left in the open
Clear screen — lock when away
Lock cabinets — sensitive material secured when not in use
Shred — all sensitive waste destroyed, not binned
🧠 Media Rules — "No Random USB":
No unmanaged USB drives with sensitive data
Residency requirements — know where data can and cannot be stored
Use approved encrypted media only
Sanitise before reuse, disposal, or any reclassification
Backups encrypted at the same level as the source data
