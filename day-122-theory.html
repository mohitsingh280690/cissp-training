2. Real-World Organizational Relevance

Databases contain an organization's crown jewels — customer PII, financial records, healthcare data, intellectual property. Database breaches consistently result in the highest costs per incident. SQL injection remains in the OWASP Top 10 after 20+ years. Database activity monitoring, encryption at rest, and proper access controls are essential for compliance (PCI DSS, HIPAA, GDPR).

4. Relational Database Concepts

TermDefinition
Relation (Table)A two-dimensional table of data
Tuple (Row)A single record in a table
Attribute (Column)A specific field/property of the data
Primary keyUnique identifier for each row; cannot be NULL; ensures entity integrity
Foreign keyLinks to primary key of another table; enforces referential integrity
SchemaThe structure/blueprint of the database — tables, columns, types, relationships
DomainThe set of allowable values for an attribute (e.g., age must be 0-150)

Normalization

Normalization is the process of organizing database tables to reduce redundancy and dependency. Each normal form (1NF → 2NF → 3NF → BCNF) progressively eliminates specific types of redundancy.
FormRule
1NFNo repeating groups; each cell contains a single atomic value
2NF1NF + no partial dependencies (every non-key attribute depends on entire primary key)
3NF2NF + no transitive dependencies (non-key attributes depend only on the primary key, not on other non-key attributes)
Normalization Security Benefit: Normalized databases reduce data redundancy, which means: (1) fewer places to update data → fewer inconsistencies; (2) smaller attack surface (data stored in one place); (3) access control is more precise (sensitive data isolated in specific tables).

5. Database Integrity & ACID Properties

ACID Properties

PropertyDefinitionExample
AtomicityTransaction completes FULLY or not at all (all-or-nothing)Bank transfer: debit $100 from A AND credit $100 to B — both happen or neither happens
ConsistencyTransaction brings database from one valid state to another valid state; all rules/constraints satisfiedAfter transfer: total balance across all accounts remains unchanged
IsolationConcurrent transactions don't interfere with each other; each appears to execute sequentiallyTwo simultaneous transfers from same account don't create phantom money
DurabilityOnce committed, transaction results are permanent even if system crashesAfter "commit confirmed," the transfer survives a power outage

BASE Properties (NoSQL)

PropertyDescription
Basically AvailableSystem guarantees availability (may serve stale data)
Soft stateState may change over time even without input (due to eventual consistency)
Eventual consistencySystem will become consistent eventually, but not immediately after a write
ACID vs BASE: ACID (relational) prioritizes consistency and integrity — every read returns the latest write. BASE (NoSQL) prioritizes availability and partition tolerance — reads may return stale data temporarily. CAP Theorem: a distributed system can guarantee only two of three: Consistency, Availability, Partition tolerance.

6. Database Security Controls

ControlDescription
ViewsVirtual tables showing only specific columns/rows; restrict what users can see without modifying underlying tables; provides logical access control
Stored proceduresPre-compiled database operations; users call the procedure instead of writing raw SQL; prevents SQL injection if properly implemented
Encryption at restTransparent Data Encryption (TDE) — encrypts database files on disk; protects against physical theft
Column-level encryptionEncrypt specific sensitive columns (SSN, credit card); more granular than TDE
Database Activity Monitoring (DAM)Monitors all database queries in real-time; detects anomalies, unauthorized access, policy violations
RBAC in databaseGRANT/REVOKE permissions; roles define what operations users can perform on which tables

Polyinstantiation

Polyinstantiation: Allowing multiple rows with the same primary key but different security classifications. Used in multilevel security (MLS) databases to prevent inference attacks. A user with SECRET clearance sees one version of a record; a user with TOP SECRET sees a different version with the actual data. Prevents lower-clearance users from inferring that classified data exists by seeing "access denied" on specific rows.

8. Database Attacks

AttackDescriptionDefense
SQL InjectionInserting malicious SQL into application queries; can read, modify, or delete data; bypass authenticationParameterized queries (prepared statements); input validation; stored procedures; least privilege; WAF
AggregationCombining multiple pieces of non-sensitive data to derive sensitive information; individually harmless, collectively classifiedAccess controls on aggregate queries; classify aggregated data at higher level
InferenceDeducing classified information from observable patterns or available unclassified dataPolyinstantiation; noise injection; restrict statistical queries; cell suppression
Data diddlingUnauthorized modification of data before or during entry into the systemInput validation; integrity checks; audit logging; separation of duties
TOC/TOURace condition in database operations — data changes between check and useProper locking; serializable isolation level; atomic transactions
Aggregation vs Inference:
• Aggregation: Combining many pieces of low-classification data to create high-classification information. Think: collecting all employee salaries to derive total payroll budget.
• Inference: DEDUCING sensitive information from available clues. Think: noticing that a classified field returns "access denied" reveals that something classified exists there.
Both are database-specific threats tested on CISSP. Polyinstantiation specifically defends against inference.

9. Memory Tricks & Mnemonics

🧠 ACID — "All Changes Isolated and Durable"
Atomicity = All or nothing
Consistency = Constraints always satisfied
Isolation = Independent transactions
Durability = Data survives crashes
🧠 Aggregation vs Inference
Aggregation = Adding up (aggregate = sum) non-sensitive data to reach sensitive
Inference = Inferring (guessing/deducing) sensitive data from clues
"Aggregation = Addition; Inference = Intelligence"
🧠 Polyinstantiation — "Poly = Many versions"
Multiple instances of same record at different classification levels. Prevents inference by showing "cover" data to lower clearances instead of "access denied."
🧠 Database Terms — "Relation = Table = R = Row is Tuple, Column is Attribute"
Table = Relation
Row = Tuple = Record
Column = Attribute = Field
