3. Routing Fundamentals

📖 Routing: The process by which a Layer 3 device (router) determines the best path for a packet to travel from source to destination across interconnected networks, based on the destination IP address. Routers make forwarding decisions using a routing table.
Routing Table Entry Components: Destination Network | Next Hop | Interface | Metric | AD | Source ─────────────────────┼──────────────────┼───────────┼────────┼─────┼─────── 192.168.10.0/24 | Directly Connected│ Gi0/0 │ 0 │ 0 │ C 10.0.0.0/8 | 172.16.1.2 │ Gi0/1 │ 110 │ 90 │ EIGRP 0.0.0.0/0 | 203.0.113.1 │ Gi0/2 │ 1 │ 1 │ Static Key fields: • Destination Network: range of IP addresses matched against packet destination • Next Hop: IP address of next router (or "exit interface" for directly connected) • Metric: cost of the path (lower = better, within same protocol) • Administrative Distance (AD): trust ranking when same destination learned from multiple protocols (lower AD = more trusted)

Administrative Distance (AD)

Route SourceAdministrative DistanceTrust Level
Directly Connected0Absolute trust — physically connected
Static Route1Manually configured by admin
EIGRP (internal)90Cisco proprietary hybrid protocol
OSPF110Open standard link-state protocol
RIP120Old distance-vector; limited hop count
EIGRP (external)170Routes redistributed into EIGRP from another protocol
BGP (external)20Internet routing (eBGP trusted more than iBGP)
Unknown / Unroutable255Not used — route not installed
🔑 AD vs. Metric: When the same destination is learned from two different routing protocols, the router uses Administrative Distance to choose (lower AD wins). When the same destination is learned from two routes within the same protocol, the router uses Metric (lower metric wins; metric meaning varies by protocol — hops for RIP, bandwidth/delay for EIGRP, cost for OSPF).

4. Static vs. Dynamic Routing

AspectStatic RoutingDynamic Routing
ConfigurationManually entered by admin; does not change unless admin modifies itAutomatically learned and updated by routing protocols (OSPF, EIGRP, BGP)
ScalabilityPoor — impractical for large networks; each route managed manuallyGood — protocols discover and propagate routes automatically
AdaptabilityCannot adapt to topology changes; if a link fails, traffic does not automatically reroute unless a floating static route is configuredConverges automatically to find alternate paths when links fail
SecurityMore secure — admin controls exactly what is routed; no risk of malicious route injection via protocolRisk of route injection attacks if protocol authentication is not configured
Bandwidth OverheadNone — no routing updates sentProtocol messages consume bandwidth (BW is minimal on modern networks)
CISSP Use CaseDefault route (0.0.0.0/0 → ISP); management access; stub networks with single exit pathCore enterprise routing; internet edge (BGP); data centre
💡 Exam Tip: Static routes have AD=1, which means if a static route exists, it will always be preferred over any dynamically learned route to the same destination (except directly connected, AD=0). A floating static route is a static route configured with a higher AD (e.g., AD=200) so it is only used as a backup when dynamic routes disappear — a common failover technique.

5. Distance-Vector Protocols

📖 Distance-Vector Protocol: Each router only knows its own directly connected networks and what its neighbours have told it. Routes are shared as "distance and direction" (hop count or composite metric + next hop). Routers share entire routing tables with direct neighbours at regular intervals. Sometimes called "routing by rumour."

RIP – Routing Information Protocol

AspectRIPv1RIPv2
StandardRFC 1058RFC 2453
MetricHop count (max 15; 16 = unreachable)Hop count (max 15; 16 = unreachable)
Subnet SupportClassful only — no VLSM/CIDRClassless — supports VLSM, CIDR
AuthenticationNonePlain text or MD5
UpdatesBroadcast; every 30 secondsMulticast (224.0.0.9); every 30 seconds
ConvergenceSlow (minutes)Slow (minutes)
StatusObsoleteLegacy — rarely used; replaced by OSPF/EIGRP
⚠️ Count-to-Infinity Problem: Distance-vector protocols like RIP suffer from a convergence problem where after a link failure, routers keep incrementing hop counts for a lost route ("counting to infinity" = 16 in RIP). Solutions: split horizon (don't advertise a route back to where you learned it), route poisoning (immediately advertise unavailable routes as infinite cost), and holddown timers (ignore updates for a broken route during convergence). All are built into RIP but convergence is still slow.

IGRP & EIGRP (Cisco Protocols)

ProtocolTypeMetricNotes
IGRPDistance-Vector (Cisco, legacy)Bandwidth, delay, reliability, load, MTUObsolete; AD=100; replaced by EIGRP
EIGRPHybrid / Advanced Distance-VectorBandwidth + Delay (primary); reliability and load optionalCisco proprietary (now open standard); AD=90; fast convergence via DUAL algorithm; partial updates (not full table)
🔑 Why EIGRP is Called "Hybrid": EIGRP uses DUAL (Diffusing Update Algorithm) to maintain a topology table and pre-calculate backup routes (feasible successors). It converges nearly as fast as link-state protocols but uses distance-vector concepts. Unlike RIP, it sends only partial incremental updates (not the full routing table every 30 seconds), making it much more bandwidth efficient.

6. Link-State Protocols

📖 Link-State Protocol: Each router has a complete map of the entire network topology (Link State Database / LSDB). Routers flood Link State Advertisements (LSAs) describing their directly connected links to all other routers. Every router runs Dijkstra's Shortest Path First (SPF) algorithm independently to compute the best path to every destination.

7. BGP – Border Gateway Protocol

📖 BGP (Border Gateway Protocol): The routing protocol of the internet. BGP is an exterior gateway protocol (EGP) that routes traffic between autonomous systems (ASes) across the internet. Operates over TCP port 179. Uses path attributes (AS Path, Next Hop, Local Preference, MED) rather than simple metrics to make routing decisions. Defined in RFC 4271.
🔑 Autonomous System (AS): A collection of IP networks under a common administrative domain (e.g., an ISP, large enterprise, or cloud provider) that has a single routing policy. Each AS is identified by an ASN (Autonomous System Number) — 16-bit (historic) or 32-bit. Example: Google is AS15169; Cloudflare is AS13335. BGP routes traffic between ASes.

iBGP vs. eBGP

TypeDescriptionAD
eBGP (External BGP)BGP session between routers in different autonomous systems (e.g., your company's router ↔ ISP's router)20

⚠️ BGP Was Designed for Trust, Not Security: BGP was designed in 1989 for interconnecting trusted networks. There is no built-in cryptographic verification that an AS actually owns the prefixes it announces. Anyone with a BGP-speaking router can claim to own any IP prefix. The solution is RPKI (Resource Public Key Infrastructure) — cryptographically signed Route Origin Authorizations (ROAs) that certify which AS is authorized to originate a prefix.

8. Routing Security Threats

ThreatDescriptionProtocol AffectedDefence
BGP Prefix HijackingAttacker announces ownership of victim's IP prefix; traffic is redirected to attacker or droppedBGPRPKI + ROA; prefix filtering; MANRS compliance
Route Injection (Interior)Attacker injects malicious routes into OSPF/EIGRP/RIP via an unauthenticated connection to the routing domainOSPF, EIGRP, RIPRouting protocol authentication (MD5/SHA); isolate routing protocols to dedicated VLANs; limit adjacencies
Route Poisoning DoSAttacker floods routing updates claiming routes as unreachable, causing legitimate routes to be removed; network becomes unreachableDistance-vector protocolsRoute authentication; rate limiting update processing; monitoring for anomalous routing table changes
Routing LoopMisconfiguration causes packets to circulate indefinitely, consuming bandwidth (TTL eventually drops packets)AnyProper design; split horizon; route poisoning; holddown timers; STP equivalent in routing
OSPF LSA Flooding AttackAttacker floods OSPF area with crafted LSAs to overload the SPF computation or inject false topology informationOSPFOSPF MD5 authentication; max-LSA limits; rate limiting
Black Hole RoutingSpecifically advertising a route that points to null interface (intentional or via attacker injection) → packets silently discardedAnyMonitoring for routing anomalies; RTBH (Remote Triggered Black Hole) is a legitimate DDoS mitigation tool when used with filters
🔑 RTBH (Remote-Triggered Black Hole Routing): A legitimate defensive technique where IP prefixes under DDoS attack are advertised to upstream ISPs with a next-hop pointing to null (discard). This drops attack traffic at the edge rather than letting it traverse the entire network. The trade-off is that legitimate traffic to that prefix is also discarded — acceptable if the destination is unreachable anyway due to the attack volume.

💡 Exam Tip: The CISSP exam will not ask you to calculate OSPF costs or configure BGP. Focus on concepts: what routing protocols exist, their general characteristics (distance-vector vs. link-state), their security weaknesses, and the compensating controls. The most tested BGP scenario is hijacking/route injection with RPKI as the solution. The most tested interior routing security concept is the need for protocol authentication.

10. Memory Tricks & Mnemonics

🧠 Routing Protocol Types — "DV Sees Rumours; LS Sees the Map":
Distance-Vector = "routing by rumour" — only knows what neighbours say; RIP, IGRP
Link-State = "routing by map" — knows entire topology; runs Dijkstra SPF; OSPF, IS-IS
Hybrid (EIGRP) = best of both — distance-vector with fast convergence and topology table
🧠 Administrative Distance — "Static Wins Almost Everything":
Connected=0 → Static=1 → eBGP=20 → EIGRP internal=90 → OSPF=110 → RIP=120 → iBGP=200
Memory: "Clearly Stated, Eric's OSPF Rather Ignores Borders"
(Connected, Static, eBGP, EIGRP, OSPF, RIP, iBGP)
🧠 BGP Security = RPKI + MD5 + Prefix Lists:
RPKI = proves ownership of IP space (Route Origin Authorization)
MD5 = authenticates the BGP TCP session
Prefix Lists = filters what routes are accepted/advertised
"RPki Manages Prefixes safely."
🧠 OSPF vs. RIP Key Differences:
OSPF = Open standard, SPF algorithm, Partial updates, Fast convergence
RIP = Rumour-based, Interval full-table flood, Poor scalability (max 15 hops)
For CISSP: always prefer OSPF over RIP for enterprise; RIP is legacy/insecure.
🧠 BGP Hijacking Example — "Pakistan, China, Facebook":
These three real-world incidents illustrate BGP's trust-by-default weakness. Remember them for scenario questions about "what attack could redirect global internet traffic?" — the answer is BGP prefix hijacking.
