1In STRIDE threat classification, an attacker manipulates application logs to delete evidence of their intrusion, falsely claiming they never accessed the system. Which STRIDE category does this PRIMARILY violate, and which security property does it threaten?
✅ Correct: C. STRIDE Repudiation is the threat that an actor can deny performing an action because evidence is unavailable, destroyed, or untrustworthy. The security property it threatens is Non-repudiation — the ability to prove that a specific entity performed a specific action. While log modification also involves Tampering (integrity), the primary threat described here is the ability to deny the action occurred (repudiation). Controls: write-once (WORM) log storage, remote/centralised log aggregation, cryptographic log signing, and immutable audit trails prevent repudiation attacks.
2A web application reflects unsanitised user input back to other users' browsers. An attacker injects a script that steals authenticated session tokens when victims visit the page. Under STRIDE, which category MOST accurately captures the primary risk of session token theft?
✅ Correct: C. The question specifically asks about the risk of session token theft — the tokens themselves are sensitive authentication data that are disclosed to an unauthorised party. Under STRIDE, unauthorised exposure of sensitive information is Information Disclosure. The chain reaction (B — Spoofing, D — EoP) may follow from the stolen tokens, but the primary STRIDE category for the theft act is Information Disclosure. CISSP requires precision: XSS has multiple STRIDE implications (Tampering for script injection, Information Disclosure for data theft, Spoofing for session hijacking) — the question anchors on the token theft phase.
3A security architect must choose between STRIDE and PASTA for threat modeling a high-value banking application. Management specifically asks for a methodology that ties threats to business risk impact, incorporates real-world attack simulation, and generates metrics for executive reports. Which methodology is MOST appropriate, and why?
✅ Correct: B. PASTA (Process for Attack Simulation and Threat Analysis) is a seven-stage, risk-centric methodology: (1) Define business objectives, (2) Define technical scope, (3) Application decomposition, (4) Threat analysis, (5) Vulnerability analysis, (6) Attack modelling, (7) Risk and impact analysis. Its business-centric design and attack simulation stage produce executive-level risk metrics and business impact scores. STRIDE is a simpler threat categorisation framework excellent for developer-level DFD analysis, but it does not natively produce business risk metrics or attack simulation outputs. DREAD (D) is a risk scoring model, not a full threat modeling methodology.
4In an attack tree, a root node representing "Gain Admin Access to Database" has two child nodes connected via an AND gate: "Obtain valid DBA credentials" AND "Bypass MFA". What does this AND gate indicate, and how does it affect the overall attack feasibility?
✅ Correct: B. In attack trees: OR gate means ANY single child condition is sufficient to achieve the parent goal (easier for attacker — multiple paths to success). AND gate means ALL child conditions must be satisfied simultaneously (harder for attacker — defender needs to block only ONE sub-goal to break the entire attack path). In this example, MFA is functioning as an AND gate — even if an attacker steals credentials, they cannot succeed without also bypassing MFA. This is why layered security (defence-in-depth) is so powerful: each additional AND condition in the attacker's path dramatically reduces attack feasibility.
5During Data Flow Diagram (DFD) analysis in a STRIDE threat model, a security analyst identifies a point where customer payment data moves from a public-facing web server (lower trust) to an internal payment processing service (higher trust). What does this point represent, and what unique threats does it introduce?
✅ Correct: B. A Trust Boundary in a DFD marks where data flows between components operating at different privilege or trust levels. Every trust boundary crossing is a critical threat modeling focus area because: untrusted/user-controlled data entering a higher-trust process could cause injection attacks; missing authentication at the boundary could enable Spoofing; missing integrity checks could enable Tampering. Key threats at trust boundaries: all six STRIDE categories apply, but Spoofing, Tampering, and Elevation of Privilege are particularly critical. Defence: validate, sanitise, and authenticate all data crossing trust boundaries regardless of source.
6A threat intelligence report states: "APT29 leveraged encoded PowerShell commands executed via scheduled tasks to achieve persistence on compromised hosts in the energy sector." In the TTP (Tactics, Techniques, and Procedures) framework, the encoded PowerShell execution is specifically classified as:
✅ Correct: C. In TTP (Tactics, Techniques, Procedures): Tactic = the adversary's high-level strategic goal (e.g., Persistence, Lateral Movement, Exfiltration). Technique = the general method used to achieve the tactic (e.g., PowerShell execution, Spearphishing). Procedure = the specific, detailed, actor-attributed implementation — exactly HOW a specific threat actor executed the technique in a specific campaign. "Encoded PowerShell commands via scheduled tasks, attributed to APT29 in the energy sector" is a Procedure-level detail. Procedures have the highest intelligence value for detection (specific IOCs, signatures, behaviour patterns) but the shortest shelf life (adversaries rotate procedures quickly).
7A developer argues that obscuring API endpoint names and removing version headers from HTTP responses is "threat modeling" because it makes it harder for attackers to find vulnerabilities. A security architect disagrees. Which principle BEST explains why this approach is INSUFFICIENT as a primary threat mitigation strategy?
✅ Correct: B. Kerckhoffs' Principle states that a cryptographic system (or security system broadly) should be secure even if everything about the design is public knowledge — security must not depend on secrecy of the design. Security through obscurity is not a threat model; it is an ineffective layer that: (1) delays but does not prevent attacker discovery; (2) fails completely once the system is enumerated; (3) does nothing against STRIDE categories that rely on exploiting logic flaws, not system discovery. Obscurity measures (header removal, endpoint obfuscation) can be useful as supplementary hardening but are never a substitute for fixing underlying vulnerabilities.
8During a STRIDE analysis of an internal HR microservice, an analyst marks data flows between two internal services — both within the same trust boundary — as "safe" and skips STRIDE analysis. A security reviewer flags this as an error. What MOST accurately describes the flaw in this reasoning?
✅ Correct: B. Assuming internal data flows are safe is a classic threat modeling error (the "castle-and-moat" fallacy). Modern threat models must account for: insider threats (malicious or compromised employees); post-compromise lateral movement (an attacker who breaches one internal service can attack others); supply chain compromise; Zero Trust principles explicitly reject implicit internal trust. Even internal service-to-service communication must be: mutually authenticated, authorised (least privilege), encrypted in transit, and logged for audit. STRIDE applies to internal flows: a compromised service can spoof another, tamper with data, or disclose information laterally.
9At what phase of the Software Development Life Cycle (SDLC) does threat modeling provide the GREATEST return on investment, and why?
✅ Correct: C. The NIST and CISSP-endorsed principle (also reflected in the "Rule of Ten"): the cost to fix a security defect increases by approximately an order of magnitude at each SDLC phase. A design flaw fixed at the Design phase costs 1× baseline. The same flaw fixed at: Coding ≈ 10×, Testing ≈ 100×, Post-release ≈ 1,000×. Threat modeling at Design enables: architectural countermeasures (authentication at the right layer, encryption of the right data); framework-level controls; security-by-design rather than bolt-on security. Testing (A) finds implementation bugs, not design flaws. Requirements (D) is too early — some threats require design context to identify.
10A privacy engineer is tasked with threat modeling a healthcare analytics platform that aggregates de-identified patient records from multiple sources. Combining age, ZIP code, admission date, and department fields allows re-identification of patients. Which threat modeling framework is MOST specifically designed to identify this type of privacy threat?
✅ Correct: D. LINDDUN (Linkability, Identifiability, Non-repudiation, Detectability, Disclosure of information, Unawareness, Non-compliance) is a privacy-specific threat modeling framework that mirrors STRIDE's structure but targets privacy threats. Linkability — the ability to correlate records across datasets to re-identify individuals — is a named LINDDUN category, directly applicable to the quasi-identifier re-identification scenario described. STRIDE's Information Disclosure (A) covers security data breaches but lacks the nuance for privacy-specific threats like linkability, inference attacks, and consent violations. LINDDUN is increasingly relevant for GDPR, HIPAA, and cross-border privacy compliance contexts tested in CISSP.
