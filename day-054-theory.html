3. Layer 5 – Session

📖 Layer 5 (Session): Manages sessions (dialogues) between applications. Responsible for establishing, maintaining, synchronizing, and terminating communication sessions between two hosts. Handles session checkpointing and recovery.
AspectDetail
PDUData (no specific L5 PDU name in practice)
Protocols/APIsNetBIOS, RPC (Remote Procedure Call), PPTP (Point-to-Point Tunneling Protocol), SIP (Session Initiation Protocol), NFS (Network File System)
FunctionsSession establishment/teardown, dialog control (simplex, half-duplex, full-duplex), synchronization checkpoints, session recovery after interruption

Communication Modes (Dialog Control)

ModeDirectionExample
SimplexOne-way onlyRadio broadcast, TV signal, fire alarm sensor
Half-DuplexBoth directions, but not simultaneouslyWalkie-talkie, CB radio
Full-DuplexBoth directions simultaneouslyTelephone, modern Ethernet, VoIP
💡 Exam Tip: TLS/SSL are often described as operating at Layer 5 (Session) and Layer 6 (Presentation), because TLS manages the secure session AND handles encryption. The CISSP exam may place TLS at L5, L6, or "between L4 and L7" — all are acceptable. The key point is that TLS sits above TCP (L4) and below Application (L7).

4. Layer 6 – Presentation

📖 Layer 6 (Presentation): Translates data between the network format and the application format. Handles data representation, character encoding, compression, and encryption/decryption. Often called the "syntax layer" because it ensures data is in a usable format.
AspectDetail
PDUData
FunctionsData translation (ASCII ↔ EBCDIC), character encoding (UTF-8, Unicode), compression (JPEG, MPEG), encryption/decryption (TLS/SSL), serialization (XML, JSON, ASN.1)
StandardsJPEG, MPEG, GIF, ASCII, Unicode, TLS/SSL (encryption aspect), XDR (External Data Representation)
🔑 Encryption at Layer 6: Encryption and decryption are conceptually a Layer 6 function — they transform data into a different representation. When the CISSP exam asks "at which layer does encryption occur?", Layer 6 (Presentation) is the traditional OSI answer. However, real-world encryption happens at various layers: IPSec at L3, TLS at L5/L6, application encryption at L7.
⚠️ Layer 6 in Practice: The Presentation layer is one of the least used layers in the TCP/IP model (it's absorbed into the Application layer). However, the CISSP exam tests the OSI model as a conceptual framework, so know what L6 handles: translation, compression, and encryption.

5. Layer 7 – Application

📖 Layer 7 (Application): The closest layer to the end user. Provides network services directly to user applications. This is NOT the application itself — it is the interface between the network and the application.
AspectDetail
PDUData
ProtocolsHTTP/HTTPS, FTP, SMTP, POP3, IMAP, DNS, DHCP, SNMP, LDAP, SSH, Telnet, NTP, RDP
DevicesApplication-layer firewalls / proxies, WAFs (Web Application Firewalls), load balancers (L7)
FunctionsNetwork services for applications: file transfer, email, directory services, web browsing, remote access

Layer 7 Security Concerns

AttackDescriptionCountermeasure
SQL InjectionInject malicious SQL through application input to manipulate the databaseParameterized queries, input validation, WAF
Cross-Site Scripting (XSS)Inject malicious scripts into web pages viewed by other usersOutput encoding, Content Security Policy (CSP), WAF
DNS Poisoning / SpoofingCorrupt DNS cache to redirect users to malicious sitesDNSSEC (DNS Security Extensions), DNS over HTTPS (DoH)
HTTP Flood (DDoS)Overwhelm web server with legitimate-looking HTTP requestsRate limiting, CDN, WAF, behavioral analysis
PhishingFraudulent emails/sites impersonating trusted entitiesSPF, DKIM, DMARC, user awareness training
💡 Exam Tip: When a CISSP question mentions "application-layer gateway" or "proxy firewall," both operate at Layer 7. They can inspect the full content of the traffic (including payload), unlike packet-filtering firewalls (L3) or stateful firewalls (L3-L4). A WAF is specifically an L7 firewall for web applications.

6. TCP/IP Model

📖 TCP/IP Model: A 4-layer practical networking model (also called the DoD model or Internet Protocol Suite) that describes how the internet actually works. Unlike the OSI model (which is conceptual/reference), TCP/IP is the implementation model.
TCP/IP LayerEquivalent OSI LayersProtocols
4. ApplicationL5 + L6 + L7 (Session + Presentation + Application)HTTP, FTP, SMTP, DNS, SSH, TLS, SNMP, DHCP
3. TransportL4 (Transport)TCP, UDP
2. InternetL3 (Network)IP, ICMP, IGMP, IPSec, ARP*
1. Network AccessL1 + L2 (Physical + Data Link)Ethernet, Wi-Fi (802.11), PPP, Frame Relay
⚠️ ARP Layer Placement: ARP placement varies by source. ARP resolves L3 (IP) addresses to L2 (MAC) addresses, so it bridges both layers. In the OSI model, it's typically placed at Layer 2 (Data Link). In the TCP/IP model, it's sometimes placed at the Internet layer. The CISSP exam may accept either — focus on what ARP does rather than its exact layer.

8. Encapsulation & Decapsulation

📖 Encapsulation: The process of adding layer-specific headers (and sometimes trailers) as data moves DOWN through the protocol stack. Each layer wraps the data from the layer above with its own control information.
ENCAPSULATION (Sending) ╔══════════════════════════════════════════════════╗ ║ Application Data ║ ║ ┌─────────────────────────────────────────────┐ ║ ║ │ L4 Header │ Data = Segment │ ║ ║ ├───────────┼──────────────────────────────────┤ ║ ║ │ L3 │ L4 │ Data = Packet │ ║ ║ ├────┼──────┼──────────────────────────────────┤ ║ ║ │L2 │ L3 │ L4 │ Data │ L2 Trlr = Frame │ ║ ║ ├────┼─────┼─────┼──────┼──────────────────────┤ ║ ║ │ 1010110101010101010101010101010 = Bits │ ║ ║ └──────────────────────────────────────────────┘ ║ ║ Direction: DOWN the stack → ║ ╚══════════════════════════════════════════════════╝
🔑 Decapsulation: The reverse process on the receiving host. As data moves UP the stack, each layer strips its header, processes the control information, and passes the remaining data to the layer above. Layer 2 also strips the trailer (FCS — Frame Check Sequence used for error detection).

9. Memory Tricks & Mnemonics

🧠 Upper Layers — "SPA":
Session (L5) — manages dialogs
Presentation (L6) — translates, compresses, encrypts
Application (L7) — network services for apps
🧠 TCP/IP vs. OSI: "TCP/IP merges the edges: top 3 → Application, bottom 2 → Network Access. Middle stays the same: Transport = Transport, Network = Internet."
🧠 Where Encryption Happens (OSI): "L6 = Presentation = Encryption" (conceptual). But remember: IPSec=L3, TLS=L5/L6, App encryption=L7. The exam answer for "which layer handles encryption?" is typically L6 unless a specific protocol is named.
🧠 Simplex / Half / Full: "Simplex = one-way Street, Half = Walkie-talkie, Full = Phone call"
