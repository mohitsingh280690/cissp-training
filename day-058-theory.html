3. Layer 2 Switching Fundamentals

📖 Layer 2 Switch: A network device that operates at the Data Link layer, forwarding Ethernet frames based on destination MAC addresses. Switches build a MAC address table (also called CAM table — Content Addressable Memory) by learning source MAC addresses from incoming frames.

How a Switch Forwards Frames

Switch MAC Address Table (CAM Table): ┌────────────────┬──────────┬───────────┐ │ MAC Address │ Port │ VLAN │ ├────────────────┼──────────┼───────────┤ │ AA:BB:CC:11:22:33 │ Fa0/1 │ VLAN 10 │ │ AA:BB:CC:44:55:66 │ Fa0/2 │ VLAN 10 │ │ AA:BB:CC:77:88:99 │ Fa0/3 │ VLAN 20 │ └────────────────┴──────────┴───────────┘ Frame Handling Logic: 1. KNOWN UNICAST → Forward only to the specific port (MAC in table) 2. UNKNOWN UNICAST → FLOOD to all ports in the VLAN (learning phase) 3. BROADCAST → FLOOD to all ports in the VLAN 4. MULTICAST → Flood to all ports unless IGMP snooping is configured Switches LEARN source MAC of every incoming frame. If destination MAC not in table → flood and learn.

CAM Table Overflow (MAC Flooding) Attack

AspectDetail
MechanismAttacker floods the switch with frames using thousands of different spoofed source MAC addresses. The CAM table fills up (it has a fixed size, typically 8,000–16,000 entries). Once full, the switch cannot learn new entries and begins to flood all frames to all ports — behaving like a hub.
ImpactAll unicast traffic is flooded — attacker in same VLAN can capture all traffic via a sniffer. Effectively turns a secure switch into a promiscuous hub.
ToolMacof (part of dsniff suite) — can send ~155,000 MAC entries per minute
DefencePort Security — limits the number of MAC addresses allowed per port (e.g., max 2). Excess MACs trigger: shutdown (default), restrict (drop + log), or protect (drop silently).
💡 Exam Tip: CAM table overflow (MAC flooding) causes a switch to behave like a hub and flood all traffic. The primary defence is port security with a maximum MAC address limit. This is distinct from MAC spoofing (where attacker uses a legitimate MAC) — port security with sticky MAC addresses addresses both.

4. Spanning Tree Protocol (STP)

📖 STP (Spanning Tree Protocol — IEEE 802.1D): A Layer 2 protocol that prevents network loops in Ethernet networks with redundant paths. Loops at Layer 2 cause broadcast storms that consume all network bandwidth and crash bridged networks. STP logically blocks redundant links, creating a loop-free tree topology.
STP Port States: Blocking → Listening → Learning → Forwarding → Disabled ┌─────────┬────────────────────────────────────────────────────┐ │ State │ Behaviour │ ├─────────┼────────────────────────────────────────────────────┤ │Blocking │ Receives STP BPDUs; does NOT forward user frames │ │Listening│ Receives + sends BPDUs; does NOT forward or learn │ │Learning │ Learns MAC addresses; does NOT forward user frames │ │Forwarding│ Fully operational — forwards frames + learns MACs │ │Disabled │ Administratively down │ └─────────┴────────────────────────────────────────────────────┘ STP Convergence Time: 30–50 seconds (old STP/802.1D) RSTP (Rapid STP, 802.1w): Convergence in ~1–3 seconds MSTP (Multiple STP, 802.1s): One STP instance per VLAN group

STP Security Concerns

AttackMechanismImpactDefence
Root Bridge Attack (STP Hijacking)Attacker connects a device and sends superior BPDU (lower Bridge ID / Priority = 0). Switch accepts it as new Root Bridge; traffic is re-routed through attacker's device.MITM — all traffic now passes through attacker; network disruption during STP re-convergenceBPDU Guard — shuts down any port that receives a BPDU (access ports should never receive BPDUs); Root Guard — prevents a port from becoming a root port (protects specific ports from receiving superior BPDUs)
STP DoSContinuously sending topology change BPDUs forces constant STP re-convergence, causing network instabilityNetwork disruption; loop briefly during convergenceBPDU Guard on all access ports; BPDU Rate Limiting
🔑 BPDU Guard vs. Root Guard:
BPDU Guard = "err-disable the port if it receives ANY BPDU." Configured on access (end-device) ports where BPDUs should NEVER arrive. Prevents rogue switches/hubs from connecting.
Root Guard = "this port is NOT allowed to become a root port." Configured on ports that connect to distribution switches. The switch can still participate in STP, but cannot become the root bridge via designated ports protected with Root Guard.

5. VLANs – Virtual Local Area Networks

📖 VLAN (Virtual LAN): A logical grouping of network devices into separate broadcast domains, regardless of physical location. VLANs provide Layer 2 segmentation — devices in different VLANs cannot communicate directly; they must go through a router or Layer 3 switch (inter-VLAN routing). Defined in IEEE 802.1Q.

Why VLANs Are a Security Control

Security BenefitExplanation
Broadcast Domain IsolationBroadcast traffic stays within a VLAN; attackers in one VLAN cannot use broadcasts to discover or attack devices in another VLAN
Traffic SegmentationSensitive systems (finance, HR, servers) are in their own VLAN; a compromise in the user VLAN does not automatically grant access to server VLAN
Regulatory SegmentationPCI-DSS requires cardholder data environment (CDE) to be segmented from non-CDE; VLANs are the standard mechanism
Reduced Attack SurfaceLayer 2 attacks (ARP poisoning, CAM flooding) are confined to a single VLAN
Simplified Firewall RulesFirewalls control inter-VLAN traffic; VLAN IDs become the source/destination in rules instead of individual IPs
⚠️ VLANs Are NOT a Complete Security Boundary: VLANs provide Layer 2 isolation, but inter-VLAN routing (Layer 3) must also be controlled by a firewall or ACL. Without firewall rules, a router will happily forward traffic between any two VLANs. VLANs define the segments; firewalls control the flows. Both are required.

VLAN Types

VLAN TypeDescriptionSecurity Use
Data VLANCarries end-user traffic; most VLANs are data VLANsSeparate by department, function, or sensitivity
Native VLANUntagged VLAN on trunk ports (default is VLAN 1); frames without an 802.1Q tag belong to the native VLANChange native VLAN from VLAN 1; always tag native VLAN; primary VLAN hopping attack vector
Management VLANUsed for out-of-band management (SSH to switch/router)Separate from user VLANs; restrict access to management VLAN tightly; not VLAN 1
Voice VLANCarries VoIP traffic; needs QoS prioritizationSeparate VoIP from data; VoIP has different encryption/authentication needs
Default VLAN (VLAN 1)All ports belong to VLAN 1 by default; carries STP/CDP/VTP management trafficNever use VLAN 1 for user data; assign all used ports to other VLANs

6. 802.1Q Trunking

📖 802.1Q Trunk: A trunk link carries traffic for multiple VLANs between switches (or between a switch and a router/firewall). Each frame is tagged with a 4-byte 802.1Q header that includes the VLAN ID (VID) (12 bits → 4,094 usable VLANs). The receiving device reads the tag and forwards the frame to the correct VLAN.
802.1Q Frame Tag (inserted after source MAC): ┌──────────┬──────────┬──────────┬──────────┬────────────────┐ │Dest MAC │ Src MAC │ 0x8100 │ Tag Ctrl │ Original Data │ │(6 bytes) │(6 bytes) │(EtherType│ (2 bytes)│ │ │ │ │ = 802.1Q)│ │ │ └──────────┴──────────┴──────────┴──────────┴────────────────┘ │ ┌───────────┴───────────┐ │ 3 bits PCP (Priority) │ │ 1 bit DEI (Drop Elg.) │ │ 12 bits VLAN ID (VID) │ └───────────────────────┘ VID 0 = no VLAN (untagged / priority only) VID 1 = default VLAN (avoid using) VID 4095 = reserved Usable: VIDs 2–4094 (4,093 VLANs)

Access Ports vs. Trunk Ports

Port TypeDescriptionSecurity Configuration
Access PortConnects to an end device (PC, server, printer). Belongs to exactly one VLAN. Frames are untagged on the wire — the switch adds/removes VLAN tag internally.Assign to correct VLAN; enable port security; enable BPDU Guard; disable DTP (auto-negotiation)
Trunk PortConnects to another switch, router, or firewall. Carries multiple VLANs simultaneously. Frames are tagged with 802.1Q headers.Explicitly allow only required VLANs; change native VLAN; disable DTP; restrict to authorized uplinks only
🔑 DTP (Dynamic Trunking Protocol): Cisco proprietary protocol that automatically negotiates trunk links. An attacker can send DTP negotiation frames from an access port to convince the switch to form a trunk — gaining access to all VLANs. Always disable DTP on access ports with switchport nonegotiate. Trunk links should be statically configured, not auto-negotiated.

7. VLAN Hopping Attacks

📖 VLAN Hopping: An attack that allows traffic from one VLAN to bypass Layer 2 isolation and reach another VLAN without passing through a router/firewall. There are two primary techniques: switch spoofing and double tagging.

Attack Technique 1: Switch Spoofing

AspectDetail
MechanismAttacker's device sends DTP frames to negotiate a trunk link with the switch. If DTP is enabled (default on Cisco), the switch's access port converts to a trunk port. Attacker now receives traffic from all VLANs on that trunk.
PrerequisiteDTP must be active on the access port (which is the default on many Cisco switches)
DefenceDisable DTP on all access ports (switchport mode access + switchport nonegotiate); explicitly configure access or trunk — never leave in default/auto mode

Attack Technique 2: Double Tagging (802.1Q Double Encapsulation)

Double Tagging Attack: Attacker frame has TWO 802.1Q tags: [Outer Tag: VLAN 1 (native)] [Inner Tag: VLAN 10 (target)] Step 1: Attacker (on VLAN 1 = native) sends double-tagged frame to Switch A Step 2: Switch A strips OUTER tag (VLAN 1 = native = untagged) and forwards the INNER-tagged frame (still tagged as VLAN 10) out the trunk to Switch B Step 3: Switch B sees a VLAN 10 frame → delivers it to VLAN 10 hosts Result: Attacker sent a frame INTO VLAN 10 across a VLAN boundary! KEY: Return traffic CANNOT traverse back (one-directional attack) This is a one-way attack — useful for DoS or sending unsolicited frames, not for interactive sessions.
AspectDetail
RequirementAttacker must be in the NATIVE VLAN (or a VLAN that matches the native VLAN); double tagging exploits the native VLAN's untagged behaviour
LimitationOne-way attack — return traffic cannot directly reach the attacker via the same path; useful for sending malformed frames, not establishing sessions
DefenceChange the native VLAN to an unused VLAN ID (e.g., VLAN 999) on all trunk ports; do not assign any user ports to the native VLAN; explicitly tag the native VLAN (vlan dot1q tag native)
💡 Exam Tip: Double tagging works only when the attacker is on the same VLAN as the native VLAN of the trunk. The #1 VLAN hopping defence the CISSP exam expects: "Change the native VLAN on all trunk links to an unused VLAN ID that is not assigned to any access ports." This eliminates double tagging entirely.

8. Port Security & Switch Hardening

Port Security

📖 Port Security: A switch feature that restricts access to a port based on MAC address. You can configure a maximum number of allowed MAC addresses per port, and specify what happens when a violation is detected (shutdown, restrict, or protect).
ConfigurationDescription
Maximum MAC AddressesLimit the number of source MACs allowed; default 1; typically set to 1–3 for workstations; higher for IP phones or hubs
Static MACManually specify which MAC addresses are allowed on the port; most secure but high admin overhead
Sticky (Dynamic) MACSwitch learns and locks the first N MACs dynamically; saves to running config; good balance of automation and security
Violation: ShutdownPort is err-disabled (goes completely down); requires manual intervention to re-enable; most secure default
Violation: RestrictDrops excess frames; logs SNMP trap and syslog; port stays up; useful for monitoring without disruption
Violation: ProtectDrops excess frames; NO logging/alerting; port stays up; least visible — the worst option for security awareness

Switch Hardening Checklist

Hardening StepWhy It Matters
Disable unused ports and assign to an unused "parking" VLANOpen unused ports allow unauthorized device connection
Disable DTP (auto negotiation) on all access portsPrevents switch spoofing VLAN hopping attack
Change native VLAN from VLAN 1 to an unused VLANPrevents double tagging VLAN hopping; eliminates VLAN 1 as attack surface
Enable BPDU Guard on all access portsPrevents STP hijacking by rogue switches
Enable DHCP snooping to build binding tablePrevents rogue DHCP servers; feeds DAI
Enable Dynamic ARP Inspection (DAI)Prevents ARP poisoning / MITM
Enable IP Source Guard (IPSG)Filters traffic by IP/MAC from DHCP snooping binding table — prevents IP spoofing
Restrict SNMP to read-only; upgrade to SNMPv3Prevents management plane compromise
Enable management VLAN separate from VLAN 1Management traffic segmented from user traffic
Use SSH, not Telnet, for managementEncrypted management sessions
Enable 802.1X port authenticationRequires devices to authenticate (EAP/RADIUS) before gaining network access
🔑 802.1X Port-Based Network Access Control (PNAC): 802.1X is the IEEE standard for port-based authentication. A device connects to a switch port but traffic is blocked until the device authenticates via EAP (Extensible Authentication Protocol) to a RADIUS server. Used to enforce network access control (NAC) — only authenticated devices get VLAN assignments. This is the gold-standard for preventing unauthorized physical connections.

9. Private VLANs (PVLANs)

📖 Private VLAN (PVLAN — RFC 5517): Subdivides a VLAN into sub-VLANs with restricted communication. Used when devices need to access a common service (e.g., a shared gateway) but should not be able to talk to each other — typical in hosting environments, DMZs, and hotel/guest networks.
PVLAN Port TypeCan Talk ToUse Case
Promiscuous PortAll ports in the PVLANRouter, firewall, shared server — the "gateway" port
Isolated PortOnly the promiscuous port; NOT to other isolated portsDMZ servers, hotel room ports, guest WiFi — complete isolation between tenants
Community PortOther ports in the same community AND the promiscuous portGroups of servers that need to talk to each other (e.g., web cluster) but not to other groups
💡 Exam Tip: The key PVLAN exam scenario: "A hosting provider wants servers on the same subnet to not communicate with each other, only with the upstream router." Answer = Private VLAN with isolated ports + promiscuous uplink. This is also useful in the DMZ for servers that don't need east-west communication.

11. Memory Tricks & Mnemonics

🧠 VLAN Hopping Defences — "Native is Nasty; DTP is Dangerous":
Double Tagging exploits the NATIVE VLAN → Change native VLAN to unused ID, tag native
Switch Spoofing exploits DTP → Disable DTP on all access ports (nonegotiate)
Remember: "Both attacks target the trunk. Harden the trunk and the access port."
🧠 STP Defence Pair — "Guard the Bridge, Guard the BPDU":
BPDU Guard = protects the access port from receiving BPDUs (shuts down if it does)
Root Guard = protects a trunk/uplink port from accepting a new Root Bridge
BPDU Guard = end-device ports; Root Guard = uplink/distribution ports.
🧠 Port Security Violation Actions — "SRP":
Shutdown = err-disable (most secure; raises alarm by going down)
Restrict = drops + logs (security + no disruption)
Protect = drops silently (worst choice — no visibility)
Default is Shutdown in most implementations.
🧠 Layer 2 Security Tech Stack — "PS DAI DHCP BPDU 802":
Port Security → prevents CAM overflow / unauthorized MACs
DAI → prevents ARP poisoning
DHCP Snooping → prevents rogue DHCP; feeds DAI
BPDU Guard → prevents STP hijack
802.1X → authenticates devices before network access
Deploy all five for a hardened Layer 2 environment.
🧠 PVLAN Ports — "PICs are Private":
Promiscuous = talks to all
Isolated = talks only to promiscuous
Community = talks to community + promiscuous
Use PVLAN when tenants share a subnet but must not communicate laterally.
