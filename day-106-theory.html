2. Real-World Organizational Relevance

SIEM is critical infrastructure for any organization with regulatory compliance requirements. PCI DSS Requirement 10 mandates log monitoring. HIPAA requires audit controls. SOX demands computer-generated audit trails. Without SIEM, these requirements are nearly impossible to meet at scale.

However, SIEM is also one of the most commonly underutilized security investments. Organizations deploy SIEM but fail to tune correlation rules, address alert fatigue, or staff the SOC adequately — resulting in a false sense of security.

3. SIEM Architecture & Components

SIEM (Security Information and Event Management): A platform that combines Security Information Management (SIM) — long-term log storage and compliance reporting — with Security Event Management (SEM) — real-time monitoring, correlation, and alerting. SIEM provides a single pane of glass for security visibility.

Correlation — The SIEM Differentiator

Event Correlation: The process of analyzing events from multiple sources to identify patterns that indicate a security incident. Individual events may appear benign; correlated events reveal attacks. Example: a failed VPN login (VPN log) + successful login from unusual country (IAM log) + large file download (proxy log) + after-hours activity (time correlation) = potential compromised account and data exfiltration.

Time Synchronization (NTP)

⚠️ NTP is Critical for SIEM: All systems must synchronize to the same time source. Without consistent timestamps, correlation fails — events from different systems cannot be accurately sequenced. Use NTP (Network Time Protocol) pointed at authoritative time sources. For forensic purposes, document the time zone and synchronization status of all evidence sources.

5. SIEM Use Cases & Rules

Use Case CategoryExample Rules
Brute force detectionAlert: more than 10 failed logins to same account within 5 minutes
Account compromiseAlert: successful login from two geographically distant locations within impossible travel time
Privilege escalationAlert: user added to admin group; new service account created; sudo/runas from unexpected account
Data exfiltrationAlert: large outbound data transfer to unknown destination; DNS tunneling patterns; unusual cloud storage uploads

6. SOAR – Security Orchestration, Automation & Response

SOAR: A platform that integrates with SIEM and other security tools to automate repetitive response tasks, orchestrate workflows across tools, and manage the incident response lifecycle. SOAR takes the alerts SIEM generates and automates the response.

SIEM vs SOAR

CapabilitySIEMSOAR
Primary functionDetection — collect, correlate, alertResponse — automate, orchestrate, manage
Log managementCore capabilityNot a primary function
Correlation rulesCore capabilityConsumes SIEM alerts as triggers
Automated responseLimited (alert + basic actions)Core capability — complex multi-step playbooks
Case managementBasicFull incident lifecycle tracking
Tool integrationIngests logs from toolsBidirectional — reads from AND writes to tools (firewall, EDR, ticketing, email)

SOAR Automation Examples

  • Phishing response: Receive phishing report → extract URLs/attachments → check threat intel → quarantine email from all mailboxes → block sender → notify reporter — all automated
  • Malware containment: EDR detects malware → SOAR isolates endpoint → captures forensic data → creates incident ticket → notifies analyst
  • Threat intel enrichment: SIEM generates alert with IP → SOAR queries VirusTotal, WHOIS, threat intel feeds → enriches incident with context → presents analyst with full picture

7. UEBA – User & Entity Behavior Analytics

UEBA: Uses machine learning and statistical models to establish behavioral baselines for users and entities (devices, applications, servers) and detect anomalies that deviate from normal patterns. UEBA excels at detecting insider threats and compromised accounts that signature-based detection misses.

UEBA vs Traditional SIEM Rules

FeatureSIEM RulesUEBA
Detection methodPredefined rules/signaturesBehavioral baselines + anomaly detection
Known attacksEffective (write a rule for known pattern)Not primary purpose
Unknown/novel attacksCannot detect what isn't definedDetects deviations from normal behavior regardless of attack type
Insider threatsDifficult — insiders use legitimate accessExcels — detects unusual behavior even with legitimate credentials
TuningManual rule writing and maintenanceSelf-learning with supervised/unsupervised ML
Exam Context — UEBA for Insider Threats: When the exam asks about detecting insider threats or compromised accounts that are using legitimate credentials, UEBA is typically the best answer. Traditional rules fail because the access is technically authorized — only the BEHAVIOR is abnormal.

8. Threat Hunting

Threat Hunting: The proactive, iterative search for threats that have evaded existing detection mechanisms. Unlike monitoring (reactive — wait for alerts), hunting is hypothesis-driven: the hunter formulates a hypothesis about potential attacker activity and searches for evidence to confirm or deny it.

Threat Hunting vs Monitoring

CharacteristicSecurity MonitoringThreat Hunting
ApproachReactive — wait for alertsProactive — search for hidden threats
TriggerAutomated alert firesHypothesis, intelligence, or intuition
Skill levelSOC Tier 1-2 analystsSenior analysts with deep expertise
OutputIncident ticketsNew detection rules, IoCs, process improvements, found threats
AssumesDetection rules will catch threatsAssumes threats are ALREADY in the environment, undetected

Hunting Methodologies

  • Intelligence-driven: Use threat intelligence (new IoCs, TTPs from reports) to search the environment for matching activity
  • Hypothesis-driven: "If an attacker compromised our VPN, they would likely perform lateral movement via RDP to our file servers" → search for RDP connections from VPN subnet to file servers
  • Analytics-driven: Use UEBA or statistical analysis to identify outliers that warrant investigation

10. Memory Tricks & Mnemonics

🧠 SIEM = SIM + SEM
SIM = Store logs, compliance reports (Information Management — historical)
SEM = Real-time monitoring, alerts, correlation (Event Management — operational)
"SIM stores; SEM sees"
🧠 SIEM Pipeline — "CNECA"
Collect → Normalize → Enrich → Correlate → Alert
"Can Never Evade Correlation Alerts"
🧠 SIEM Detects, SOAR Responds
SIEM = "What happened?" (visibility)
SOAR = "What do we do about it?" (action)
"SIEM is your eyes; SOAR is your hands"
🧠 Threat Hunting Assumption
"They're already in." Threat hunting assumes compromise has already occurred and seeks to find what automated tools missed.
