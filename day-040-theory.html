3. Trusted Computing Base (TCB)

📖 Trusted Computing Base (TCB): The totality of protection mechanisms within a computer system — including hardware, firmware, and software — that is responsible for enforcing the security policy. The TCB is the "trusted foundation" upon which all security depends.

TCB Components

ComponentRole in TCB
HardwareCPU, memory management unit (MMU), I/O controllers
FirmwareBIOS/UEFI, boot loader, microcode
SoftwareOS kernel, security kernel, reference monitor
ConfigurationSecurity policies, access control tables
🔑 Key Principles:
• TCB should be as small as possible — smaller TCB = easier to verify = more trustworthy
• Everything INSIDE the TCB must be trusted
• Everything OUTSIDE the TCB is untrusted and must be mediated by the TCB
• If the TCB is compromised, ALL security guarantees are void
• TCSEC B3 and A1 require a minimal (small) TCB
⚠️ Critical Understanding: The TCB is NOT the entire operating system. It is only the subset of the system responsible for security enforcement. A well-designed TCB excludes non-security-relevant code to minimize the attack surface.

4. Security Perimeter

📖 Security Perimeter: The boundary that separates the trusted TCB from the untrusted remainder of the system. Everything inside the security perimeter is part of the TCB; everything outside is untrusted.

  │           Security Perimeter                 │

💡 Exam Tip: Communication across the security perimeter must occur through trusted channels/paths. Any path that bypasses the security perimeter is a covert channel — a vulnerability.

5. Reference Monitor & Security Kernel

📖 Reference Monitor: An abstract concept — a theoretical access control mechanism that mediates ALL access by subjects to objects. It is the "gatekeeper" that checks every access request against the security policy.
📖 Security Kernel: The actual implementation of the reference monitor concept in hardware, firmware, and software. If the reference monitor is the blueprint, the security kernel is the building.

Reference Monitor Properties

PropertyMeaningWhy It Matters
Complete MediationMust mediate EVERY access request — no exceptionsBypassing even one check could allow unauthorized access
Tamper-ProofCannot be altered or bypassed by untrusted subjectsIf attackers can modify it, security policy becomes meaningless
Verifiable (Small)Must be small enough to be thoroughly tested and verifiedA large, complex implementation cannot be proven correct
🔑 Reference Monitor vs. Security Kernel:
• Reference Monitor = abstract CONCEPT (theoretical ideal)
• Security Kernel = concrete IMPLEMENTATION (actual code/hardware)
• The security kernel implements the reference monitor concept
• Three properties: Complete Mediation, Tamper-Proof, Verifiable

6. Trusted Platform Module (TPM)

📖 Trusted Platform Module (TPM): A dedicated hardware chip (or firmware equivalent) on the motherboard that provides cryptographic functions, secure key storage, and platform integrity measurement. Standardized as ISO/IEC 11889.

TPM Capabilities

FunctionDescription
Secure Key StorageStores encryption keys, certificates, and passwords in tamper-resistant hardware
Platform IntegrityMeasures boot process components (PCR registers); detects unauthorized changes
Secure BootValidates that only trusted firmware and OS components load during boot
Full Disk EncryptionStores BitLocker/LUKS keys; releases only if platform integrity check passes
Remote AttestationProves to a remote party that the platform is in a known-good state
Random Number GenerationProvides hardware-based random number generator for cryptographic operations
🔑 Key Facts:
• TPM is a chip on the motherboard (hardware root of trust)
• Stores keys that never leave the chip
• TPM 2.0 is required for Windows 11
• Uses Platform Configuration Registers (PCRs) to store integrity measurements
• Enables measured boot — each component measures the next before loading
💡 Exam Tip: TPM = hardware chip on motherboard = root of trust. If the question says "hardware-based secure key storage on the endpoint," think TPM. If it says "central server for protecting CA keys," think HSM.

7. Hardware Security Module (HSM)

📖 Hardware Security Module (HSM): A dedicated, tamper-resistant hardware device that manages cryptographic keys and performs cryptographic operations. HSMs are standalone devices (often rack-mounted or USB-attached) used for high-value key management.

TPM vs. HSM

FeatureTPMHSM
Form FactorChip on motherboardStandalone device (rack, USB, PCI card)
ScopeProtects ONE endpointServes multiple systems (centralized)
Primary UsePlatform integrity, secure boot, disk encryptionPKI key management, code signing, payment processing
CostLow (built into motherboard)High ($10K–$100K+ for enterprise HSMs)
PerformanceBasic crypto operationsHigh-performance crypto acceleration
Tamper ResponseTamper-resistantTamper-evident AND tamper-responsive (zeroizes keys)
StandardISO 11889 / TCG TPM 2.0FIPS 140-2/140-3
🔑 HSM Use Cases:
• Protecting Certificate Authority (CA) root keys
• Payment card processing (PCI DSS requires HSM for PIN management)
• Code signing keys
• Database encryption key management
• Cloud HSM services (AWS CloudHSM, Azure Dedicated HSM)

8. Trusted Paths & Trusted Channels

📖 Trusted Path: A secure, tamper-proof communication channel between a user and the TCB. Ensures the user is communicating directly with the security system and not with a spoofed interface (e.g., Ctrl+Alt+Del for Windows login — Secure Attention Sequence).
📖 Trusted Channel: A secure communication path between two components of the TCB (process-to-process within the trusted system), or between two trusted systems.
TypeBetweenExample
Trusted PathUser ↔ TCBCtrl+Alt+Del (SAS) → ensures real login prompt; smart card reader direct to OS
Trusted ChannelTCB ↔ TCBIPSec tunnel between two trusted servers; encrypted inter-process communication within kernel
⚠️ Covert Channels: A communication path that violates the security policy — it transfers information in a way NOT intended by the system designers. Covert channels bypass the reference monitor and are classified as:
• Covert Storage Channel: Information transmitted through shared storage (e.g., disk space, file locks)
• Covert Timing Channel: Information transmitted through timing of operations (e.g., CPU load patterns)

9. Memory Tricks & Mnemonics

🧠 "TCB = Trusted Core Building": The TCB is the core building of security — if any brick in the foundation fails, the whole building collapses. Keep it as small as possible.
🧠 "Reference Monitor: CTV":
• Complete Mediation (checks EVERYTHING)
• Tamper-proof (can't be modified)
• Verifiable (small enough to test)
The reference monitor is your "CTV" — it's like a security camera that Covers everything, is Tamper-proof, and is Verifiable.
🧠 "TPM = This PC's Module, HSM = High-Security Machine":
• TPM is a chip IN your PC (motherboard)
• HSM is a separate machine for high-security crypto operations
