3. IPv4 Addressing

📖 IPv4 Address: A 32-bit logical address written in dotted-decimal notation (4 octets separated by dots). Each octet ranges from 0 to 255. Example: 192.168.1.100. Total theoretical address space: 2³² = ~4.3 billion addresses.
IPv4 Address Structure (32 bits): 192 . 168 . 1 . 100 ┌────────┬────────┬────────┬────────┐ │11000000│10101000│00000001│01100100│ └────────┴────────┴────────┴────────┘ Octet 1 Octet 2 Octet 3 Octet 4 (8 bits) (8 bits) (8 bits) (8 bits)
🔑 Network vs. Host Portion: Every IP address has two parts: the network portion (identifies the network) and the host portion (identifies the specific device). The subnet mask determines the dividing line between these two parts.

⚠️ Special Addresses:
  • 127.0.0.0/8 — Loopback (127.0.0.1 = localhost). Not a Class A usable range.
  • 0.0.0.0 — "This host" / default route
  • 255.255.255.255 — Limited broadcast (local subnet only)
  • 169.254.0.0/16 — APIPA (Automatic Private IP Addressing) — assigned when DHCP fails
💡 Exam Tip: Classful addressing is largely obsolete (replaced by CIDR), but the CISSP exam still tests on address classes, especially Class A/B/C ranges and private address ranges. Know the first-octet boundaries.

5. Private Address Ranges (RFC 1918)

📖 RFC 1918: Defines three private IP address ranges that are not routable on the public internet. Any organization can use these internally. Routers on the internet will drop packets with private source/destination addresses.
ClassPrivate RangeCIDRNumber of Addresses
A10.0.0.0 – 10.255.255.25510.0.0.0/816,777,216
B172.16.0.0 – 172.31.255.255172.16.0.0/121,048,576

💡 Exam Tip: The most commonly tested private range boundary is Class B: 172.16.x.x through 172.31.x.x. Students often forget it stops at 172.31, not 172.255. The CIDR for this range is /12, not /16.

6. CIDR Notation

📖 CIDR (Classless Inter-Domain Routing): Replaces classful addressing with variable-length subnet masks. Written as IP/prefix-length (e.g., 192.168.1.0/24). The prefix length indicates how many leading bits are the network portion. Introduced in 1993 to slow IPv4 address exhaustion and reduce routing table sizes.

🔑 Usable Hosts Formula: Usable hosts = 2^(32 − prefix) − 2. The two subtracted addresses are the network address (all host bits = 0) and the broadcast address (all host bits = 1). Exception: /31 point-to-point links (RFC 3021) and /32 host routes.

7. Subnetting Fundamentals

📖 Subnetting: The practice of dividing a single network into smaller logical networks (subnets) by borrowing bits from the host portion of the address. This is a critical security control because it creates network segments that can be isolated with firewalls and ACLs.

Why Subnet? (Security Benefits)

BenefitExplanation
Network SegmentationIsolate sensitive systems (e.g., HR, finance, servers) into separate subnets
Reduced Broadcast DomainSmaller subnets = less broadcast traffic = better performance and less information leakage
Access ControlFirewalls and ACLs between subnets can restrict lateral movement
MonitoringTraffic between subnets passes through routers/firewalls that can log and inspect
ContainmentA compromise in one subnet is harder to spread to others (blast radius reduction)

Subnetting Example

Network: 192.168.10.0/24 → Split into 4 subnets (/26) Subnet 1: 192.168.10.0/26 (Hosts: .1 – .62) → Servers Subnet 2: 192.168.10.64/26 (Hosts: .65 – .126) → Finance Subnet 3: 192.168.10.128/26 (Hosts: .129 – .190) → HR Subnet 4: 192.168.10.192/26 (Hosts: .193 – .254) → General Borrowing 2 bits from host portion: /24 → /26 = 2² = 4 subnets, each with 62 usable hosts
💡 Exam Tip: For the CISSP, focus on WHY you subnet (security segmentation, containment, monitoring) rather than performing complex subnetting math. However, know the powers of 2, how to calculate usable hosts (2^n − 2), and the concept of borrowing bits.

8. IPv6 Addressing

📖 IPv6: 128-bit addresses written as eight groups of four hexadecimal digits separated by colons (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334). Provides approximately 3.4 × 10³⁸ unique addresses — essentially unlimited.

⚠️ IPv6 Security Concerns:
  • Dual-stack risks: Running IPv4 and IPv6 simultaneously means two attack surfaces; if IPv6 isn't monitored, attackers can use it to bypass IPv4 security controls
  • IPv6 tunneling: Protocols like Teredo and 6to4 can tunnel IPv6 over IPv4, potentially bypassing firewalls
  • Router Advertisement (RA) spoofing: Attackers can send rogue RAs to redirect traffic (mitigated with RA Guard)
  • Large address space: Traditional scanning is impractical, but this makes asset discovery harder for defenders too

9. NAT & PAT

📖 NAT (Network Address Translation): Translates private (RFC 1918) IP addresses to public IP addresses at a network boundary device (router/firewall). This conserves public IPv4 addresses and hides the internal network structure from external observers.
TypeDescriptionMapping
Static NAT1:1 mapping — one private address permanently maps to one public address10.0.0.5 ↔ 203.0.113.5
Dynamic NATMany-to-many — private addresses map to a pool of public addresses on first-come basis10.0.0.x ↔ pool of 203.0.113.x
PAT (Port Address Translation)Many-to-one — multiple private addresses share a single public IP, differentiated by port numbers. Also called NAT Overload.10.0.0.5:1025 → 203.0.113.1:40001
10.0.0.6:1026 → 203.0.113.1:40002
🔑 NAT Security Implications:
  • Obscurity (not security): NAT hides internal addresses but is NOT a security control by itself — it should be combined with firewalls
  • Breaks end-to-end: NAT modifies packet headers, which can break IPSec AH (Authentication Header) since AH authenticates the original header
  • Stateful: NAT devices maintain a translation table — they must track all active connections
  • PAT is most common: Home routers and most corporate internet gateways use PAT to share one public IP among many internal hosts
💡 Exam Tip: The CISSP exam does NOT consider NAT a security control — it's an address conservation mechanism. If a question asks "which control hides internal IP addresses from external attackers?", the answer is typically NAT, but remember it provides obscurity, not security. Always pair NAT with proper firewall rules.

10. Memory Tricks & Mnemonics

🧠 Class A/B/C First-Octet Ranges:
A = 1–126 (starts low, big networks)
B = 128–191 (middle range, medium networks)
C = 192–223 (starts high, small networks)
Remember: "127 is loopback, so Class A stops at 126"
🧠 Private Ranges — "10, 172, 192":
10.0.0.0/8 — the whole Class A
172.16.0.0/12 — Class B (16 through 31 only!)
192.168.0.0/16 — Class C (most familiar: home routers)
Memory aid: "1 ten, 172 sixteen, 192 one-sixty-eight"
🧠 NAT Types — "S-D-P":
Static = 1:1 (Single map)
Dynamic = many:pool (Dynamic pool)
PAT = many:1 (Port-based sharing)
🧠 IPv6 — "128 bits, 8 groups, hex colons": IPv6 is 128 bits = 8 groups × 16 bits per group, written in hex with colons. IPv4 has 32 bits = 4 octets in decimal with dots.
