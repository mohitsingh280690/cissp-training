3. OSI Model Overview

📖 OSI Model: The Open Systems Interconnection model is a conceptual 7-layer framework standardized by ISO (ISO 7498) that describes how data moves from an application on one device to an application on another device. Each layer serves the layer above it and is served by the layer below it.
OSI MODEL — 7 LAYERS ╔═══════════════════════════════════╗ ║ Layer 7 │ Application ║ ← User interface, HTTP, FTP, SMTP ║ Layer 6 │ Presentation ║ ← Data format, encryption, compression ║ Layer 5 │ Session ║ ← Session management, RPC, NetBIOS ╠═══════════╪═══════════════════════╣ (covered in Day 54) ║ Layer 4 │ Transport ║ ← TCP/UDP, ports, segmentation ║ Layer 3 │ Network ║ ← IP, routing, logical addressing ║ Layer 2 │ Data Link ║ ← MAC, switches, frames ║ Layer 1 │ Physical ║ ← Cables, hubs, signals ╚═══════════════════════════════════╝ ↑ Data moves UP on receiving host ↓ Data moves DOWN on sending host

4. Layer 1 – Physical

📖 Layer 1 (Physical): Handles the transmission of raw bitstreams over a physical medium. Concerned with electrical signals, light pulses, radio frequencies, connectors, cable specifications, and timing.
AspectDetail
PDUBits
DevicesHubs, repeaters, cables, modems, wireless access points (physical radio)
Protocols/StandardsEthernet (physical specs: 10BASE-T, 100BASE-TX, 1000BASE-T), RS-232, DSL, SONET, 802.11 (physical radio)
AddressingNone — no logical or physical addressing at this layer
FunctionDefines voltage levels, data rates, maximum distances, physical connectors, encoding schemes

Layer 1 Security Concerns

Attack/ThreatDescriptionCountermeasure
Wiretapping / Cable tappingPhysical interception of signals on copper or fiberFiber optic (harder to tap), conduit protection, physical access controls
Electromagnetic emanationsTEMPEST — devices emit EM radiation that can be captured and decodedTEMPEST shielding (Faraday cage), zone-based facility design
Signal jammingInterference with wireless signals to cause denial of serviceFrequency hopping (FHSS), spectrum monitoring, physical shielding
Physical destructionCutting cables, damaging hardwareRedundant paths, physical security, cable plant protection
💡 Exam Tip: A hub operates at Layer 1 — it repeats signals to all ports (no intelligence). A switch operates at Layer 2 — it uses MAC addresses to forward frames only to the correct port. Know this distinction.

5. Layer 2 – Data Link

📖 Layer 2 (Data Link): Provides reliable transfer of data across the physical link. Handles framing, physical addressing (MAC), error detection, and media access control. Divided into two sublayers: LLC (Logical Link Control) and MAC (Media Access Control).
AspectDetail
PDUFrames
DevicesSwitches, bridges, NICs (network interface cards)
ProtocolsEthernet (802.3), Wi-Fi (802.11), PPP, ARP, LLDP, STP (Spanning Tree Protocol)
AddressingMAC addresses — 48-bit (6 bytes), burned-in hardware address, format: AA:BB:CC:DD:EE:FF; first 3 bytes = OUI (manufacturer), last 3 = device
SublayersLLC: multiplexing, flow control, error detection. MAC: media access (CSMA/CD for Ethernet, CSMA/CA for Wi-Fi)

Layer 2 Security Concerns

AttackDescriptionCountermeasure
ARP Spoofing / PoisoningAttacker sends fake ARP replies to associate their MAC with a victim's IP, enabling MitM or DoSDynamic ARP Inspection (DAI), static ARP entries, 802.1X
MAC FloodingOverwhelm switch's CAM table with random MACs, forcing it to broadcast all traffic (hub behavior)Port security (limit MACs per port), 802.1X
VLAN HoppingAttacker jumps between VLANs via switch spoofing (DTP) or double tagging (802.1Q)Disable DTP; set native VLAN to unused VLAN; explicitly configure trunk ports
STP ManipulationAttacker sends forged BPDU frames to become the root bridge, redirecting trafficBPDU Guard, Root Guard on access ports
🔑 ARP (Address Resolution Protocol): Resolves IP addresses (Layer 3) to MAC addresses (Layer 2). ARP operates at Layer 2 but bridges L2 and L3. ARP has no authentication — any device can claim any IP-to-MAC mapping, making ARP spoofing trivial.

6. Layer 3 – Network

📖 Layer 3 (Network): Handles logical addressing and routing — determining the best path for data to travel from source to destination across multiple networks. This is where IP addressing and routing protocols operate.
AspectDetail
PDUPackets
DevicesRouters, Layer 3 switches, firewalls (packet-filtering)
ProtocolsIP (IPv4/IPv6), ICMP, IGMP, IPSec, routing protocols (OSPF, BGP, RIP, EIGRP)
AddressingIP addresses — logical addresses assigned to interfaces; IPv4 (32-bit), IPv6 (128-bit)
FunctionsRouting, logical addressing, fragmentation/reassembly, TTL (time to live)

Layer 3 Security Concerns

AttackDescriptionCountermeasure
IP SpoofingForge source IP address to bypass ACLs or impersonate trusted hostsIngress/egress filtering (BCP 38/RFC 2827), uRPF (unicast reverse path forwarding)
ICMP AttacksSmurf (ICMP amplification), Ping of Death, ICMP redirect manipulationRate-limit ICMP, disable directed broadcasts, filter ICMP at firewalls
Route HijackingBGP hijacking — announce false routes to redirect internet trafficRPKI (Resource Public Key Infrastructure), BGP route filtering, MANRS
Fragmentation AttacksOverlapping fragments (Teardrop) or tiny fragments that evade firewall inspectionFragment reassembly at firewall, drop overlapping fragments
💡 Exam Tip: IPSec operates at Layer 3. It provides encryption (ESP) and authentication (AH) at the network layer. This means it can protect all traffic between two hosts/networks regardless of the application — unlike TLS which operates at a higher layer.

7. Layer 4 – Transport

📖 Layer 4 (Transport): Provides end-to-end communication between applications on different hosts. Handles segmentation, flow control, error recovery, and port-based addressing. TCP and UDP are the primary transport protocols.
AspectDetail
PDUSegments (TCP) / Datagrams (UDP)
DevicesStateful firewalls, load balancers
ProtocolsTCP (Transmission Control Protocol), UDP (User Datagram Protocol), TLS (spans L4-L7 but primarily L4-L5)
AddressingPort numbers — 16-bit (0–65535); Well-known: 0–1023; Registered: 1024–49151; Dynamic: 49152–65535
FunctionsSegmentation/reassembly, flow control, error recovery (TCP), multiplexing via ports

TCP vs. UDP

FeatureTCPUDP
ConnectionConnection-oriented (3-way handshake: SYN → SYN-ACK → ACK)Connectionless
ReliabilityGuaranteed delivery (acknowledgments, retransmission)Best-effort (no guarantees)
OrderingOrdered delivery (sequence numbers)Unordered
Flow ControlYes (sliding window)No
OverheadHigher (20+ byte header)Lower (8-byte header)
Use CasesHTTP, HTTPS, FTP, SMTP, SSHDNS (queries), DHCP, VoIP, streaming, SNMP, TFTP

Layer 4 Security Concerns

AttackDescriptionCountermeasure
SYN FloodSend massive SYN requests without completing handshake; exhausts server's connection tableSYN cookies, rate limiting, firewall SYN proxy
Port ScanningProbe open ports to discover running services (Nmap)Close unnecessary ports, IDS/IPS detection, firewall rules
Session HijackingPredict or steal TCP sequence numbers to take over an established sessionEncrypted sessions (TLS), random initial sequence numbers, IPSec
UDP FloodSend mass UDP packets to random ports; host replies with ICMP unreachable, consuming resourcesRate limiting, firewall rules, DDoS mitigation services
🔑 Common Port Numbers (CISSP Must-Know):
FTP: 20 (data), 21 (control) · SSH: 22 · Telnet: 23 · SMTP: 25 · DNS: 53 · HTTP: 80 · POP3: 110 · IMAP: 143 · HTTPS: 443 · LDAP: 389 · LDAPS: 636 · RDP: 3389 · SNMP: 161/162

💡 Exam Tip: Encapsulation adds headers as data moves DOWN the stack: Data → Segment → Packet → Frame → Bits. Decapsulation removes headers as data moves UP. Each layer only communicates with its peer layer on the other host.

9. Memory Tricks & Mnemonics

🧠 OSI Layers (Bottom → Top): "Please Do Not Throw Sausage Pizza Away"
Physical → Data Link → Network → Transport → Session → Presentation → Application
🧠 PDU Names (Bottom → Top): "Big Frogs Pounce Silently" — Bits → Frames → Packets → Segments
🧠 Layer Devices:
L1: Hub (dumb repeater)
L2: Switch (uses MAC addresses)
L3: Router (uses IP addresses)
L4: Firewall (stateful, uses ports)
🧠 TCP vs UDP: "TCP is like registered mail (guaranteed delivery, signed receipt). UDP is like dropping a postcard in the mailbox (no guarantee, no receipt, but fast)."
