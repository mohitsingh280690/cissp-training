3. DES (Data Encryption Standard)

📖 DES: A symmetric block cipher adopted by NIST in 1977 as a federal standard (FIPS 46-3). Based on IBM's Lucifer cipher, modified by the NSA. Uses a Feistel network structure.
PropertyValue
Block Size64 bits
Key Size64 bits total, but only 56 bits are effective (8 bits used for parity)
Rounds16 rounds of Feistel operations
StructureFeistel network (splits block in half; each round transforms one half using the other)
StatusBROKEN / DEPRECATED — 56-bit key is too short for brute force
🔑 Feistel Network: A structure where the data block is split into two halves (L and R). In each round, one half is transformed using a round function and XORed with the other half, then they swap. The beauty: the same structure works for both encryption and decryption (just reverse the key schedule). DES uses 16 rounds of this process.
⚠️ DES Is Broken: In 1998, the EFF's "Deep Crack" machine brute-forced a DES key in 56 hours. In 1999, a distributed effort broke it in 22 hours. The 56-bit key space (256) is far too small for modern computing. DES should NEVER be selected on the exam as a secure option.

4. Triple DES (3DES / TDEA)

📖 3DES: Applies DES three times with two or three different keys to increase security. Officially called Triple Data Encryption Algorithm (TDEA). Was the interim replacement for DES before AES was standardized.
PropertyValue
Block Size64 bits (same as DES)
ProcessEncrypt – Decrypt – Encrypt (EDE) with different keys
StatusDEPRECATED — NIST disallowed after 2023; slow, small block size

3DES Keying Options

OptionKeysEffective StrengthNotes
Keying Option 13 independent keys (K1 ≠ K2 ≠ K3)168 bits (effective ~112 bits due to meet-in-the-middle attack)Strongest; 3 unique keys
Keying Option 2K1 = K3, K2 different112 bits (effective ~80 bits)Most commonly used
Keying Option 3K1 = K2 = K356 bits (equivalent to single DES)Only for backward compatibility; provides no extra security
💡 Exam Tip: 3DES process is always E-D-E (Encrypt-Decrypt-Encrypt). With Keying Option 3 (all keys same), the middle decrypt undoes the first encrypt, leaving just one DES encryption — backward compatible with DES. The effective security of Keying Option 1 is ~112 bits (not the theoretical 168) due to meet-in-the-middle attacks.

5. AES (Advanced Encryption Standard)

📖 AES: Symmetric block cipher selected by NIST in 2001 (FIPS 197) through an open competition. The winning algorithm was Rijndael, designed by Belgian cryptographers Joan Daemen and Vincent Rijmen. AES is the current worldwide standard for symmetric encryption.
PropertyValue
Block Size128 bits (always — regardless of key size)
Key Sizes128, 192, or 256 bits
Rounds10 (128-bit key), 12 (192-bit key), 14 (256-bit key)
StructureSubstitution-Permutation Network (SPN) — NOT Feistel
StatusCURRENT STANDARD — no practical attacks exist

AES Round Operations

StepOperationShannon Property
SubBytesSubstitution using S-box lookup tableConfusion
ShiftRowsCircular shift of rows in the state matrixDiffusion
MixColumnsMatrix multiplication on each columnDiffusion
AddRoundKeyXOR with round key derived from key scheduleKey mixing
🔑 AES vs DES — Key Differences:
• AES uses SPN (Substitution-Permutation Network); DES uses Feistel network
• AES block size = 128 bits; DES block size = 64 bits
• AES is much faster than 3DES and more secure
• AES-256 is considered quantum-resistant for symmetric crypto (Grover's algorithm halves effective key length, so 256 → 128, still secure)
💡 Exam Tip: AES block size is ALWAYS 128 bits. The key size varies (128/192/256), but the block size does not. Rijndael (the original algorithm) supported other block sizes, but NIST standardized only 128-bit blocks as AES. This is a common exam question.

7. Block Cipher Modes of Operation

📖 Modes of Operation: Methods that define how a block cipher encrypts data larger than one block. They determine how blocks are chained, whether an IV is used, and what properties (confidentiality, integrity) are provided.
ModeFull NameIV?Parallelizable?Error PropagationKey Properties
ECBElectronic CodebookNoYes (encrypt + decrypt)Single block onlyINSECURE — identical plaintext blocks → identical ciphertext blocks; pattern leakage
CBCCipher Block ChainingYesDecrypt onlyCurrent + next blockEach plaintext block XORed with previous ciphertext block before encryption; widely used
CFBCipher FeedbackYesDecrypt onlyCurrent + next blockCreates a stream cipher from a block cipher; processes sub-block units
OFBOutput FeedbackYesKeystream can be precomputedSingle bit onlyCreates a stream cipher; single bit errors don't propagate; never reuse IV
CTRCounterNonce (counter)Yes (encrypt + decrypt)Single bit onlyEncrypts counter values; fully parallelizable; random access to blocks
GCMGalois/Counter ModeNonceYes (encrypt + decrypt)Authentication tagAEAD — provides both encryption AND authentication; used in TLS 1.3
  ECB Mode (INSECURE):                CBC Mode (Secure):

  P₁ → [E(K)] → C₁                  IV ⊕ P₁ → [E(K)] → C₁
  P₂ → [E(K)] → C₂                  C₁ ⊕ P₂ → [E(K)] → C₂
  P₃ → [E(K)] → C₃                  C₂ ⊕ P₃ → [E(K)] → C₃

  If P₁ = P₃, then C₁ = C₃          Even if P₁ = P₃, C₁ ≠ C₃
  → Pattern leakage!                 → No pattern leakage
⚠️ NEVER Use ECB Mode: ECB encrypts each block independently with the same key. Identical plaintext blocks produce identical ciphertext blocks, leaking patterns. The famous "ECB Penguin" demonstrates this — encrypting a bitmap image in ECB mode reveals the image outline in the ciphertext. Always use CBC, CTR, or GCM instead.
🔑 GCM (Galois/Counter Mode) — The Gold Standard:
• Provides Authenticated Encryption with Associated Data (AEAD)
• Combines CTR mode encryption with GHASH authentication
• Both encrypts AND authenticates data in a single pass
• Fully parallelizable → excellent performance
• Required by TLS 1.3 (AES-128-GCM and AES-256-GCM)
• Produces an authentication tag that detects tampering

9. Memory Tricks & Mnemonics

🧠 "DES = 56-64-16": 56-bit effective key, 64-bit block, 16 rounds. "AES = 128 block always, keys 128/192/256, rounds 10/12/14." The block size of AES NEVER changes.
🧠 "Never ECB": ECB = "Everyone Can Browse" your patterns. Identical blocks in → identical blocks out. The ECB Penguin proves it. Always choose CBC, CTR, or GCM.
🧠 "GCM = Gets Confidentiality + Message auth": GCM provides both encryption (confidentiality) and authentication (integrity + authenticity) in one operation. It's the gold standard and used in TLS 1.3.
🧠 "3DES = EDE = Encrypt-Decrypt-Encrypt": Three operations, not three encryptions. The middle step is decrypt. With Keying Option 3 (same key), the decrypt undoes the first encrypt, making it backward-compatible with single DES.
