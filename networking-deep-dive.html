1. The OSI Model — 7 Layers Explained

📖 OSI Model: The Open Systems Interconnection (OSI) model is a conceptual framework published by ISO in 1984 that standardises how network systems communicate. It divides communication into 7 distinct layers, each with a specific job. Think of it like a postal system — each department handles one part of delivering your letter without needing to know what the other departments do.
🧠 Mnemonic Top→Bottom (Layer 7→1): “All People Seem To Need Data Processing”
Application → Presentation → Session → Transport → Network → Data Link → Physical

Mnemonic Bottom→Top (Layer 1→7): “Please Do Not Throw Sausage Pizza Away”
Physical → Data Link → Network → Transport → Session → Presentation → Application
+-------------------------------------------------------------------+ | Layer 7 | APPLICATION | HTTP, FTP, SMTP, DNS, SSH, SNMP, RDP | | Layer 6 | PRESENTATION | SSL/TLS, JPEG, MPEG, ASCII, Unicode | | Layer 5 | SESSION | NetBIOS, RPC, PPTP, NFS, SQL sessions | | Layer 4 | TRANSPORT | TCP, UDP | | Layer 3 | NETWORK | IP, ICMP, OSPF, BGP, RIP, IGMP | | Layer 2 | DATA LINK | Ethernet, ARP, 802.11, VLAN (802.1Q) | | Layer 1 | PHYSICAL | Cables, Hubs, Repeaters, Radio, Bits | +-------------------------------------------------------------------+ Data flows DOWN (L7 to L1) on SENDER -- each layer adds a HEADER Data flows UP (L1 to L7) on RECEIVER -- each layer strips its HEADER

Layer 1 — Physical L1

The Physical layer deals with raw bits — 0s and 1s transmitted as electrical signals (copper cable), light pulses (fibre optic), or radio waves (wireless). It defines cables, connectors, voltage levels, pin layouts, and signal timing. This layer has no concept of addresses or intelligence — it simply moves bits from one point to another.

Security threatsPhysical wiretapping, signal interception/eavesdropping, jamming, cable cutting
⚠️ Exam Trap: A hub operates at Layer 1 and broadcasts all incoming frames to every single connected port regardless of the intended destination. All ports share one collision domain. A hub has zero intelligence. A switch (Layer 2) learns MAC addresses and sends frames only to the correct port — completely different behaviour.

Layer 2 — Data Link L2

The Data Link layer provides node-to-node delivery within the same network segment. It uses MAC addresses (48-bit hardware addresses burned into NICs at manufacture) for local addressing. It detects transmission errors using CRC (Cyclic Redundancy Check) but does not correct them — corrupted frames are simply discarded.

Data Link has two sub-layers:

  • LLC (Logical Link Control): Interfaces with Layer 3; handles flow control and error notification upward
  • MAC (Media Access Control): Hardware addressing; controls how devices share access to the physical medium

FunctionFrame data, MAC addressing, error detection (CRC), medium access control
ProtocolsEthernet (802.3), Wi-Fi (802.11), ARP, VLAN tagging (802.1Q), Spanning Tree (802.1D), PPP
DevicesSwitch, Bridge, NIC, Wireless Access Point

💡 Critical ARP Fact: ARP (Address Resolution Protocol) resolves IP addresses to MAC addresses. When device A wants to reach IP 192.168.1.10, it broadcasts “Who has 192.168.1.10? Tell 192.168.1.5.” The owner of that IP replies with its MAC address. ARP has zero authentication — any device can send a fake ARP reply claiming any IP. This is why ARP poisoning attacks work: an attacker poisons ARP caches so traffic gets redirected through the attacker’s machine.

Layer 3 — Network L3

The Network layer handles end-to-end delivery across multiple different networks. It uses IP addresses (logical, software-assigned addresses — not burned into hardware) and makes routing decisions — choosing the best path for a packet to travel from source host to destination host, potentially crossing many networks along the way.

Security threatsIP spoofing, route hijacking, ICMP attacks (Ping of Death, Smurf), fragmentation attacks, TTL manipulation

Layer 4 — Transport L4

The Transport layer provides end-to-end communication between specific applications on different hosts. It uses port numbers to identify which application the data belongs to (port 80 = web, port 25 = email). The two dominant protocols are TCP (reliable, ordered) and UDP (fast, unreliable).

AttributeDetail

🔐 TCP Three-Way Handshake (memorise this):
Step 1 — SYN: Client sends SYN with a random Initial Sequence Number (ISN-c)
Step 2 — SYN-ACK: Server acknowledges (ACK = ISN-c + 1) and sends its own SYN with ISN-s
Step 3 — ACK: Client acknowledges server’s SYN (ACK = ISN-s + 1) — connection is established

A SYN flood attack sends thousands of SYN packets with spoofed source IPs. The server sends SYN-ACK responses that are never answered, filling its connection table with half-open connections until it cannot accept legitimate connections. Defense: SYN cookies.

Layer 5 — Session L5

The Session layer establishes, manages, and terminates sessions (logical conversations) between applications. It provides synchronisation checkpoints for long data transfers — if a 2GB file transfer fails at 80%, a checkpoint allows resumption from that point rather than restarting from zero.

AttributeDetail

ProtocolsNetBIOS, RPC, PPTP, L2TP (partly), NFS (partly), SQL session handling
PDUData
Security threatsSession hijacking, replay attacks, session fixation

Layer 6 — Presentation L6

The Presentation layer is the translator, encoder, and formatter. It ensures data from the sending application can be understood by the receiving application regardless of different internal formats. It handles character encoding conversion, data compression, and encryption/decryption.

AttributeDetail
FunctionData format translation (ASCII ↔ EBCDIC), character encoding, compression, encryption/decryption
Protocols / FormatsSSL/TLS (encryption), JPEG, GIF, PNG, MPEG, ASCII, EBCDIC, Unicode, Base64

Security threatsEncoding bypass attacks, malformed data attacks, format string vulnerabilities
⚠️ Exam Trap — SSL/TLS Layer: CISSP questions using the OSI model typically place SSL/TLS at Layer 6 (Presentation). In the TCP/IP model and in real-world practice, SSL/TLS operates between Transport and Application. Both are technically defensible. The exam usually specifies which model to use — follow that context.

Layer 7 — Application L7

The Application layer is where end-user applications interface directly with the network. This is the only layer users directly interact with. Web browsers (HTTP/HTTPS), email clients (SMTP/POP3/IMAP), and file transfer clients (FTP) all operate here.

💡 CISSP Exam Strategy for OSI Questions:
  • “Which layer does X operate at?” → use the table above
  • “A packet-filtering firewall works at which layers?” → Layers 3 & 4 (stateless, inspects IP headers and port numbers)
  • “A stateful inspection firewall works at which layers?” → Layers 3, 4, and 5 (tracks connection state)
  • “An application proxy/gateway works at which layer?” → Layer 7 (inspects full application payload)
  • “Which device separates broadcast domains?” → Router (Layer 3)
  • “Which device separates collision domains?” → Switch (Layer 2, one collision domain per port)

2. TCP/IP Model & Data Encapsulation

📖 TCP/IP Model (DoD Model): The TCP/IP model, also called the DoD (Department of Defense) model, is the practical framework that actually powers the Internet. Unlike the OSI model which is a conceptual reference, TCP/IP is what is actually implemented. It has 4 layers that map to the OSI model’s 7 layers.

⚠️ Exam Trap: CISSP uses both OSI and TCP/IP models and switches between them. OSI is used for conceptual discussions and protocol placement. TCP/IP is used for practical discussions. Know both models and be comfortable translating between them.

Data Encapsulation — How Data Actually Travels the Network

Encapsulation is the process of wrapping data with protocol-specific headers as it passes down the layers. Each layer adds its own header (and sometimes a trailer). On the receiving end, each layer strips its header — this is called de-encapsulation.

SENDER (Application layer down to Physical): Application Data: "GET /index.html HTTP/1.1\r\nHost: example.com" | v [ Layer 4: TCP adds header: src port, dst port, seq#, ack#, flags ] TCP Segment: [ TCP HDR | HTTP DATA ] | v [ Layer 3: IP adds header: src IP, dst IP, TTL, protocol=6 ] IP Packet: [ IP HDR | TCP HDR | HTTP DATA ] | v [ Layer 2: Ethernet adds header (src MAC, dst MAC) + CRC trailer ] Ethernet Frame: [ ETH HDR | IP HDR | TCP HDR | HTTP DATA | CRC ] | v [ Layer 1: converted to electrical/optical/radio signals ] Bits: 01001011... RECEIVER (Physical up to Application): reverse process Bits -> Strip ETH header -> Strip IP header -> Strip TCP header -> HTTP Data
🔐 Critical Encapsulation Facts for CISSP:
  • Each layer’s header is only read/processed by the same layer on the remote device
  • An Ethernet frame (L2) knows nothing about IP addresses — only MAC addresses
  • An IP packet (L3) knows nothing about port numbers — that is TCP/UDP’s job
  • A router only reads up to the IP header. It does NOT open the TCP segment or application data
  • A stateful firewall reads up to the TCP header (L3+L4+L5 state)
  • An application proxy reads the full payload all the way to Layer 7

3. IP Addressing — IPv4, IPv6, Subnetting & CIDR

📖 IP Address: An IP address is a logical (software-assigned) address used to identify a device on a network. Unlike MAC addresses (hardware-burned), IP addresses can be changed and are assigned administratively. They operate at Layer 3.

IPv4 — The Most Common Version

IPv4 uses 32-bit addresses written in dotted-decimal notation (e.g., 192.168.1.100). Each group is called an octet (8 bits, value 0–255). There are approximately 4.3 billion possible IPv4 addresses — a number the world has exhausted, which is why NAT and IPv6 exist.

💡 Subnetting Logic: Usable hosts = 2n − 2, where n = number of host bits. You subtract 2 because the network address (all host bits = 0) and broadcast address (all host bits = 1) cannot be assigned to hosts. For CISSP, you rarely need to calculate subnets — just understand the concepts and recognise common masks.

NAT — Network Address Translation

NAT translates private (internal) IP addresses to public (external) IP addresses, allowing many internal devices to share a single public IP. PAT (Port Address Translation), also called NAT Overload, additionally maps unique port numbers to each internal session, enabling hundreds of devices to share one public IP simultaneously.

⚠️ Critical CISSP Trap: NAT is NOT a security control. It is an addressing mechanism that provides some incidental obscurity of internal addresses, but it does not provide authentication, authorisation, or access control. Do not select NAT as a security solution on the exam. Also: NAT breaks end-to-end IP connectivity and can interfere with IPSec.

ICMP — The Network’s Diagnostic Tool

ICMP (Internet Control Message Protocol) operates at Layer 3 and is used for network diagnostics and error reporting. It is not a transport protocol — it does not carry application data. The ping command uses ICMP Echo Request/Reply. The traceroute command uses ICMP TTL-exceeded messages.

6. Network Architecture & Segmentation

🔐 Security Principle: Proper network segmentation limits the blast radius of a breach. If an attacker compromises one segment, segmentation prevents them from freely accessing other segments. This is a fundamental security architecture principle tested heavily on CISSP.

DMZ — Demilitarized Zone (Screened Subnet)

📖 DMZ / Screened Subnet: A DMZ (Demilitarized Zone), also called a Screened Subnet, is a network segment that sits between the untrusted external network (internet) and the trusted internal network. It hosts services that must be accessible from the internet (web servers, email servers, DNS servers, FTP servers) while protecting the internal network.
INTERNET (untrusted) | [External Firewall] <-- filters inbound internet traffic | [ D M Z ] <-- Web Server, Mail Server, DNS Server, Proxy | [Internal Firewall] <-- much stricter rules; only allows necessary traffic | INTERNAL NETWORK (trusted) <-- databases, workstations, internal servers
AspectDetail
PurposeAllow public access to certain services without exposing the internal trusted network
Traffic rules (example)Internet → DMZ: HTTP/HTTPS only | DMZ → Internet: blocked or very restricted | DMZ → Internal: only specific database queries | Internal → DMZ: management traffic only
Dual firewall (preferred)Two separate firewalls from different vendors. Even if one vendor has a vulnerability, the other may not. Stronger defence-in-depth.
Single firewall (3-leg)One firewall with 3 interfaces: external, DMZ, internal. Less secure (single point of failure) but more common due to cost.

What does NOT go in the DMZDomain controllers, databases with sensitive data, internal file servers, authentication servers
⚠️ Exam Trap: A DMZ requires two firewalls in the ideal design (one facing internet, one facing internal). A single firewall with three interfaces (external, DMZ, internal) is sometimes called a “screened subnet” but is less secure because a single firewall compromise exposes everything. The exam may ask which is the “most secure” design — answer is dual firewalls, ideally from different vendors.

VLANs — Virtual Local Area Networks

📖 VLAN: A VLAN (Virtual LAN) is a logical grouping of network devices regardless of their physical location. VLANs partition a single physical switch into multiple isolated Layer 2 broadcast domains. Defined by IEEE 802.1Q standard.
VLAN ConceptExplanation
VLAN IDA number (1–4094) identifying the VLAN. VLAN 1 is the default VLAN on Cisco switches — best practice is to not use VLAN 1 for production traffic and change the native VLAN.
Access PortA switch port assigned to exactly one VLAN. End devices (PCs, printers) connect to access ports. Frames on access ports have NO VLAN tag — the switch adds/removes the tag.
Trunk PortA switch port that carries traffic for multiple VLANs simultaneously. Tagged frames use 802.1Q headers. Used between switches and between switches and routers.
802.1Q TaggingThe standard for VLAN tagging. Inserts a 4-byte tag into the Ethernet frame header containing the VLAN ID.
Native VLANThe VLAN whose traffic is sent untagged on a trunk port. Default is VLAN 1. Must be changed for security — used in VLAN hopping attacks.
Inter-VLAN routingVLANs are isolated by default — to route traffic between VLANs you need a Layer 3 device (router or L3 switch). This is intentional — it allows controlled, firewall-inspected traffic between VLANs.

Air Gap

An air gap is a security measure where a computer or network is physically isolated from unsecured networks (including the internet). There is literally an air gap — no wired or wireless connection. Used for the most sensitive systems (nuclear control, classified military networks, industrial control systems).

💡 Air Gap Limitations: Even air-gapped systems have been compromised. The Stuxnet worm (discovered 2010) infected Iranian nuclear centrifuge control systems that were air-gapped — delivered via USB drives. Air gaps require strict physical security AND removable media controls.

7. Firewalls — Types, Functions & Placement

📖 Firewall: A firewall is a network security device (hardware, software, or both) that monitors and controls incoming and outgoing network traffic based on predetermined security rules. The fundamental principle is implicit deny: everything not explicitly permitted is blocked.

Firewall Types — Ordered by Sophistication

1. Packet-Filtering Firewall (Stateless)

The simplest type. Inspects each packet in isolation looking only at header information. It has no memory of previous packets — it does not track connection state.

AttributeDetail
OSI LayerLayer 3 and Layer 4 (IP headers and port numbers)
What it inspectsSource IP, destination IP, source port, destination port, protocol (TCP/UDP/ICMP)
What it does NOT inspectConnection state, payload content, application-layer data
AdvantagesVery fast, low overhead, works at wire speed
DisadvantagesCannot detect fragmentation attacks, IP spoofing, SYN flood (can’t distinguish half-open connections), cannot block split DNS or other app-layer attacks
Typical useRouter ACLs (Access Control Lists), basic network perimeter filtering

2. Stateful Inspection Firewall

The most common type of firewall today. It tracks the state of network connections in a state table. A packet is only allowed through if it is part of a known, established, legitimate connection.

AttributeDetail
OSI LayerLayers 3, 4, and 5 (inspects state across session)
What it inspectsAll packet-filter attributes PLUS connection state (new, established, related, invalid)
State tableRecords each connection: source/dest IP, source/dest port, protocol, state (SYN_SENT, ESTABLISHED, etc.)
AdvantagesUnderstands whether a return packet is legitimate (part of established session) vs. unsolicited. Blocks SYN flood more effectively. Blocks IP spoofing of return traffic.
DisadvantagesDoes not inspect application payload. Cannot stop attacks that use legitimate ports (e.g., malware over port 443).
Typical useEnterprise network firewalls, perimeter defence

3. Circuit-Level Gateway

Operates at the Session layer (Layer 5). It monitors TCP handshakes to verify that sessions are legitimate before allowing traffic. Once a session is established, it forwards packets without further inspection.

AttributeDetail
OSI LayerLayer 5 (Session)

Key characteristicHides internal network topology (internal client IP is hidden behind the gateway’s IP)
ExampleSOCKS proxy
LimitationDoes not inspect application data — malware in allowed sessions passes through

4. Application-Level Gateway (Application Proxy Firewall)

The most thorough and most resource-intensive type. It acts as a full proxy — the client connects to the proxy, the proxy connects to the destination server. The proxy inspects the full application-layer payload in both directions.

AttributeDetail
OSI LayerLayer 7 (Application)

AdvantagesCan detect malware, policy violations, data exfiltration in application content. Can enforce application-specific policies (e.g., block HTTP PUT but allow GET). Completely hides internal network. Deep inspection of commands.
DisadvantagesSlowest type (must fully process every packet). Each protocol needs its own proxy (HTTP proxy, FTP proxy, etc.). Can be a bottleneck.
Example servicesWeb proxy (Squid), email security gateway, DNS firewall

Combines stateful inspection with deep packet inspection (DPI), integrated IPS, application awareness, user identity awareness, and threat intelligence. This is the modern enterprise standard.

💡 Firewall Best Practices (CISSP loves these):

8. IDS & IPS — Detection vs Prevention

📖 IDS vs IPS:
IDS (Intrusion Detection System): Passive monitoring system that detects suspicious activity and generates alerts. It does NOT block traffic. Think of it as a security camera — it records and alerts but does not physically stop the intruder.

IPS (Intrusion Prevention System): Active, inline system that detects AND can automatically block malicious traffic in real time. It sits directly in the traffic path and can drop packets, reset connections, or block IPs.

🔐 False Positives vs False Negatives — Critical Concept:

False Positive (FP): The IDS/IPS generates an alert for legitimate, benign traffic. A false alarm. Security staff investigate and waste time. If IPS, legitimate traffic is blocked causing service disruption. Annoying and costly, but not a security breach.

False Negative (FN): The IDS/IPS does not alert when a real attack is occurring. The attack passes through undetected. This is the most dangerous outcome — you believe you are safe when you are not.

CISSP Exam Rule: When asked which is more dangerous or more critical to minimise — False Negatives. An undetected attack is more dangerous than a false alarm.

Split Tunneling

⚠️ Split Tunneling — A Security Risk:
Split tunneling is a VPN configuration where only traffic destined for the corporate network goes through the encrypted VPN tunnel. Internet-bound traffic goes directly through the user’s local ISP connection, bypassing the corporate network entirely.

Security Risk: If a user’s machine is compromised while split tunneling, malware can reach the corporate VPN while also communicating with its C&C server over the direct internet connection — bypassing all corporate security controls (filtering, monitoring, DLP).

Secure alternative: Force all traffic through the VPN tunnel (“full tunnel” or “tunnel all traffic”). This allows corporate security controls to inspect all traffic.

CISSP Exam: Split tunneling is generally presented as a security risk. The exam will typically favour disabling it.

10. Wireless Network Security (WEP → WPA3)

📖 Wireless Security Fundamentals: Wireless networks are inherently more exposed than wired networks because radio signals can be received by any device within range — there is no physical access control. Wireless security protocols provide encryption, authentication, and integrity protection for over-the-air communications.

Wireless Security Protocol Evolution

WEP — Wired Equivalent Privacy (Broken — Do Not Use)

AttributeDetail
Year introduced1997 (original 802.11)

Why it is brokenThe 24-bit IV is too short — IVs repeat after collecting enough traffic (on a busy network, within minutes). Attackers capture enough packets to statistically determine the key. Tools like Aircrack-ng can crack WEP in minutes regardless of key length.
AuthenticationOpen System (no auth) or Shared Key (challenge-response — actually LESS secure because it reveals keystream)

AttributeDetail

AttributeDetail

13. ⚠️ Exam Traps — The Most Common CISSP Networking Mistakes

⚠️ How CISSP Traps Work: CISSP networking questions almost always have two plausible-looking answers. The trap is usually the answer that is technically correct but not the best choice, or a concept that sounds similar but is subtly different.
1
Hub vs Switch behaviour: A hub (L1) broadcasts all frames to ALL ports always. A switch (L2) unicasts frames to the correct port after learning MAC addresses. Hub = zero intelligence, shared collision domain, all traffic visible to all ports. Switch = intelligent, one collision domain per port.
2
Routers separate broadcast domains; Switches separate collision domains: A router blocks broadcasts (ARP, DHCP, ICMP). A switch creates separate collision domains per port but passes broadcasts. VLANs on a switch also separate broadcast domains at Layer 2, but require a router/L3 switch for inter-VLAN routing.
3
ARP operates at the Layer 2/3 boundary, NOT purely Layer 3: ARP resolves IP addresses (L3) to MAC addresses (L2). It has no authentication whatsoever. This is the root cause of ARP poisoning attacks. ARP requests are broadcast; responses can be forged by any device on the segment.
4
Packet filter = stateless; Stateful inspection = connection tracking: A packet-filtering firewall inspects each packet independently (L3/L4 headers only, no connection memory). A stateful inspection firewall maintains a connection state table and knows whether a packet is part of an established session. Stateless cannot block asymmetric attacks; stateful can.
5
Circuit-level gateway validates TCP handshake but does NOT inspect payload: It operates at Layer 5, confirms the TCP session is legitimate, then relays packets without reading content. An application proxy (Layer 7) inspects full payload. If a question asks what can stop application-layer attacks (SQL injection, etc.), circuit-level gateway is wrong; application proxy is right.
6
IDS detects and alerts (passive, out-of-band); IPS blocks (active, inline): IDS cannot prevent an attack in progress — it only alerts so humans can respond. IPS is inline and can drop packets automatically. False positives in an IPS cause outages; in an IDS they cause alert fatigue. Deploy IDS first to tune, then switch to IPS.
7
False Negatives are MORE dangerous than False Positives: False positive = alert on legitimate traffic (annoying). False negative = miss a real attack (dangerous). CISSP will ask which is more critical to minimise — always false negatives. An undetected breach is far worse than a nuisance alert.
8
AH provides NO encryption; ESP provides both encryption and authentication: AH (Protocol 51) = auth + integrity, zero confidentiality. ESP (Protocol 50) = encryption + auth + integrity. If you need a VPN that encrypts data, you need ESP. Also: AH is broken by NAT (NAT changes the IP header that AH authenticates). ESP works with NAT via NAT-T (UDP 4500).
9
Tunnel mode wraps entire original packet; Transport mode only wraps payload: Tunnel mode (site-to-site VPNs) = new outer IP header + encrypted original packet. Transport mode (host-to-host) = original IP header stays visible, only payload encrypted. For any VPN connecting networks or remote users, Tunnel mode with ESP is the standard answer.
10
WPA3 uses SAE (not PSK); WPA2 uses PSK: WPA2 Personal uses a pre-shared passphrase vulnerable to offline dictionary attacks (capture 4-way handshake, brute-force offline). WPA3 Personal uses SAE (Simultaneous Authentication of Equals) which provides forward secrecy and resists offline cracking. WPA3 = current best practice.
11
EAP-TLS requires client certificates; PEAP requires only server certificate: EAP-TLS = most secure, mutual TLS authentication with certificates on both client and server. Requires PKI infrastructure. PEAP = tunnel using server cert only, client credentials (username/password) sent inside the tunnel. If no PKI for clients, choose PEAP. If maximum security, choose EAP-TLS.
12
SSID hiding and MAC filtering are NOT real security controls: SSID: visible in probe requests, discoverable in seconds with Wireshark. MAC address: transmitted in cleartext in every 802.11 frame, trivially spoofed. Neither protects data. Use WPA3 encryption and 802.1X authentication instead. Selecting SSID hiding or MAC filtering as a security answer on CISSP is wrong.
13
DNS port 53 uses UDP for queries, TCP for zone transfers: Standard queries use UDP (fast, small). Zone transfers and large responses (DNSSEC) use TCP. A firewall rule blocking all TCP port 53 would break DNS zone transfers. Both need to be considered in firewall rules for DNS infrastructure.
14
DHCP uses UDP: server port 67, client port 68: DHCP is fully UDP-based. Client broadcasts DHCP Discover from port 68 to 255.255.255.255:67. Server responds to port 68. Firewall rules for DHCP must allow UDP 67 and 68. DHCP starvation (flooding DHCP with fake requests) and rogue DHCP servers are common attacks; defend with DHCP snooping on switches.
15
SNMPv1 and v2c send community strings in cleartext: Community strings are essentially passwords. They are transmitted in plaintext in v1 and v2c. Any network sniffer can capture them. This is not a subtle vulnerability — it is a fundamental design flaw. Always use SNMPv3 with authPriv mode (authentication + AES encryption). There is no acceptable reason to use v1 or v2c in production.
16
SFTP (port 22) is NOT FTP with SSL; FTPS is FTP with SSL: SFTP = SSH File Transfer Protocol. Runs over SSH. Port 22. Completely separate from FTP. FTPS = FTP + TLS. Implicit FTPS port 990, Explicit FTPS port 21 (upgrades to TLS). If a question mentions SFTP and asks for the port, answer is 22. If it asks about FTP over SSL, that is FTPS.
17
NAT is NOT a security control: NAT translates private-to-public IPs. It incidentally obscures internal addresses but provides no authentication, access control, confidentiality, or integrity. It is an addressing mechanism. NAT also breaks IPSec AH and can break end-to-end protocols. Do not select NAT as the answer when a security control is asked for.
18
SSL 3.0 and TLS 1.0/1.1 are all broken/deprecated: SSL 3.0 = POODLE attack. TLS 1.0 = BEAST, POODLE-TLS. TLS 1.1 = deprecated per RFC 8996. Minimum acceptable = TLS 1.2. Current best = TLS 1.3 (removes legacy cipher suites, mandatory forward secrecy, faster handshake). PCI-DSS requires TLS 1.2 minimum.
19
TACACS+ uses TCP and encrypts entire payload; RADIUS uses UDP and encrypts only the password: TACACS+ (port 49/TCP) separates auth, authorisation, and accounting; encrypts everything. RADIUS (ports 1812/1813/UDP) combines auth+authorisation; only encrypts the password field. TACACS+ preferred for device administration; RADIUS preferred for network access control (wireless, VPN).
20
Smurf = ICMP amplification; Fraggle = UDP amplification: Both send packets to a broadcast address with a spoofed victim source IP so all hosts on the network reply to the victim. Smurf uses ICMP Echo Request. Fraggle uses UDP (ports 7/19). Both are largely historical but tested. The key differentiator on the exam is the protocol used.
21
IPv6 has NO broadcast: IPv6 replaced broadcast with multicast (devices opt in to groups) and anycast (nearest node responds). There is no broadcast address in IPv6. Any question describing IPv6 broadcast traffic or broadcast addresses is describing something that does not exist. Also: IPv6 loopback is ::1 (not 127.0.0.1).
22
BGP is exterior (between Autonomous Systems); OSPF/RIP/EIGRP are interior (within AS): BGP = the internet routing protocol; routes between ISPs and large enterprises. OSPF, RIP, EIGRP = interior routing within one organisation. BGP hijacking (advertising false routes) is a real, nation-state-level attack. RPKI provides cryptographic route origin validation.
23
Ideal DMZ uses TWO firewalls; one firewall with 3 interfaces is a compromise: Dual-firewall DMZ = external firewall faces internet, internal firewall protects internal network, DMZ is sandwiched between them. If the external firewall vendor has a zero-day vulnerability, the internal firewall (ideally different vendor) still protects. Single 3-legged firewall = single point of failure. Exam: choose dual firewalls for highest security.
24
VLAN hopping defence: change the native VLAN and disable DTP: Double-tagging exploits native VLAN (default: VLAN 1). Fix: change native VLAN to an unused VLAN that no hosts are assigned to. Switch spoofing exploits DTP (Dynamic Trunking Protocol). Fix: disable DTP on all access ports, explicitly configure them as access ports. Never put production hosts on the native VLAN.
25
Kerberos clock skew tolerance is 5 minutes: Kerberos uses ticket timestamps as anti-replay protection. If client and KDC clocks differ by more than 5 minutes (default), authentication fails. The solution is NTP synchronisation. This is intentional design. If CISSP describes Kerberos intermittently failing, the answer involves time synchronisation (NTP), not certificates or routing.
26
Split tunneling = security risk (bypass of corporate controls): Split tunneling sends only corporate-bound traffic through the VPN; internet traffic goes direct. Risk: compromised endpoint can reach C&C servers over direct internet while connected to corporate VPN, bypassing all corporate security controls. CISSP favours disabling split tunneling and routing all traffic through the VPN (full tunnel/force-tunnel).
27
169.254.x.x APIPA means DHCP failed: APIPA (Automatic Private IP Addressing) is self-assigned when a Windows device gets no DHCP response. Range: 169.254.0.0/16. An APIPA host can talk to other APIPA hosts on the same local segment but cannot reach any other network, cannot get to the internet, and cannot contact domain services. Seeing 169.254.x.x = check DHCP server, check network connectivity.
28
Packet filter cannot stop application-layer attacks; WAF/App proxy required: A packet filter sees port 80 and passes HTTP traffic through regardless of content. SQL injection, XSS, and command injection all travel over port 80/443. Only an application proxy or WAF (Layer 7) that reads the HTTP payload can detect and block these. If asked what stops SQL injection at the network level, the answer is WAF, not firewall.

14. 🧪 20 Practice Questions with Explanations

🎯 Instructions: Read every option before selecting an answer. Understand WHY each wrong option is wrong. That reasoning is what the exam tests.
Q1. A packet-filtering firewall provides security at which OSI layers?
  • A. Layers 2 and 3 only
  • B. Layers 3 and 4
  • C. Layers 5 through 7
  • D. Layer 7 only
B. Packet filters inspect IP headers (Layer 3: src/dst IP, protocol) and TCP/UDP port numbers (Layer 4). They do not track connection state or inspect application content. Stateful adds L5 state tracking; application proxy adds L7.
Q2. Which IPSec protocol provides BOTH confidentiality AND authentication?
  • A. AH (Authentication Header)
  • B. ESP (Encapsulating Security Payload)
  • C. IKE
  • D. AH combined with HMAC
B. ESP (Protocol 50) encrypts the payload (confidentiality) and authenticates it (integrity/origin). AH (Protocol 51) provides authentication and integrity ONLY — zero encryption. IKE is the key negotiation protocol. AH is also broken by NAT. For VPN encryption, always ESP.
Q3. A company needs to connect two geographically separated office networks via an encrypted tunnel over the internet. Which combination is MOST appropriate?
  • A. IPSec Transport mode with AH
  • B. IPSec Tunnel mode with ESP
  • C. IPSec Transport mode with ESP
  • D. SSL VPN in clientless mode
B. Site-to-site VPN between offices requires Tunnel mode (wraps entire original packet, hides internal IPs) and ESP (encryption + authentication). Transport mode is for host-to-host, not network-to-network. AH provides no encryption. Clientless SSL VPN is for individual user browser access, not network-to-network connectivity.
Q4. WPA2 Personal is specifically vulnerable to which attack when a weak passphrase is used?
  • A. KRACK attack against the passphrase directly
  • B. Offline dictionary attack against a captured 4-way handshake
  • C. ARP poisoning of the access point
  • D. SSL stripping on the wireless link
B. An attacker captures the WPA2 4-way handshake (forced by deauthentication or waited for). They then run offline dictionary/brute-force tools (like Hashcat) against the captured handshake, requiring no further contact with the AP. WPA3 SAE prevents offline attacks. KRACK is a nonce reuse vulnerability unrelated to passphrase cracking. ARP and SSL stripping are not wireless protocol attacks.
Q5. What is the PRIMARY security risk of enabling split tunneling on a corporate VPN?
  • A. It increases latency for all network traffic
  • B. It prevents access to internal resources
  • C. A compromised endpoint could bridge the corporate network with malicious internet connections, bypassing security controls
  • D. It is not supported by modern VPN protocols
C. With split tunneling, only corporate-bound traffic goes through the VPN. Internet traffic bypasses all corporate security controls. If the endpoint is malware-infected, the malware can communicate freely with its C&C over direct internet while the VPN simultaneously provides access to internal corporate resources — the endpoint becomes a pivot point.
Q6. Which device operates at Layer 2 and makes forwarding decisions using MAC addresses?
  • A. Router
  • B. Hub
  • C. Switch
  • D. Gateway
C. A switch operates at Layer 2, learns MAC addresses in its CAM table, and forwards frames to specific ports. Router = Layer 3, uses IP addresses. Hub = Layer 1, broadcasts to ALL ports with no intelligence. Gateway = Layer 7, translates between protocols.
Q7. An organisation wants 802.1X enterprise wireless but cannot deploy client certificates to employee devices. Which EAP method is BEST?
  • A. EAP-TLS (requires client and server certificates)
  • B. PEAP (requires server certificate only)
  • C. LEAP (deprecated, vulnerable)
  • D. WEP with RADIUS
B. PEAP requires only a server-side certificate. Clients authenticate via username/password inside an encrypted TLS tunnel. EAP-TLS is more secure but requires client certificates (the given constraint). LEAP is deprecated and vulnerable to dictionary attacks. WEP is broken regardless of what authentication is added to it.
Q8. An attacker sends ICMP Echo Requests to a network broadcast address with the victim's IP as the spoofed source address. What is this attack?
  • A. Smurf attack
  • B. Fraggle attack
  • C. Ping of Death
  • D. Teardrop attack
A. Smurf = ICMP Echo Request to broadcast address with spoofed victim source IP, causing all hosts to reply to the victim (amplification DoS). Fraggle = same concept but uses UDP. Ping of Death = single oversized ICMP packet. Teardrop = fragmented packets with overlapping offsets. ICMP + broadcast amplification = Smurf.
Q9. Which SNMP configuration provides BOTH authentication and encrypted management traffic?
  • A. SNMPv1 with strong community strings
  • B. SNMPv2c with ACLs
  • C. SNMPv3 with authNoPriv
  • D. SNMPv3 with authPriv
D. SNMPv3 authPriv provides authentication (HMAC) AND privacy/encryption (AES). authNoPriv provides authentication only, no encryption. v1 and v2c send community strings in cleartext regardless of how complex they are — a sniffer captures them. ACLs restrict who can query but don't encrypt the data in transit. Only authPriv is acceptable for production.
Q10. A user visiting their bank website is redirected to a convincing phishing site because a DNS resolver's cache was corrupted. What attack occurred?
  • A. DNS hijacking
  • B. DNS cache poisoning
  • C. BGP hijacking
  • D. MITM via ARP poisoning
B. DNS cache poisoning injects false records into a resolver's cache, causing all users of that resolver to receive incorrect IP addresses. DNS hijacking changes the DNS server configured on a device/router (different attack vector). BGP hijacking affects internet routing (different layer). ARP poisoning is local Layer 2 redirection. The clue is "DNS resolver's cache was corrupted."
Q11. A security analyst needs to monitor ALL traffic on a network segment without affecting performance or traffic flow. What is the best approach?
  • A. Deploy an IPS inline on the segment
  • B. Connect a NIDS to a SPAN/mirror port on the switch
  • C. Replace all switches with hubs for visibility
  • D. Enable SNMPv1 polling on all devices
B. A NIDS on a SPAN (mirror) port receives a passive copy of all traffic without any impact on the production path (out-of-band). IPS inline adds latency and is a single point of failure. Replacing switches with hubs degrades performance catastrophically (shared collision domains). SNMP polling collects device statistics, not actual packet content.
Q12. Which BEST describes a false negative in an IDS?
  • A. IDS alerts on legitimate web browsing traffic
  • B. IDS generates duplicate alerts for one event
  • C. IDS fails to generate an alert when a real attack occurs
  • D. IDS misclassifies the attack type
C. A false negative = IDS misses a real attack; it does not fire an alert. This is the most dangerous outcome. False positive = alerts on legitimate traffic. B = duplicate alerting. D = misclassification. For CISSP, false negatives are always presented as the more dangerous of the two error types.
Q13. Which architecture BEST protects an internal network while allowing internet users to access a public web server?
  • A. Place the web server on the internal network and open port 443 inbound through the firewall
  • B. Place the web server in a DMZ between an external firewall and an internal firewall
  • C. Use NAT to hide the web server's real IP address from the internet
  • D. Deploy WPA3 encryption on the web server's network segment
B. A DMZ (screened subnet) places public-facing servers between two firewalls. If the web server is compromised, an attacker faces the internal firewall before reaching the trusted network. A places the server on the internal network (breach = full internal access). NAT is an addressing mechanism, not a security architecture. WPA3 is wireless encryption, irrelevant here.
Q14. Which protocol directly replaces Telnet for secure interactive remote administration?
  • A. HTTPS
  • B. SNMPv3
  • C. SSH
  • D. SFTP
C. SSH (port 22) is the direct, exact replacement for Telnet. It provides encrypted interactive terminal sessions. Telnet sends everything including passwords in cleartext. HTTPS is for web interfaces. SNMPv3 is for network monitoring/management queries. SFTP provides encrypted file transfer via SSH, not interactive terminal sessions.
Q15. An attacker poisons the ARP cache of hosts on a LAN to associate the attacker's MAC with the default gateway's IP. What is the PRIMARY resulting threat?
  • A. MAC flooding causing broadcast storm
  • B. Man-in-the-middle interception of all traffic leaving the LAN
  • C. VLAN hopping to adjacent network segments
  • D. Denial of service via bandwidth exhaustion
B. When hosts believe the gateway's IP maps to the attacker's MAC, all outbound traffic flows through the attacker's machine (MITM). The attacker can eavesdrop on or modify all unencrypted traffic. MAC flooding exhausts the CAM table (different attack). VLAN hopping is a different Layer 2 attack. DoS via bandwidth exhaustion is not what ARP poisoning achieves.
Q16. Which is the MOST secure wireless configuration for a corporate environment?
  • A. WPA2 Personal with a 20-character passphrase and hidden SSID
  • B. WPA3 Enterprise with 802.1X and EAP-TLS
  • C. WPA2 Enterprise with MAC address filtering
  • D. WEP with 256-bit keys
B. WPA3 Enterprise + EAP-TLS = strongest available: AES-GCMP-256 encryption, per-user certificate-based mutual authentication, 802.1X access control. Hidden SSID is security theatre. MAC filtering is security theatre (MAC addresses are trivially spoofed). WEP is broken regardless of key length. WPA2 Enterprise is acceptable but inferior to WPA3 Enterprise.
Q17. DNSSEC protects against which specific threat?
  • A. Encrypting DNS queries to prevent eavesdropping
  • B. Hiding the contents of DNS responses from ISPs
  • C. Providing cryptographic proof that DNS records are authentic and untampered
  • D. Encrypting DNS zone transfers between authoritative servers
C. DNSSEC adds digital signatures to DNS records, allowing resolvers to verify their authenticity — this prevents DNS cache poisoning. DNSSEC does NOT encrypt queries (use DoH or DoT for privacy). It does not hide responses from intermediaries. Zone transfer security uses TSIG or TLS separately. The key word is "authenticate records," not "encrypt queries."
Q18. A server is becoming unresponsive because its TCP connection table is flooded with thousands of half-open connections from spoofed source IPs. What attack is this?
  • A. SYN flood
  • B. Smurf attack
  • C. Teardrop attack
  • D. Session hijacking
A. SYN flood = sends SYN packets with spoofed source IPs, server sends SYN-ACK to spoofed IPs which never respond, filling the half-open connection table until no new connections can be accepted. Defense: SYN cookies. Smurf uses ICMP broadcast amplification. Teardrop uses fragmented packets with bad offsets. Session hijacking steals authenticated sessions.
Q19. A penetration tester demonstrates VLAN hopping via double-tagging. Which change BEST mitigates this?
  • A. Enable VLAN 1 on all trunk ports as the default
  • B. Increase the number of VLANs deployed
  • C. Change the native VLAN on trunk ports to an unused, dedicated VLAN with no hosts assigned
  • D. Enable DTP negotiation on access ports
C. Double-tagging exploits the native VLAN (default: VLAN 1). Changing the native VLAN to an unused VLAN that no devices belong to neutralises this attack. A perpetuates the vulnerability (VLAN 1 as native is the problem). B is irrelevant to double-tagging. D (enabling DTP on access ports) makes switch-spoofing VLAN hopping easier — DTP should be disabled on all access ports.
Q20. Kerberos authentication in a specific office is failing intermittently with "clock skew too great" errors. What is the MOST likely cause and fix?
  • A. Expired user certificates — renew PKI certificates
  • B. RADIUS server unreachable — restore the RADIUS service
  • C. NTP failure — resynchronise clocks so the difference between clients and the KDC is within 5 minutes
  • D. BGP routing failure — restore routing tables
C. Kerberos uses ticket timestamps as anti-replay protection. If client clocks differ from the KDC by more than 5 minutes (default skew limit), tickets are rejected. The fix is repairing NTP synchronisation. "Clock skew too great" is a definitive Kerberos error message. Kerberos uses shared secret keys, not PKI certificates. RADIUS is for 802.1X, not Kerberos. BGP is routing, not authentication.
