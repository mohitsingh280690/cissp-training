3. Hash Functions Fundamentals

📖 Cryptographic Hash Function: A one-way mathematical function that takes an input of any size and produces a fixed-size output (the hash, digest, or fingerprint). It is computationally infeasible to reverse the hash to find the original input, or to find two different inputs that produce the same hash.

Five Essential Properties

PropertyDefinitionWhy It Matters
DeterministicSame input always produces the same hashEnables verification — hash the file again and compare
Fixed OutputOutput is always the same length regardless of input sizeSHA-256 always produces 256 bits whether input is 1 byte or 1 TB
Preimage ResistanceGiven a hash h, infeasible to find any input m such that hash(m) = hCannot reverse the hash to find the original data (one-way)
Second Preimage ResistanceGiven input m1, infeasible to find a different m2 where hash(m1) = hash(m2)Cannot forge a different document with the same hash
Collision ResistanceInfeasible to find ANY two different inputs m1, m2 where hash(m1) = hash(m2)Strongest property — no two arbitrary inputs should ever collide
⚠️ Collision vs. Preimage: Collision resistance means you cannot find any two inputs that hash the same. Preimage resistance means given a specific hash, you cannot find an input that produces it. Collision resistance is a stronger requirement (birthday attack makes collisions easier to find).
🔑 What Hashing Provides:
✅ Integrity — any change to the input changes the hash
❌ Not Confidentiality — hashing is not encryption; the original data is not hidden
❌ Not Authentication — anyone can compute a hash (unless combined with a key → HMAC)
❌ Not Non-repudiation — requires asymmetric keys (digital signature)

💡 Exam Tip: SHA-2 and SHA-3 are both current NIST standards. SHA-3 uses sponge construction (Keccak), not the Merkle-Damgård structure used by MD5, SHA-1, and SHA-2. This is the key architectural difference — if SHA-2's internal structure is ever compromised, SHA-3's completely different design ensures it remains safe.
🔑 SHA-2 Family: SHA-2 is a family of algorithms (SHA-224, SHA-256, SHA-384, SHA-512), not a single algorithm. SHA-256 and SHA-512 are the primary members; SHA-224 and SHA-384 are truncated versions. All use the same Merkle-Damgård construction with different internal constants and output sizes.

5. HMAC (Hash-Based Message Authentication Code)

📖 HMAC: A construction that combines a cryptographic hash function with a secret key to provide both integrity and authentication. Format: HMAC-SHA256(key, message). Defined in RFC 2104.
FeaturePlain HashHMAC
Integrity✅ Yes✅ Yes
Authentication❌ No✅ Yes (only key holders can generate/verify)
Non-repudiation❌ No❌ No (shared key — both parties can generate)
Key RequiredNoYes — symmetric secret key
⚠️ HMAC ≠ Non-repudiation: HMAC uses a shared symmetric key, so both parties can produce the same HMAC. This means it proves the message came from someone who has the key, but not which key holder sent it. For non-repudiation, you need digital signatures (asymmetric private key).

6. Attacks on Hash Functions

AttackDescriptionCountermeasure
Birthday AttackExploits the birthday paradox: finding any two inputs with the same hash is easier than finding a specific preimage. For an n-bit hash, a collision can be found in approximately 2^(n/2) operations instead of 2^n.Use hash algorithms with at least 256-bit output (SHA-256+)
Rainbow TablePrecomputed table of hashes for common passwords. Attacker compares stolen password hashes against the table for instant lookup.Salt every hash with a unique random value before hashing
Brute Force / DictionaryTry all possible inputs (brute force) or common passwords (dictionary) until a matching hash is found.Use slow hash functions (bcrypt, scrypt, Argon2) for passwords; increase hash output size for other uses
Collision AttackFind two different inputs that produce the same hash output. MD5 and SHA-1 are both vulnerable.Use SHA-256 or SHA-3; avoid MD5 and SHA-1 entirely
Length ExtensionExploit Merkle-Damgård construction to append data to a message and compute a valid hash without knowing the secret. Affects hash(secret ‖ message).Use HMAC (double-hashing with key) instead of hash(key ‖ message); SHA-3 is immune
💡 Exam Tip: The birthday attack reduces collision-finding from 2^n to 2^(n/2). For SHA-256 (256-bit), collision resistance is 2^128 — still computationally infeasible. For MD5 (128-bit), collision resistance is only 2^64 — feasible with modern hardware. This is why the output size matters.

Password Hashing Best Practices

PracticePurpose
SaltingAdd a unique random value to each password before hashing. Defeats rainbow tables and ensures identical passwords produce different hashes.
Key StretchingDeliberately slow the hash computation (bcrypt, scrypt, Argon2, PBKDF2) to make brute force impractical.
PepperingAdd a system-wide secret (stored separately from the database) to further protect against database compromise.

7. Digital Signatures

📖 Digital Signature: A cryptographic technique that uses the signer's private key to create a unique signature of a message or document. Anyone with the signer's public key can verify the signature. Provides integrity + authentication + non-repudiation.

How Digital Signatures Work

DIGITAL SIGNATURE PROCESS ╔══════════════════════════════════════════════════╗ ║ ║ ║ SIGNING (Sender — Alice): ║ ║ 1. Hash the message → produces digest ║ ║ 2. Encrypt the digest with Alice's PRIVATE key ║ ║ 3. Attach signature to message → send both ║ ║ ║ ║ VERIFICATION (Receiver — Bob): ║ ║ 1. Hash the received message → digest₁ ║ ║ 2. Decrypt the signature with Alice's PUBLIC ║ ║ key → digest₂ ║ ║ 3. Compare digest₁ and digest₂ ║ ║ Match = valid (integrity + authenticity) ║ ║ Mismatch = tampered or wrong signer ║ ║ ║ ╚══════════════════════════════════════════════════╝
🔑 Three Guarantees of Digital Signatures:
✅ Integrity — the hash proves the message hasn't been altered
✅ Authentication — the private key proves the identity of the signer
✅ Non-repudiation — the signer cannot deny signing (only they have the private key)
⚠️ Digital Signatures Do NOT Provide Confidentiality: The message itself is sent in cleartext. The signature only proves integrity and origin. For confidentiality, the message must also be encrypted (e.g., S/MIME encrypts then signs, or signs then encrypts).

8. Digital Signature Algorithms

AlgorithmBased OnCapabilitiesNotes
RSA SignaturesInteger factorizationEncryption + SignaturesMost widely used; ≥2048-bit keys; used in TLS certificates
DSA (Digital Signature Algorithm)Discrete logarithmSignatures ONLYFIPS 186 standard; cannot encrypt; key sizes 1024–3072; being phased out
ECDSAElliptic curve discrete logarithmSignatures ONLYECC variant of DSA; shorter signatures; used in Bitcoin, TLS
EdDSA (Ed25519)Twisted Edwards curvesSignatures ONLYDeterministic (no random nonce needed); faster; used in SSH, Signal
💡 Exam Tip: DSA can ONLY sign — it cannot encrypt. RSA can do both encryption and signatures. This is a common exam question. Also know: DSA requires a random nonce (k) per signature. If the same nonce is reused, the private key can be recovered (this is how Sony's PS3 ECDSA key was extracted in 2010).

9. Memory Tricks & Mnemonics

🧠 "CIA-N of Digital Signatures": Digital signatures give you Integrity + Authentication + Non-repudiation. NOT Confidentiality. Remember: "Signatures don't hide — they prove."
🧠 Hash Algorithm Status: "MD5 and SHA-1 are DEAD. SHA-2 is STANDARD. SHA-3 is the SPARE." (SHA-3 exists as a backup with a different internal design.)
🧠 Birthday Attack Formula: For n-bit hash → collision in 2^(n/2). "Half the bits, half the security." MD5 (128-bit) → 2^64 collisions. SHA-256 → 2^128 collisions.
🧠 "Salt, Pepper, Stretch": Password hashing recipe — Salt (unique per password, stored with hash), Pepper (system-wide secret, stored separately), Stretch (slow functions: bcrypt, scrypt, Argon2).
