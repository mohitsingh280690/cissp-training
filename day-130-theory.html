5. Database Security Essentials

ACID Properties

PropertyMeaning
AtomicityAll or nothing
ConsistencyValid state to valid state
IsolationTransactions don't interfere
DurabilityCommitted = permanent

Database Attacks

AttackDescriptionDefense
AggregationCombining non-sensitive data to derive sensitive informationRestrict aggregate queries; classify combined data higher
InferenceDeducing classified info from cluesPolyinstantiation; noise; cell suppression
SQL injectionMalicious SQL through input fieldsParameterized queries (ALWAYS the best answer)
Polyinstantiation: Multiple records with same key at different classification levels. Prevents inference by showing "cover" data instead of "access denied" to lower-cleared users.

ConceptKey Exam Point

mTLSMutual TLS; both sides authenticate; standard for service-to-service

8. Commonly Confused Concepts

Concept AConcept BKey Difference
XSSCSRFXSS exploits user's trust in site; CSRF exploits site's trust in user's browser
AggregationInferenceAggregation = combining data; Inference = deducing from clues
SASTDASTSAST = code analysis (white box, early); DAST = running app testing (black box, late)
OAuthOIDCOAuth = authorization only; OIDC = authentication + authorization (adds ID token)
Session fixationSession hijackingFixation = set ID BEFORE login; Hijacking = steal ID AFTER login
Fail secureFail openSecure = deny on error (default); Open = allow on error (safety only)
Pen testVulnerability assessmentPen test = exploit vulnerabilities; VA = identify vulnerabilities
WhitelistBlacklistWhitelist = define what's allowed (best); Blacklist = define what's blocked (weak)
Data poisoningEvasion attackPoisoning = attack training phase; Evasion = attack inference phase
ContainersVMsContainers share kernel (less isolation); VMs separate kernels (more isolation)
IASTRASPIAST = QA testing tool (finds); RASP = production protection (blocks)
