2. Real-World Organizational Relevance

Ransomware has elevated backup strategy from a routine IT task to a board-level concern. Organizations with tested, air-gapped backups can recover from ransomware without paying. Those with only online backups often find them encrypted alongside production data. The 3-2-1 rule and immutable backup storage have become essential defenses, not just best practices.

3. Backup Types — Full, Incremental, Differential

The Archive Bit

The archive bit (also called archive flag or modification flag) is a file attribute that is SET (turned on) whenever a file is created or modified. Backup software uses this bit to determine which files need to be backed up. A full or incremental backup CLEARS the bit; a differential backup does NOT.

WEEK EXAMPLE — Full on Sunday, Daily Backups Mon-Sat INCREMENTAL Strategy: Sun: Full (all data) → clears bits Mon: Inc = Mon changes only → clears bits Tue: Inc = Tue changes only → clears bits Wed: Inc = Wed changes only → clears bits RESTORE Wed = Full + Mon Inc + Tue Inc + Wed Inc (4 tapes) DIFFERENTIAL Strategy: Sun: Full (all data) → clears bits Mon: Diff = Mon changes → does NOT clear bits Tue: Diff = Mon+Tue changes → does NOT clear bits Wed: Diff = Mon+Tue+Wed changes → does NOT clear bits RESTORE Wed = Full + Wed Diff only (2 tapes)
The Critical Exam Distinction:
• Incremental clears the archive bit → next incremental only captures NEW changes since last backup
• Differential does NOT clear the archive bit → keeps growing because it recaptures all changes since last full
• Restore trade-off: Incremental = fastest backup, slowest restore; Differential = moderate backup, moderate restore
Copy Backup: A full backup that does NOT clear the archive bit. Used for ad-hoc copies (e.g., before a migration) without disrupting the regular backup schedule. Useful when you need a snapshot but don't want to affect incremental/differential chains.

4. Backup Rotation Schemes

Grandfather-Father-Son (GFS)

GFS is a hierarchical backup rotation scheme:
• Son = Daily backups (incremental or differential) — retained for 1 week
• Father = Weekly full backups — retained for 1 month
• Grandfather = Monthly full backups — retained for 1 year (or longer per retention policy)

Tower of Hanoi

A more complex rotation scheme that uses multiple backup sets cycled at different frequencies (every day, every 2nd day, every 4th day, etc.). Provides longer retention with fewer media sets than GFS but is more complex to manage. Less commonly tested than GFS.

Exam Focus: GFS is the most commonly tested rotation scheme. Know the three levels (daily/weekly/monthly) and that Grandfather provides the longest retention. Tower of Hanoi is mentioned but rarely tested in depth.

5. The 3-2-1 Backup Rule

The 3-2-1 Rule:
3 copies of data (1 production + 2 backups)
2 different media types (e.g., disk + tape, or disk + cloud)
1 copy off-site (geographically separate location)

Modern extension — 3-2-1-1: Add 1 immutable/air-gapped copy (protection against ransomware that targets backup systems)

Why Each Component Matters

ElementProtects Against
3 copiesAny single point of failure — if one copy is corrupted or destroyed, two remain
2 media typesMedia-specific failures — a vulnerability affecting one medium (e.g., disk firmware bug) won't affect the other
1 off-siteSite-level disasters — fire, flood, earthquake at primary site
1 immutable (3-2-1-1)Ransomware — attackers increasingly target backup systems; immutable copies cannot be encrypted or deleted

6. Backup Media & Storage

MediumStrengthsWeaknesses
Tape (LTO)Lowest cost per GB; excellent for long-term archival; air-gapped when stored off-site; high capacity (LTO-9 = 18+ TB native)Slow sequential access; physical handling risks; requires compatible drive; degradation over time
Disk-based (SAN/NAS)Fast random access; quick restore; supports deduplication and compressionHigher cost per GB than tape; subject to same logical attacks as production (ransomware)
Cloud storageOff-site by default; scalable; no hardware management; often supports immutability (object lock)Egress costs; restore speed dependent on bandwidth; cloud provider dependency; potential data sovereignty issues
Optical (Blu-ray, WORM)WORM (Write Once Read Many) — truly immutable; excellent for compliance/legal holdsLimited capacity; slow; niche use cases
Electronic Vaulting vs Remote Journaling vs Database Shadowing:
• Electronic vaulting: Batch transfers of backup data to off-site location (scheduled, not real-time)
• Remote journaling: Transaction logs transferred to remote site in near real-time (journal/transaction based)
• Database shadowing: Complete database copy maintained at remote site in real-time (most current, most expensive)
Ordered by currency and cost: Vaulting < Journaling < Shadowing

7. Backup Verification & Testing

Verification ActivityDescription
Media integrity checkVerify backup media is readable and not corrupted (checksum/hash verification)
Test restoreActually restore data from backup to verify it works — the ONLY way to truly confirm backup integrity
Backup logs reviewReview backup job logs for errors, warnings, or incomplete runs
Random file restorePeriodically restore random files to verify backup content matches expectations
Full DR test restoreComplete system restoration as part of DR testing (Day 114)
⚠️ "Untested Backups Are NOT Backups": An organization that has never test-restored from its backups has unverified assumptions, not a backup strategy. Backups must be tested regularly. Exam scenarios frequently involve discovering during a disaster that backups are corrupt or incomplete because they were never tested.

8. RPO & RTO Alignment with Backups

RPO RequirementBackup Strategy Needed
RPO = 0 (zero data loss)Synchronous replication — NOT traditional backups (still need backups for corruption protection)
RPO = 1 hourContinuous Data Protection (CDP) or hourly snapshots; transaction log shipping
RPO = 4 hoursSnapshots or incremental backups every 4 hours
RPO = 24 hoursNightly full or differential backup
RPO = 1 weekWeekly full backup (non-critical data)
Continuous Data Protection (CDP): Captures EVERY change to data in real-time, allowing recovery to any point in time (not just backup intervals). True CDP achieves near-zero RPO. Near-CDP captures changes at very frequent intervals (every few minutes) but not every write operation.

9. Memory Tricks & Mnemonics

🧠 Incremental vs Differential — "I Clean, D Doesn't"
Incremental Clears the archive bit → only captures changes since LAST backup
Differential Doesn't clear → captures all changes since last FULL
🧠 Restore Speed — "Incremental = I Need everything; Differential = Just 2"
Incremental restore: Full + ALL incrementals in sequence (slowest restore)
Differential restore: Full + latest differential (just 2 — faster restore)
🧠 3-2-1 Rule
3 copies, 2 media types, 1 off-site
"Three copies, Two types, One far away"
🧠 GFS — "Grandfather = Grandest; Father = Monthly check; Son = daily chore"
Son = Daily (disposable, short retention)
Father = Weekly (kept for a month)
Grandfather = Monthly (long-term archive — the patriarch, kept longest)
🧠 Vaulting → Journaling → Shadowing = "VJS = Value Goes Up"
Vaulting = batch (least current, cheapest)
Journaling = near real-time transactions
Shadowing = real-time full copy (most current, most expensive)
