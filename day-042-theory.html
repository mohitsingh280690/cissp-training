3. OS Hardening

📖 OS Hardening: The process of reducing the attack surface by removing unnecessary components, services, and default configurations. Goal: leave only what is needed for the system's function.

Hardening Steps

StepActionExample
1. Remove Unnecessary SoftwareUninstall unused applications, services, protocolsRemove IIS on a database server; disable Telnet
2. Disable Unnecessary ServicesStop and disable services not required for functionDisable Print Spooler on servers that don't print
3. Close Unnecessary PortsBlock ports not used by the applicationOnly allow 443 on a web server; block all others
4. Apply Security PatchesInstall all current security updatesMonthly Windows updates; Linux kernel patches
5. Configure Access ControlsSet file permissions, user rights, group policiesRemove "Everyone" from shares; enforce least privilege
6. Enable Logging & AuditingConfigure security event loggingEnable Windows Security Event Log; syslog forwarding
7. Change Default CredentialsRename/disable default accounts; change default passwordsRename "Administrator"; disable "Guest"
8. Apply Configuration BaselinesUse security baselines from trusted sourcesCIS Benchmarks; DISA STIGs; Microsoft Security Baselines
💡 Exam Tip: CIS Benchmarks and DISA STIGs are the two most commonly referenced hardening baselines on the CISSP exam. CIS is industry-standard; DISA STIGs are U.S. DoD-specific.

4. Patch Management

📖 Patch Management: The systematic process of identifying, testing, and applying software updates (patches) to fix vulnerabilities and bugs. A critical preventive control.

Patch Management Lifecycle

PhaseActivity
1. MonitoringMonitor vendor notifications, CVE databases, threat intelligence for new patches
2. AssessmentEvaluate which patches are relevant and prioritize by risk (CVSS score)
3. TestingTest patches in a non-production environment before deployment
4. DeploymentRoll out patches to production systems per change management process
5. VerificationConfirm patches were applied successfully and systems are functioning
6. DocumentationRecord patch status, exceptions, and timeline for compliance
⚠️ Virtual Patching: When a patch cannot be immediately applied (legacy system, testing delay), a virtual patch can be deployed via WAF or IPS rules to block exploitation of the vulnerability until the real patch is installed. This is a compensating control, not a substitute for actual patching.

5. Security Modes of Operation

📖 Security Modes: Define the relationship between user clearances, system classification, and need-to-know requirements. Determine how a system handles users with different trust levels accessing data of different classifications.
ModeClearanceNTK ApprovalNTK for ALL DataDescription
DedicatedALL users cleared for HIGHEST levelALL users approvedYes — all users NTK all dataSystem processes ONLY one classification level; all users cleared and need-to-know for all data
System HighALL users cleared for HIGHEST levelALL users approved for SOMENo — not all users NTK all dataAll users cleared to highest level but NOT all have need-to-know for all data; MAC controls restrict
CompartmentedALL users cleared for HIGHEST levelSome users approved for SOMENoAll cleared to highest level; formal access approval required for each compartment
Multilevel (MLS)NOT all cleared for highestSome users approved for SOMENoUsers with DIFFERENT clearance levels on the same system; requires B3/A1 TCB with labels and MAC
🔑 Key Distinctions:
• Dedicated: Everyone is cleared, everyone needs to know everything → simplest, most restrictive use of system
• System High: Everyone is cleared but NOT everyone NTK everything → need-to-know varies
• Compartmented: Like System High but with formal compartment approvals → adds formal access approval
• Multilevel (MLS): Mixed clearance levels → MOST complex, requires strongest TCB (B3/A1), labels, MAC
💡 Exam Tip: The exam typically presents a scenario with clearance details and asks which mode applies. Key question: "Are ALL users cleared for the highest level?" If NO → Multilevel. If YES → check need-to-know details to distinguish Dedicated, System High, or Compartmented.

6. Process Isolation & Sandboxing

Process Isolation

Covered in Day 41 — each process has its own virtual address space enforced by MMU hardware. The OS kernel manages page tables and denies cross-process access.

Sandboxing

📖 Sandboxing: A technique that runs an application in a restricted, isolated environment where it has limited access to system resources. If the application is compromised, damage is contained within the sandbox.
Sandbox TypeExampleUse Case
Browser SandboxChrome renders each tab in a separate processPrevents compromised web content from affecting OS
Application SandboxiOS/Android app sandboxesApps can only access their own data; must request permission for others
OS-Level SandboxWindows Sandbox, Firejail (Linux)Run untrusted applications in disposable environments
Malware AnalysisCuckoo Sandbox, Any.RunSafely execute and analyze suspicious files

7. MAC vs. DAC in Operating Systems

FeatureDAC (Discretionary)MAC (Mandatory)
Who ControlsResource owner sets permissionsSystem policy enforces labels
BasisIdentity / group membershipSecurity labels (classification + compartment)
FlexibilityHigh — users can modify permissionsLow — only admin can change labels/policy
Security LevelLower — users can be tricked/coerced into granting accessHigher — policy cannot be overridden by users
ImplementationWindows NTFS ACLs, Unix file permissionsSELinux, AppArmor, Trusted Solaris
TCSEC LevelDivision C (C1, C2)Division B (B1, B2, B3)
🔑 Common OS Security Extensions:
• SELinux (Security-Enhanced Linux): MAC implementation developed by NSA; uses security contexts (labels) on every file and process
• AppArmor: Path-based MAC for Linux; simpler than SELinux; profile-based restrictions
• Windows MIC (Mandatory Integrity Control): Uses integrity levels (Low, Medium, High, System) — "no write up" for processes

8. Memory Tricks & Mnemonics

🧠 "Dedicated = Dedicated to ONE thing": Everyone cleared, everyone needs to know everything. The system is dedicated to one classification and one purpose.
🧠 "Security Modes Escalation: D-S-C-M":
• Dedicated → simplest (all clear, all NTK)
• System High → all clear, different NTK
• Compartmented → all clear, formal compartment access
• Multilevel → mixed clearances, most complex (B3/A1 TCB required)
Remember: "Don't Stress — Compartments Multiply" (D-S-C-M, increasing complexity)
🧠 "MAC = Military, DAC = Desktop": MAC is used in military/government systems with labels. DAC is used on everyday desktops where users control their own file permissions.
