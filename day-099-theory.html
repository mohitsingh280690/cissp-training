The Key Distinction: Vulnerability assessments IDENTIFY weaknesses; penetration tests EXPLOIT them to prove real-world impact. Audits verify COMPLIANCE; risk assessments evaluate BUSINESS IMPACT.

7. Top Exam Traps & Distractors

#TrapCorrect Answer
1"SOC 1 covers security controls"Wrong — SOC 1 covers FINANCIAL reporting controls only; SOC 2 covers security
2"Type I proves controls work over time"Wrong — Type I is a point-in-time snapshot of DESIGN only; Type II proves operating effectiveness
3"DAST requires source code"Wrong — DAST tests the running application from outside; SAST requires source code
4"Inquiry alone is sufficient audit evidence"Wrong — Inquiry is the weakest evidence type; must be corroborated with other evidence
5"Vulnerability assessment = Penetration test"Wrong — VA identifies vulnerabilities; pen testing EXPLOITS them to prove impact
6"100% statement coverage means the code is bug-free"Wrong — Statement coverage doesn't test decision logic; branch/path coverage is stronger
7"Internal audit reports to the CIO/CISO"Wrong — Internal audit reports to the audit committee/board for independence
8"Parallel test = Full interruption test"Wrong — Parallel keeps production running; full interruption shuts it down
9"KRIs are backward-looking"Wrong — KRIs are forward-looking (leading indicators); KPIs are backward-looking (lagging)
10"SOC 3 reports contain detailed control descriptions"Wrong — SOC 3 is summary only; SOC 2 contains the details
11"Fuzzing finds logic flaws"Wrong — Fuzzing finds crashes, memory corruption, and input-handling bugs; NOT business logic errors
12"Regression testing finds new bugs"Wrong — Regression testing verifies existing functionality still works after changes
