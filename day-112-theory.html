2. Real-World Organizational Relevance

Recovery strategy selection directly determines whether an organization meets its RTO/RPO. A financial trading firm with a 15-minute RTO cannot rely on a cold site — it needs a hot site with real-time replication. Conversely, a small nonprofit with a 72-hour RTO would waste money on a hot site when a cold site suffices. The strategy must match the BIA requirements while staying within budget constraints.

Exam Trap — "Warm Site": A warm site has SOME hardware but NOT current data and NOT identical configuration. It's the compromise between hot and cold. Think: "Warm = some equipment, BUT data and config need work."
Cost vs Speed Trade-off: As activation time decreases (faster recovery), cost increases exponentially. Hot site = fastest + most expensive. Cold site = slowest + cheapest. The BIA determines which trade-off is acceptable for each business function.

Site TypeDescriptionKey Characteristics
Mobile SiteSelf-contained, transportable computing facility (trailer/container)Can be deployed anywhere; useful when geographic flexibility needed; limited capacity; setup time varies
Mirrored SiteFully redundant, real-time mirror of primary siteMost expensive; ZERO downtime; synchronous replication; essentially running simultaneously. Often considered "beyond hot"

⚠️ Reciprocal Agreements — Exam Favorite: Reciprocal agreements are the LEAST reliable/dependable alternate site option. Problems: (1) partner may not have capacity when needed; (2) both may be affected by same disaster; (3) difficult to test; (4) security/confidentiality concerns sharing facilities; (5) hard to enforce contractually. The exam will test whether you know this is the weakest option.

6. RAID — Redundant Array of Independent Disks

RAID combines multiple physical disks into a logical unit to improve performance, redundancy, or both. RAID is NOT a backup — it protects against single disk failure but not against data corruption, deletion, ransomware, or site-level disaster.
RAID LevelNameHow It WorksMin DisksFault TolerancePerformanceUsable Capacity
RAID 0StripingData split across disks in stripes2NONE — any disk fails = total lossExcellent read/write100%
RAID 1MirroringExact copy on second disk21 disk failureGood read; write same as single50%
RAID 5Striping + Distributed ParityData striped; parity distributed across all disks31 disk failureGood read; slower write (parity calc)(n-1)/n
RAID 6Striping + Double ParityLike RAID 5 but with TWO parity blocks42 disk failuresGood read; slower write than RAID 5(n-2)/n
RAID 10Mirroring + Striping (1+0)Mirrors first, then stripes across mirrors41 disk per mirror setExcellent read/write50%
RAID Exam Essentials:
• RAID 0 = NO fault tolerance (performance only) — "0 redundancy"
• RAID 1 = Mirroring — simplest redundancy; 50% capacity cost
• RAID 5 = Most popular in enterprise; tolerates 1 disk failure; needs 3+ disks
• RAID 10 = Best performance + redundancy; most expensive (50% capacity)
• RAID is NOT a backup — protects against hardware failure only, not data corruption/deletion/ransomware
⚠️ "RAID is NOT a Backup": RAID protects against disk hardware failure. It does NOT protect against: accidental deletion (mirrors instantly), ransomware (encrypts both copies), data corruption (corrupted data replicated), fire/flood (all disks destroyed), logical errors. Always combine RAID with proper backups.

9. Memory Tricks & Mnemonics

🧠 Hot, Warm, Cold — "Temperature = Readiness"
Hot = Ready NOW (minutes) — fully equipped, data replicated
Warm = Getting there (hours-days) — some equipment, need data restore
Cold = Starting from scratch (days-weeks) — empty shell, need everything
🧠 RAID Levels — "0-1-5-6-10"
0 = Zero protection (striping only)
1 = One mirror (mirroring)
5 = Five-finger spread (parity distributed)
6 = Six = double the parity of 5
10 = 1+0 = Mirror first, THEN stripe (best of both worlds)
🧠 Reciprocal = "Risky"
Both start with R. Reciprocal agreements are the Riskiest/least Reliable alternate site option.
🧠 Sync vs Async — "Sync = Same time; Async = A little later"
Synchronous: RPO = 0 (zero loss), but distance-limited (~200 km) and slower writes
Asynchronous: RPO > 0 (some loss), but works anywhere and faster writes
