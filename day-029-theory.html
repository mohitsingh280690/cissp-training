⚠️ Exam Trap
TLS (Transport Layer Security) protects data in transit, not at rest. AES-256 full-disk encryption (FDE) protects data at rest, not in transit. Intel SGX protects data in use. Matching the wrong control to the wrong state is the most common mistake on this topic.

2. Real-World Relevance

A hospital stores patient records in an encrypted database (at rest). When a doctor queries a record, the database engine transmits it over the network (in transit). The doctor's workstation loads the record into RAM so she can view and edit it (in use). Each transition exposes the data to a different attack surface, requiring a different control.

Understanding these three states is the foundation for building a defense-in-depth data protection strategy — you cannot protect data end-to-end without covering all three.

💡 Core Insight
Data does not stay in one state. As it moves through its lifecycle, it continuously transitions between states. A security architect must ensure controls are applied at every state transition — not just one.

3. The Three Data States

The CISSP CBK defines three fundamental states in which data can exist:

Data at RestStored on media — disk, SSD, USB, tape, cloud storagePhysical theft of media; unauthorized access to storageEncryption (AES-256 FDE, TDE)

✅ Exam Tip
The CISSP exam will describe a scenario and ask which control applies. Always identify the state first — then select the control that matches that state exclusively.

4. Data at Rest

🗄️ Data at Rest — Stored on Physical or Logical Media

Data at rest refers to data that is not actively being moved or processed — it exists on a storage medium awaiting retrieval. Examples include files on a hard drive, database records, backup tapes, cloud object storage (AWS S3), and archived email.

Threat Landscape

  • Physical theft: Laptop stolen → attacker accesses unencrypted files directly from the drive
  • Unauthorized logical access: Misconfigured file permissions or ACLs allow access without authentication
  • Insider threat: Privileged user copies sensitive data to an unencrypted USB drive
  • Cloud misconfiguration: S3 bucket set to "Public" exposes millions of records

Controls for Data at Rest

Full Disk Encryption (FDE)

Encrypts the entire storage device at the hardware or OS level. If a device is stolen and powered off, the attacker cannot read any data without the decryption key (typically tied to a PIN, password, or TPM chip).

  • BitLocker (Windows) — integrates with TPM 2.0; supports recovery keys in Active Directory
  • FileVault (macOS) — AES-XTS 128-bit encryption; key stored in iCloud or locally
  • VeraCrypt (cross-platform open source) — successor to TrueCrypt; supports hidden volumes
  • Self-Encrypting Drives (SEDs) — hardware-level AES encryption built into the drive firmware; no performance penalty since encryption is done by the drive controller, not the CPU

File and Folder Encryption

More granular than FDE — protects individual files or folders rather than the entire disk. Examples: EFS (Windows Encrypting File System); GPG-encrypted archives; password-protected ZIP/PDF files.

Transparent Database Encryption (TDE)

Encrypts database data files and backups at rest while allowing normal SQL queries to operate transparently. Supported by SQL Server, Oracle, MySQL, PostgreSQL. The database engine decrypts on-the-fly as authorized queries run — users and applications see no change.

Access Controls

Encryption is the primary control, but access controls (file system ACLs, NTFS permissions, database roles) layer additional protection by restricting who can even access the encrypted system.

✅ Exam Tip
FDE only protects data when the device is powered off. If a running, logged-in laptop is stolen, FDE does not help — the disk is already decrypted and mounted. Combine FDE with screen lock, strong authentication, and remote wipe capabilities for full protection.
📖 Definition: Self-Encrypting Drive (SED)
An SED performs AES encryption/decryption in hardware on the drive controller. The Data Encryption Key (DEK) is stored in the drive and never leaves it. The Authentication Credential (AC) (PIN/password) unlocks the DEK. To sanitize an SED, simply change/destroy the DEK — all data becomes instantly irrecoverable without needing to overwrite the entire drive.

5. Data in Transit

🌐 Data in Transit — Moving Across a Network

Data in transit (also called data in motion) is data that is actively being transmitted from one location to another — across a LAN, WAN, the internet, or a cloud provider's internal network.

Threat Landscape

  • Man-in-the-Middle (MitM): Attacker intercepts communications between two parties, potentially reading or modifying traffic
  • Packet sniffing / eavesdropping: Attacker uses Wireshark or similar tools to capture unencrypted traffic on a shared network segment
  • Replay attacks: Captured credentials or tokens are re-transmitted by the attacker
  • DNS hijacking / BGP hijacking: Traffic is silently rerouted through attacker-controlled infrastructure

Controls for Data in Transit

TLS (Transport Layer Security)

The dominant protocol for encrypting application-layer traffic. TLS wraps application protocols to create encrypted equivalents:

  • HTTPS = HTTP over TLS (TCP port 443)
  • SMTPS = SMTP over TLS (port 465 or STARTTLS on 587)
  • IMAPS / POP3S = secure email retrieval
  • LDAPS = secure directory queries

TLS 1.3 (current standard): Removed weak cipher suites (RC4, DES, 3DES, MD5-based MACs); mandatory Perfect Forward Secrecy (PFS); 1-RTT handshake (faster than TLS 1.2). TLS 1.0 and 1.1 are deprecated and should not be used.

IPSec

IPSec (IP Security) operates at Layer 3 (Network layer) and provides authentication, integrity, and encryption for IP traffic. Two critical modes for the exam:

ModeWhat Gets EncryptedTypical Use
Tunnel ModeEntire original IP packet (header + payload) is encrypted and encapsulated inside a new IP packetSite-to-site VPN between gateways
Transport ModeOnly the payload (upper-layer data) is encrypted; original IP header is preservedHost-to-host communication; end-to-end within a LAN

IPSec uses two sub-protocols:

  • AH (Authentication Header) — provides integrity and authentication ONLY; no encryption
  • ESP (Encapsulating Security Payload) — provides integrity, authentication, AND confidentiality (encryption)

VPN (Virtual Private Network)

Creates an encrypted tunnel over a public network:

  • IPSec VPN: Used for site-to-site connections; encrypts all traffic at Layer 3
  • SSL/TLS VPN: Used for remote access; works through firewalls more easily (port 443); client-less options via browser
  • WireGuard: Modern, lightweight VPN protocol; faster than IPSec/OpenVPN; uses ChaCha20 encryption

SSH (Secure Shell)

Replaces Telnet and FTP for secure remote administration. Provides encrypted command-line access (SSH), file transfer (SFTP, SCP), and port forwarding. Default TCP port 22.

⚠️ Exam Trap
IPSec Tunnel Mode vs. Transport Mode is heavily tested. Remember: Tunnel = two tunnels of privacy (new IP header hides original). Tunnel mode is used at the perimeter (gateway-to-gateway VPN). Transport mode is used host-to-host inside a trusted network.
✅ Exam Tip
TLS protects data in transit only. Once the server receives and decrypts the data and writes it to disk — the data is now "at rest" and TLS provides no protection. This is why defense-in-depth requires BOTH TLS AND disk encryption.

6. Data in Use

💻 Data in Use — Actively Processed in Memory

Data in use is data that is currently being processed by a CPU — loaded into RAM, CPU registers, cache memory, or GPU memory. To be processed, most encryption must first be decrypted, creating a temporary window of exposure.

Threat Landscape

  • Memory scraping: Malware reads process memory to extract plaintext data (POS terminal RAM scrapers steal credit card numbers before they're encrypted for transmission)
  • Cold boot attack: Attacker physically removes RAM chips from a running/recently-powered-off system and reads them in another machine before the charge dissipates (DRAM remanence)
  • DMA (Direct Memory Access) attack: Attacker uses Thunderbolt/FireWire/PCIe to bypass the CPU and read RAM directly (e.g., DMA-over-Thunderbolt attacks)
  • Malicious hypervisor: A compromised or rogue hypervisor reads the memory of all guest VMs running on the physical host
  • Speculative execution side channels: Meltdown, Spectre — CPU design flaws that allow processes to read memory outside their boundaries

Controls for Data in Use

Trusted Execution Environment (TEE)

A hardware-isolated, secure processing area within the main processor where sensitive code and data can execute with confidentiality and integrity guarantees — even if the host OS, hypervisor, or firmware is compromised.

Intel SGX (Software Guard Extensions)

📖 Definition: Intel SGX
Intel SGX creates hardware-enforced secure enclaves — isolated memory regions inside the CPU package that are encrypted at the hardware level. The encryption key lives only in the CPU and is never accessible to the OS, hypervisor, BIOS, or any external software, even with ring-0 (kernel) access.
  • Memory outside the enclave is encrypted with a key that only the CPU holds
  • Even a malicious OS cannot read enclave memory
  • Used for: DRM key storage, private key storage, secure computation in cloud
  • Limitation: Side-channel attacks (e.g., SGX-Step) can still leak information

ARM TrustZone

ARM's equivalent of SGX — divides the CPU into a Secure World and a Normal World. The Secure World runs a trusted OS (e.g., OP-TEE) that handles sensitive operations (fingerprint processing, mobile payments). The Normal World runs the standard OS (Android, iOS) without access to Secure World data.

Homomorphic Encryption

📖 Definition: Homomorphic Encryption
A cryptographic technique that allows computation to be performed directly on ciphertext, producing an encrypted result that, when decrypted, matches the result of performing the same computation on the plaintext — without ever decrypting the data during processing.
  • Partial Homomorphic Encryption (PHE): Supports only one operation (e.g., multiplication or addition) — used in some e-voting systems
  • Fully Homomorphic Encryption (FHE): Supports arbitrary computation — still computationally expensive; research stage for most applications
  • Key use case: Cloud data analysis — send encrypted data to a cloud provider, get encrypted results back; the cloud never sees the plaintext

Memory Scrubbing

The practice of explicitly overwriting sensitive data in memory (RAM) with zeros or random patterns immediately after it is no longer needed — reducing the window during which it is exposed to memory-based attacks. Languages like C/C++ allow direct memory overwrite; higher-level languages (Python, Java) rely on garbage collectors that may not guarantee immediate scrubbing.

✅ Exam Tip
The CISSP exam may present Intel SGX, TEE, TrustZone, or homomorphic encryption as answer options. All four are controls for data in use. If a scenario mentions a compromised hypervisor or memory scraping, the correct control is one of these — not TLS (transit) or FDE (rest).
⚠️ Data in Use: The Hardest State to Protect
Data in use represents the most technically challenging state. For most workloads, the data must be decrypted before processing. Traditional disk encryption (FDE) and TLS offer zero protection against memory scraping attacks — because the data is decrypted in RAM before any processing occurs.

Scenario Walkthroughs

Scenario 1: An analyst's laptop is stolen from a car. The laptop was powered off. Which control prevents data exposure?

→ Data is at rest (powered off = not in transit or in use). Control: FDE (BitLocker/FileVault). ✅

Scenario 2: A user is entering a credit card number on a checkout website. Which protocol protects the number as it travels from browser to server?

→ Data is in transit (moving from browser to server). Control: TLS 1.3 (HTTPS). ✅

Scenario 3: A company stores encrypted data in a public cloud. They want to allow the cloud provider to run analytics on the data without ever decrypting it. Which technology should they use?

→ Data is in use (being processed). Control: Homomorphic encryption. ✅

Scenario 4: An organization uses a multi-tenant cloud and is concerned about a rogue hypervisor reading VM memory. Which technology creates hardware-isolated memory the hypervisor cannot access?

→ Data is in use (in RAM, at risk from hypervisor). Control: Intel SGX (secure enclaves). ✅

8. Memory Tricks & Mnemonics

🧠 Mnemonic: "RAT" — The Three States
Rest → At Rest = disk encryption (AES/FDE)
Air → in trAnsit = TLS/IPSec/VPN
Thinking → in Use = TEE/SGX/Homomorphic

The data is either Resting on disk, flying through the Air (network), or being Thought about (processed in CPU).
🧠 IPSec Modes: "Tunnel Buries the Body"
Tunnel Mode = buries the entire original packet inside a new envelope — the original IP header is hidden. Used between gateways.

Transport Mode = only the payload is protected, the original IP header is exposed. Used host-to-host.
🧠 "SGX = Security Guard on the eXit"
Intel SGX creates a guard at the CPU level — even the OS and hypervisor are locked out. The "eXit" from normal OS memory space leads to an encrypted enclave that only the CPU can access.
🧠 Homomorphic = "Homework on encrypted paper"
You hand a tutor your homework in a sealed, opaque envelope. The tutor solves the problems through the envelope and hands back an equally sealed result envelope. When you open it, the answers are correct — but the tutor never read your work. That's homomorphic encryption.
