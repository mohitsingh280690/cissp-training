CISSP Exam Strategy for Domain 5: Most questions will be scenario-based, asking what the best control is. The answer almost never involves a single technology — it involves defense in depth (MFA + least privilege + monitoring). When in doubt, choose the option that applies the principle of least privilege and strongest authentication.

2. Topic-by-Topic Quick Reference (Days 70–84)

Day 70 — IAM Fundamentals: Identification (claim an identity) → Authentication (prove it) → Authorization (grant access) → Accountability (log it). IAAA model. Subject vs. object. Identity lifecycle. Provisioning/deprovisioning.
Day 71 — Authentication Methods: Type 1 = Know (passwords, PINs); Type 2 = Have (tokens, smart cards, OTP); Type 3 = Are (biometrics). MFA requires 2+ different types. Password policies: complexity, aging, history, lockout. Passphrases preferred over complex short passwords.
Day 72 — Biometrics & MFA: False Reject Rate (FRR / Type I error) vs. False Accept Rate (FAR / Type II error). CER/EER = crossover point (lower = better system). FAR is the security-critical error. Multi-factor requires different factor types — two passwords = two-step, not two-factor.
Day 73 — SSO & Kerberos: SSO reduces credential fatigue; risk = "one password to lose all access." Kerberos: KDC = AS + TGS; TGT issued by AS; service tickets issued by TGS. Attacks: pass-the-ticket, golden ticket (KRBTGT hash), Kerberoasting (SPN service tickets). SESAME extends Kerberos across orgs.
Day 74 — Federated Identity: SAML 2.0 (XML, SP-to-IdP, assertions: Authentication/Attribute/Authorization). OAuth 2.0 (authorization framework; grant types: Authorization Code with PKCE, Client Credentials). OIDC = OAuth 2.0 + identity layer (ID token = JWT). WS-Federation (Microsoft). Trust relationships: one-way/two-way/transitive.
Day 75 — DAC & MAC: DAC: resource owner controls ACLs; discretionary; ACL per object. MAC: labels (classification) + clearances; lattice-based; SELinux; no read-up (Bell-LaPadula), no write-down. Rule-based AC is a variant of MAC using rules not labels.
Day 76 — RBAC & ABAC: RBAC: permissions assigned to roles, roles assigned to users; static SoD = mutually exclusive roles; dynamic SoD = can't activate simultaneously. ABAC: attribute-based policies; most flexible; XACML standard. Temporal/context-aware access are ABAC variants.
Day 77 — Access Control Implementation: ACL: list of subjects per object; Capability Table: list of objects per subject; Access Control Matrix: both. Content-dependent AC: access based on data content (e.g., only own records). Context-dependent AC: based on circumstances (time, sequence, state). Restricted interfaces: menu-driven, encrypted databases, constrained interfaces limit actions.
Day 78 — Identity Management Lifecycle: Provisioning (joiner) → Modification (mover) → Deprovisioning (leaver). Orphan accounts (forgotten, owner left). Dormant accounts (inactive). Service accounts. Account recertification/re-attestation (quarterly). Identity governance and administration (IGA). Separation of duties prevents single-user control of critical functions.
Day 79 — Privileged Access Management (PAM): Just-in-time access. Credential vaulting. Privilege bracketing. Bastion hosts / jump servers. Session recording. Emergency break-glass procedures (dual custody). LAPS for local admin passwords. NIST SP 800-53 AC-6 (least privilege), AC-17 (remote access), AU-9 (audit protection). CIS v8 Control 5.
Day 80 — Directory Services: X.500 (1988, ancestor of LDAP). LDAP: RFC 4511; port 389 (cleartext) / port 636 (LDAPS). DN structure. Active Directory: forest = security boundary; NTDS.dit; SID filtering since Windows Server 2003. RADIUS (RFC 2865/2866; UDP 1812/1813; encrypts password only). TACACS+ (TCP 49; full packet encryption; Cisco). Diameter (RFC 6733; 4G LTE/5G mobile).
Day 81 — Access Control Attacks: Brute force / Dictionary / Rainbow tables (defeated by salting) / Credential stuffing (defeated by MFA + unique passwords) / Password spraying (evades per-account lockout). Pass-the-hash (NTLM; Credential Guard). Pass-the-ticket (Kerberos). Kerberoasting (SPN offline crack; gMSA prevents). Golden Ticket (KRBTGT hash; reset twice). Silver Ticket (service hash; stealthier). Privilege escalation (vertical/horizontal). IDOR, forced browsing, path traversal.
Day 82 — Session & Credential Management: Cookie flags: Secure (HTTPS only), HttpOnly (no JS access), SameSite (no CSRF). Session fixation (attacker sets token pre-auth) vs. hijacking (steals post-auth). Password storage: bcrypt (CPU-hard, built-in salt), scrypt/Argon2 (memory-hard), PBKDF2 (FIPS/NIST). JWT: header.payload.signature; alg:none vulnerability. FIDO2 = WebAuthn (browser↔server) + CTAP2 (browser↔hardware); phishing-resistant; origin-bound.
Day 83 — IDaaS: Cloud-based IAM platform. Components: cloud directory, SSO, MFA, lifecycle governance, conditional access, directory sync. SCIM (RFC 7643/7644): provisioning protocol (REST/JSON) — NOT authentication. Hybrid identity: Password Hash Sync / Pass-Through Auth / Federation (ADFS). PIM (Privileged Identity Management): JIT role elevation in cloud. Benefits: scalability, pre-built integrations. Risks: vendor lock-in, data sovereignty, single point of failure.
Day 84 — Zero Trust Architecture: Principles: Never Trust Always Verify; Assume Breach; Least Privilege. NIST SP 800-207. Components: PE (decides) + PA (instructs) = PDP; PEP (enforces). Micro-segmentation (east-west traffic control). SDP / ZTNA replaces VPN. BeyondCorp (Google): access = identity + device posture, NOT network location. EO 14028 + OMB M-22-09 mandate ZTA for federal agencies.

7. Common Exam Traps

⚠️ Trap 1 — Two-Factor vs. Two-Step: "Two passwords" is two-step, NOT MFA. MFA requires two different factor TYPES (know + have, know + are, have + are).
⚠️ Trap 2 — FAR vs. FRR: FAR (False Accept Rate = Type II) is the security-critical metric — it measures how often impostors are let in. A system with low FAR is secure. FRR (False Reject Rate = Type I) is the usability impact — how often legitimate users are rejected. CER = where FAR and FRR are equal.
⚠️ Trap 3 — Encryption vs. Salting: Salting defeats rainbow tables. Encryption does NOT defeat rainbow tables. Hashing with a salt makes pre-computed tables impractical.
⚠️ Trap 4 — SAML vs. SCIM vs. OIDC: SAML = authentication assertion (SSO). OIDC = authentication layer on OAuth 2.0. SCIM = provisioning (account creation/sync). OAuth 2.0 = authorization delegation (not authentication).
⚠️ Trap 5 — DAC vs. MAC Confusion: DAC = owner controls access (flexible, weaker security). MAC = system controls based on labels (rigid, higher security). When you see "labels, clearances, classification levels, military" → MAC. When you see "owner sets permissions, ACL" → DAC.
⚠️ Trap 6 — Golden vs. Silver Ticket: Golden Ticket uses KRBTGT hash = domain-wide access. Silver Ticket uses a service account hash = access only to that service. Silver is stealthier (no KDC contact).
⚠️ Trap 7 — RADIUS vs. TACACS+: RADIUS uses UDP and encrypts only the password. TACACS+ uses TCP and encrypts the ENTIRE payload. TACACS+ separates Authentication, Authorization, Accounting into separate functions. RADIUS combines A+A.
⚠️ Trap 8 — Zero Trust "Zero": Zero Trust does NOT mean zero security. It means zero IMPLICIT trust — explicit, verified trust at every access request. The word "zero" refers to not granting any trust by default.

8. Master Mnemonic Sheet

🧠 IAAA — Identification, Authentication, Authorization, Accountability
🧠 Authentication Factors — Know / Have / Are (+ Where / When for advanced ABAC)
🧠 Biometrics — FAR > FRR means poor security — FAR = security failure (bad guys in); FRR = usability failure (good guys out)
🧠 Kerberos Ticket Flow — Client → AS (get TGT) → TGS (get service ticket) → Application Server
🧠 SAML Actors — User (Principal) + IdP (Identity Provider) + SP (Service Provider)
🧠 Cookie flags — SHS: Secure, HttpOnly, SameSite
🧠 Password hashing — Bad=Fast (MD5/SHA); Good=Slow (bcrypt CPU-hard; Argon2/scrypt memory-hard)
🧠 Kerberos Attacks by ticket level — AS-REP Roasting (no pre-auth) → Kerberoasting (SPN crack) → Silver Ticket (service hash) → Golden Ticket (KRBTGT hash = "God mode")
🧠 Zero Trust — NAL — Never trust / Assume breach / Least privilege
🧠 NIST ZTA — PE-PA-PEP — Policy Engine decides; Policy Administrator instructs; Policy Enforcement Point blocks/allows
🧠 SCIM vs SAML — SCIM creates the room; SAML opens the door
🧠 RADIUS vs TACACS+ — RADIUS = UDP, password only encrypted; TACACS+ = TCP, ALL encrypted, Cisco

9. Final Exam Checklist

✅ Domain 5 Exam Checklist — Know All of These:
  • ✅ IAAA model and each component's meaning
  • ✅ Three authentication factor types + examples — and that MFA requires DIFFERENT types
  • ✅ FAR vs. FRR — which is Type I, which is Type II, which is the security risk
  • ✅ CER/EER — what it measures and that lower CER = better biometric
  • ✅ Kerberos flow: AS → TGT → TGS → service ticket → application server
  • ✅ SAML: XML-based, authentication assertions, IdP/SP trust
  • ✅ OAuth 2.0: authorization framework (access tokens, refresh tokens); OIDC adds authentication (ID token)
  • ✅ DAC: owner controls; MAC: system/labels control; RBAC: roles; ABAC: attributes + policies
  • ✅ Bell-LaPadula: confidentiality model (no read up, no write down)
  • ✅ ACL (per object), Capability Table (per subject), Access Control Matrix (both)
  • ✅ Provisioning lifecycle: joiner-mover-leaver; orphan accounts; access recertification
  • ✅ PAM: JIT access, credential vaulting, session recording, break-glass, LAPS
  • ✅ LDAP ports: 389 (cleartext), 636 (LDAPS)
  • ✅ RADIUS: UDP 1812/1813, encrypts password only; TACACS+: TCP 49, full encryption, separates AAA
  • ✅ Salting defeats rainbow tables; slow hashes (bcrypt/Argon2) protect against offline cracking
  • ✅ Pass-the-Hash: NTLM; Pass-the-Ticket: Kerberos; Golden Ticket: KRBTGT hash
  • ✅ Session fixation: pre-auth; session hijacking: post-auth
  • ✅ Cookie flags: Secure, HttpOnly, SameSite and what each prevents
  • ✅ FIDO2 = WebAuthn + CTAP2; phishing-resistant because origin-bound
  • ✅ SCIM: provisioning; SAML/OIDC: authentication — they are complementary
  • ✅ ZTA: 3 tenets; NIST SP 800-207; PE/PA/PEP roles; micro-segmentation; ZTNA replaces VPN
  • ✅ BeyondCorp: access = identity + device posture, not network location
