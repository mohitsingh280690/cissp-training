⚠️ Most Common Exam Trap: Candidates confuse the Data Owner and Data Custodian. Remember: Owner Decides; Custodian Does. Classification, access rights, and retention decisions belong to the owner. Implementation of controls belongs to the custodian.

2. Real-World Relevance

Without clear role assignments, data governance fails: IT implements controls without understanding business requirements; business units classify data without technical feasibility input; nobody takes accountability for data quality or compliance. Role clarity is the foundation of accountable data governance and is required by frameworks like ISO 27001 and regulations like GDPR.

3. Data Owner

Data Owner: A senior management individual (typically a business function head, VP, or C-suite executive) who has final accountability for a data asset. The owner is a business role, not a technical role.

Data Owner Responsibilities

  • Assign and review the data classification level
  • Determine who should have access to the data (access rights decisions)
  • Approve access requests and access exceptions
  • Define data retention periods and destruction requirements
  • Authorise declassification and reclassification
  • Accept residual risk associated with the data
  • Ensure compliance with regulatory requirements applies to their data
✅ Exam Tip: The data owner is NOT an IT role. The owner is the Vice President of Finance for financial records, the Chief Medical Officer for patient records, the Head of HR for employee files. They understand the business value and regulatory context of the data — IT does not make these decisions.

4. Data Custodian

Data Custodian: An IT professional or team responsible for the day-to-day technical management and protection of data assets according to the policies, classifications, and access decisions made by the data owner.

Data Custodian Responsibilities

  • Implement and maintain access controls as directed by the owner
  • Apply encryption at rest and in transit per classification requirements
  • Run data backups and test restoration procedures
  • Apply security patches and vulnerability management to systems holding the data
  • Maintain audit logs of data access
  • Implement data handling procedures (storage standards, secure transmission)
  • Execute data destruction when retention periods expire
Key Concept — Custodian as Agent of Owner: The custodian acts on behalf of the owner but does not make policy decisions. If the custodian disagrees with an owner's access decision (e.g., believes too many users have access), they should escalate to the owner — not unilaterally change access permissions.

5. Data Steward

Data Steward: A business-side role focused on data quality, integrity, and governance. The steward bridges business and technical teams to ensure data is accurate, consistent, complete, and compliant with data governance standards.

Data Steward responsibilities:

  • Define and enforce data quality standards (valid formats, value ranges, completeness requirements)
  • Document data definitions, taxonomies, data dictionaries
  • Monitor data quality metrics and remediate quality issues
  • Govern data usage compliance (are business users using data appropriately?)
  • Support data lineage tracking (where does the data come from? where does it go?)
✅ Exam Tip: The Data Steward is primarily a data governance and quality role. The CISSP exam distinguishes it from the Data Owner (who has accountability) and Custodian (who implements technical controls). When a scenario involves data quality, format standards, or metadata governance — the Steward is the answer. This is an emerging governance role in modern data-centric organisations.

6. System Owner

System Owner: A management individual responsible for a specific IT system — not the data stored within it, but the system itself (hardware, software, configuration, operation). The system owner ensures the system is operated securely and that security controls are properly configured and maintained.

System Owner responsibilities:

  • Ensure the system's security controls are documented (System Security Plan)
  • Coordinate with data owners to ensure appropriate data protection controls are in place
  • Accept residual risk for the system and its operation
  • Ensure the system meets regulatory and policy requirements
  • Participate in security assessments and authorisation processes (NIST RMF)

Key distinction: A hospital patient database (the data) has a Data Owner (Chief Medical Officer). The database server and application running it has a System Owner (CIO or IT Director). These can be different people — the system owner knows the technical platform; the data owner knows the data's business value.

7. Data Controller vs. Data Processor (GDPR)

GDPR (General Data Protection Regulation) introduced two key roles that apply to organisations handling personal data of EU/EEA residents:

RoleDefinition (GDPR Art. 4)ExamplePrimary Obligations
Data ControllerDetermines the purposes and means of processing personal data — decides WHY and HOW data is processedAn online retailer who collects customer names, emails, and purchase historyProvide privacy notices; establish lawful basis; respond to data subject rights requests; appoint DPO if required; report breaches within 72 hours
Data ProcessorProcesses personal data on behalf of a controller — is instructed by the controller and may not use the data for their own purposesA cloud-based CRM provider processing the retailer's customer data; an email marketing service; a payroll processing companyProcess only as instructed; implement appropriate security measures; assist controller with breach notification; delete/return data on termination; sub-processor contracts
⚠️ Critical Point — Joint Controllers: When two or more organisations jointly determine the purposes and means of processing, they are Joint Controllers and both bear controller obligations. This is a frequent GDPR compliance question for organisations that share personal data for joint marketing or research.
✅ Exam Tip: The CISSP exam increasingly tests GDPR concepts. The key test for controller vs. processor: does the organisation decide WHY and HOW the data is processed (controller) or does it process on another's instructions (processor)? A cloud provider storing your customer data is typically a processor. You are the controller.

8. User / Data Subject Responsibilities

End users who access and handle data also have security responsibilities — they are not passive participants:

  • Access only data they are authorised to access (least privilege, need-to-know)
  • Handle data according to its classification and the organisation's handling policies
  • Report data breaches, suspicious access, or misuse to the appropriate team
  • Comply with clean desk, clear screen, and physical security policies
  • Not share credentials or bypass access controls
  • Complete required security awareness and training programmes

9. Memory Tricks & Mnemonics

🧠 Owner vs. Custodian — THE Big Distinction:
O = Owner = DECIDES: classification, access rights, retention, declassification
C = Custodian = DOES IT: encryption, backups, patches, audit logs, destruction
Think: O = "Originates policy"; C = "Carries it out"
🧠 All Roles — "O-C-S-S-U":
Owner — sets classification and access policy (senior management)
Custodian — implements technical controls (IT ops)
Steward — ensures data quality and governance (business analyst / data governance)
System owner — owns the system/platform (IT manager)
User — follows policies; accesses only what is authorised
🧠 GDPR Controller vs. Processor:
Controller = Calls the shots (decides WHY and HOW data is processed)
Processor = Processes per instructions (does what the controller says)
Your cloud provider stores YOUR customer data → They are the Processor; YOU are the Controller
