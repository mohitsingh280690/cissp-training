3. Biba Integrity Model

📖 Biba Model (1977): A formal integrity model developed by Kenneth Biba. It is the mathematical inverse of Bell-LaPadula — where BLP protects confidentiality, Biba protects integrity using inverted rules.

Biba uses integrity levels (not confidentiality levels) assigned to subjects and objects. Higher integrity levels indicate more trustworthy/reliable data. The model prevents corruption of high-integrity data by lower-integrity sources.

High Integrity ← Most trusted │ Medium Integrity │ Low Integrity ← Least trusted Goal: Prevent low-integrity data from corrupting high-integrity data
⚠️ Critical Distinction: BLP uses classification/clearance levels (confidentiality). Biba uses integrity levels (trustworthiness). They look structurally similar but address opposite security properties.

4. Biba Properties (Rules)

4.1 Simple Integrity Property — "No Read Down"

🔑 Rule: A subject cannot read an object at a lower integrity level. Reading unreliable data could corrupt the subject's decision-making or output.

Example: A high-integrity financial reporting process should not read from an unvalidated, low-integrity data source — this could introduce corrupted or fraudulent data into financial reports.

4.2 Star (*) Integrity Property — "No Write Up"

🔑 Rule: A subject cannot write to an object at a higher integrity level. A less-trusted subject should not modify more-trusted data.

Example: An untrained intern's process (low integrity) should not be able to modify a production database (high integrity). This prevents unreliable sources from corrupting trusted systems.

4.3 Invocation Property

🔑 Rule: A subject cannot invoke (call/execute) another subject at a higher integrity level. This prevents a low-integrity process from leveraging a high-integrity process to indirectly modify trusted data.

6. Clark-Wilson Integrity Model

📖 Clark-Wilson Model (1987): A commercial integrity model by David Clark and David Wilson. Unlike Biba's lattice-based approach, Clark-Wilson ensures integrity through well-formed transactions and separation of duties. It is the most widely implemented integrity model in commercial environments.

Clark-Wilson recognizes that in real-world business, integrity is maintained not by access lattices, but by ensuring all data modifications go through validated, authorized processes with complete audit trails.

🔑 Two Key Integrity Goals:
1. Internal consistency: Data is internally valid and self-consistent
2. External consistency: Data accurately represents real-world facts it models

7. Clark-Wilson Components

7.1 Core Elements

ComponentAbbreviationDefinition
Constrained Data ItemsCDIHigh-integrity data objects protected by the model (e.g., account balances, financial records)
Unconstrained Data ItemsUDIInput data not yet validated (e.g., user input, external feeds). Must be validated before becoming CDIs.
Transformation ProceduresTPAuthorized programs/processes that modify CDIs. These are the "well-formed transactions" — programmed to maintain data integrity.
Integrity Verification ProceduresIVPProcedures that verify CDIs conform to integrity constraints. Like an auditor checking that debits = credits.

7.2 Clark-Wilson Access Control Triple

🔑 The Triple: (Subject, TP, CDI) — A user (subject) can only access a CDI through a specific authorized Transformation Procedure. Users never directly access data; they go through controlled programs.
CLARK-WILSON ACCESS MODEL ┌────────┐ ┌──────┐ ┌──────┐ │ Subject│ ──────→ │ TP │ ──────→ │ CDI │ │ (User) │ │(Prog)│ │(Data)│ └────────┘ └──────┘ └──────┘ │ │ │ Must be │ Maintains │ authorized │ integrity │ for this TP │ via well-formed │ │ transactions ▼ ▼ Separation IVP verifies of Duties CDI integrity

7.3 Clark-Wilson Enforcement Rules

  • C1: All IVPs must ensure CDIs are in a valid state (consistency)
  • C2: TPs must transform CDIs from one valid state to another (well-formed transactions)
  • C3: CDI access only through authorized TPs — separation of duty must be enforced
  • C4: Subjects must be authorized to execute specific TPs on specific CDIs (access triple)
  • C5: TPs must write to an append-only audit log
  • E1-E4: Enforcement rules that ensure the certification rules are implemented
💡 Exam Tip: Clark-Wilson's defining features are: (1) well-formed transactions (TPs), (2) separation of duties, and (3) the (Subject, TP, CDI) access triple. If any of these three appear in a question, the answer is Clark-Wilson.

9. Memory Tricks & Mnemonics

🧠 BLP vs. Biba — Mirror Rule:
BLP: No Read Up, No Write Down (protects Confidentiality)
Biba: No Read Down, No Write Up (protects Integrity)
Remember: They are exact opposites. Flip BLP's rules and you get Biba.
🧠 "BIBA = Boy I'm Being Accurate": Biba is about integrity — ensuring data accuracy and trustworthiness.
🧠 Clark-Wilson = "CW = Commercial World": Clark-Wilson is the commercial integrity model used in banking and business. Biba is the military/government integrity model.
🧠 CDI-TP-IVP: "Constrained Data goes through Transformation Programs, verified by Integrity Verification Procedures." Think of a bank: CDI = account balance, TP = transfer program, IVP = monthly reconciliation.
🧠 "Well-Formed Transaction" = Clark-Wilson: Whenever you see "well-formed transaction" on the exam, the answer is Clark-Wilson. No other model uses this term.
