3. What is Bell-LaPadula?

📖 Bell-LaPadula (BLP) Model: A formal state machine security model developed in 1973 by David Bell and Leonard LaPadula at MITRE Corporation for the U.S. Department of Defense. It focuses exclusively on confidentiality — preventing unauthorized disclosure of information.

BLP was the first mathematical model to formalize the concept of a secure state. It uses security labels (classification levels) assigned to subjects (users/processes) and objects (files/data) to enforce mandatory access control.

Classification Levels (Lattice)

BLP operates on a lattice of security levels, ordered from lowest to highest:

Top Secret ← Highest │ Secret │ Confidential │ Unclassified ← Lowest Subject clearance ≥ Object classification → Access may be granted

Each subject has a clearance level; each object has a classification level. The BLP rules determine what read/write operations are permitted based on these levels.

4. The State Machine Concept

🔑 State Machine: A system is modeled as a set of states. Each state defines the permissions between all subjects and objects. A secure state is one where no BLP rule is violated. If the system starts in a secure state and every transition (read, write, execute) preserves security, the system is always secure.

This is the Basic Security Theorem (BST) of BLP: if a system starts in a secure state and all state transitions are secure, every subsequent state is also secure.

💡 Exam Tip: The state machine concept is shared by BLP, Biba, and Clark-Wilson. What differs is what property each model protects: BLP = confidentiality, Biba = integrity, Clark-Wilson = integrity via well-formed transactions.

5. BLP Properties (Rules)

5.1 Simple Security Property (ss-property) — "No Read Up"

🔑 Rule: A subject cannot read an object at a higher classification level than the subject's clearance. A Secret-cleared user cannot read Top Secret documents.

5.2 Star (*) Property — "No Write Down"

🔑 Rule: A subject cannot write to an object at a lower classification level than the subject's current clearance. A Top Secret user cannot write to an Unclassified file. This prevents high-clearance subjects from leaking information to lower levels.
⚠️ Critical Understanding: The * property prevents information flow downward. Even if a Top Secret user creates an innocuous document, writing it to an Unclassified location would create a channel that could be exploited.

5.3 Strong Star Property

🔑 Rule: A subject can read and write only at their own classification level. No reading up, no writing down, no reading down, no writing up. This is the most restrictive form.

5.4 Discretionary Security Property (ds-property)

BLP also incorporates a discretionary access control matrix (like an ACL) that operates within the mandatory rules. Even if BLP allows access based on clearance levels, the DAC matrix can further restrict it.

6. Tranquility Principles

📖 Tranquility: Rules about whether classification/clearance levels can change during system operation.
PrincipleDefinition
Strong TranquilityClassification and clearance labels never change once assigned. Most restrictive — eliminates all label manipulation risks.
Weak TranquilityLabels can change, but changes must not violate the security policy. A document can be reclassified upward (never downward if it still contains classified data).
💡 Exam Tip: Strong tranquility is simpler to enforce and verify. Weak tranquility is more practical but introduces complexity. The exam may ask which type prevents label changes — the answer is Strong tranquility.

7. BLP Limitations & Criticisms

  • No integrity protection: BLP only addresses confidentiality. A subject can write up (to a higher level), which could corrupt higher-level data with unreliable lower-level input. This is the exact problem Biba's model addresses.
  • Covert channels: BLP does not address covert channels — hidden communication paths that bypass formal access controls (timing channels, storage channels).
  • No enforcement of "need to know": BLP uses clearance levels but doesn't enforce compartmentalized access within the same level.
  • Trusted subjects: BLP allows "trusted subjects" that can violate the * property when required (e.g., a system administrator declassifying data). This creates a potential bypass.
  • Static model: Under strong tranquility, BLP cannot adapt to changing business requirements without restarting the system or redefining labels.

💡 Exam Tip: BLP and Biba are exact opposites. BLP: No Read Up / No Write Down (protects confidentiality). Biba: No Read Down / No Write Up (protects integrity). Memorize this contrast — it's tested constantly.

9. Memory Tricks & Mnemonics

🧠 "BLP = No Read Up, No Write Down": Think "BLP Locks Papers" — papers (classified docs) are locked so you can't read above your level or leak (write) below it.
🧠 BLP vs. Biba Quick Rule:
BLP = "Bell Loves Privacy" → Confidentiality → No Read Up, No Write Down
Biba = "Biba loves Integrity" → Integrity → No Read Down, No Write Up
They are mirror images of each other.
🧠 Star (*) Property: Think of the * as a "splash" — information can't "splash down" from a higher classification to a lower one.
🧠 WURD for BLP: Write Up, Read Down — what BLP allows. The opposite (Read Up, Write Down) is what BLP blocks.
