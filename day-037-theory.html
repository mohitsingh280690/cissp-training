3. Brewer-Nash (Chinese Wall) Model

📖 Brewer-Nash Model (1989): A dynamic access control model that prevents conflicts of interest. Subjects are denied access to data that would create a conflict of interest with data they have already accessed. Named after the "Chinese Wall" ethical barrier concept in finance.

How It Works

Data is organized into conflict-of-interest classes (COI classes). Each class contains company datasets. Once a subject accesses one company's data in a COI class, they are blocked from accessing any other company's data in the same class.

COI Class: Banking ┌─────────┐ ┌─────────┐ ┌─────────┐ │ Bank A │ │ Bank B │ │ Bank C │ │ data │ │ data │ │ data │ └─────────┘ └─────────┘ └─────────┘ ↑ ╳ ╳ Analyst BLOCKED BLOCKED accessed (conflict) (conflict) Bank A COI Class: Oil Companies ┌─────────┐ ┌─────────┐ │ Shell │ │ BP data │ │ data │ │ │ └─────────┘ └─────────┘ ↑ ╳ Same analyst BLOCKED can access Shell (different COI class) but once Shell is accessed, BP is blocked
🔑 Key Properties:
• Dynamic: Access permissions change based on previous access history — unlike static models (BLP, Biba)
• Write rule: A subject can write to an object only if they cannot read any object in a different company dataset within the same COI class
• Over time: The wall grows taller — users lose access options as they view more data
💡 Exam Tip: Brewer-Nash is the ONLY security model that uses dynamic, history-based access decisions. If a question mentions "conflict of interest," "Chinese Wall," or "consulting firm accessing competing client data," the answer is Brewer-Nash.

4. Graham-Denning Model

📖 Graham-Denning Model (1972): Defines a formal set of eight primitive operations (protection rules) for securely creating, deleting, and managing subjects, objects, and access rights. It addresses what BLP, Biba, and Clark-Wilson do not — how to manage access rights themselves.

The Eight Primitive Operations

#OperationDescription
1Create ObjectSecurely create a new object in the system
2Create SubjectSecurely create a new subject
3Delete ObjectRemove an object and all associated rights
4Delete SubjectRemove a subject and revoke all its rights
5Read Access RightRead the access right of a subject to an object
6Grant Access RightGrant a specific access right from one subject to another
7Delete Access RightRemove a specific access right
8Transfer Access RightTransfer an access right from one subject to another
💡 Exam Tip: Graham-Denning is about managing access rights — creating, deleting, reading, granting, and transferring rights. If a question asks about "how to securely create/delete subjects and objects" or "primitive protection operations," the answer is Graham-Denning.

5. Harrison-Ruzzo-Ullman (HRU) Model

📖 HRU Model (1976): An extension of the Graham-Denning model that formally proves the decidability of access control. It uses an access control matrix and defines commands (sequences of primitive operations) to modify rights.

Key Contribution: The Safety Problem

HRU addresses: "Can the system ever reach an unsafe state?" — known as the safety problem.

⚠️ Critical Result: HRU proved that the general safety problem is undecidable — there is no algorithm that can determine for all possible systems whether an unsafe state is reachable. However, for restricted (mono-operational) systems, safety IS decidable.
AspectDetail
Based OnGraham-Denning's eight operations, formalized as commands
MatrixUses an access control matrix (subjects × objects × rights)
Key ProofGeneral safety problem is undecidable; mono-operational commands allow decidability
SignificanceEstablishes theoretical limits on what access control verification can prove

6. Lipner Model

📖 Lipner Model (1982): Combines Bell-LaPadula (confidentiality) and Biba (integrity) into a single unified model that addresses both properties simultaneously. Developed by Steven Lipner.

How It Works

Lipner assigns subjects and objects both a BLP confidentiality label and a Biba integrity label. Access decisions must satisfy both models' rules simultaneously.

Rule SourcePropertyEnforcement
BLPConfidentialityNo Read Up, No Write Down (classification-based)
BibaIntegrityNo Read Down, No Write Up (integrity-based)
CombinedBothBoth rule sets must pass for access to be granted
🔑 Lipner's Practical Approach: Lipner defined specific categories and levels for a commercial data processing environment:
• Confidentiality levels: Public, Confidential
• Integrity levels: System High, System Low
• Categories separate production data from development data and production code from development code

7. Take-Grant Model

📖 Take-Grant Model: A graph-based model that uses directed graphs to represent rights transfer between subjects and objects. It defines four primitive operations: Take, Grant, Create, and Remove.
OperationDescription
TakeA subject takes access rights from another subject or object
GrantA subject grants its access rights to another subject
CreateA subject creates a new object with specific rights
RemoveA subject removes rights from another subject or object
Take-Grant Graph Example Subject A ──take──→ Object X │ │ grant rights ▼ Subject B ──(now has rights)──→ Object X
🔑 Key Feature: The Take-Grant model can determine if rights can "leak" — whether there exists any sequence of operations that allows an unauthorized subject to gain specific rights. Unlike HRU, this analysis is decidable because the model restricts operation types.

9. Memory Tricks & Mnemonics

🧠 "Chinese Wall = Brewer-Nash = Conflict of Interest": A consulting firm builds a "wall" between competing clié data. The wall gets higher each time an analyst accesses one client — blocking access to the competitor.
🧠 Graham-Denning = "8 G.D. Rules": G.D. has 8 rules for creating/deleting/granting/transferring rights. Think "Great Definitions" for how to manage access rights.
🧠 HRU = "Hard to Resolve Universally": The general safety problem is undecidable — you can't always prove a system is safe. Hard to Resolve Universally.
🧠 Lipner = "Links BLP + Biba": Lipner Links BLP and Biba. It combines confidentiality and integrity.
🧠 Take-Grant = "TG = Take, Grant, Create, Remove": Four operations on a directed graph showing how rights flow between subjects.
