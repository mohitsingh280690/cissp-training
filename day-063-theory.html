1. Where This Fits in CISSP

Domain4 – Communication and Network Security (13%)
Why TestedWireless is pervasive in organizations — and is the #1 vector where cryptographic weakness directly enabled mass eavesdropping (WEP, WPA-TKIP). CISSP tests the evolution from WEP → WPA → WPA2 → WPA3, specific cryptographic weaknesses in each, and the full landscape of wireless attacks. You must know WPA3's SAE advantage over WPA2's PSK.
Typical Question Styles"Which wireless security protocol is considered broken and should never be used?"; "What does SAE in WPA3 prevent compared to WPA2?"; "An attacker captures wireless frames and performs an offline dictionary attack — which protocol is vulnerable?"; "Which attack exploits a rogue AP with the same SSID?"

Hiding SSID is NOT a security control — it's security through obscurity. MAC filtering is easily bypassed by spoofing. 802.1X is the enterprise authentication framework, not an encryption protocol.

🔑 Security-Relevant 802.11 Amendments:
802.11i (2004) — the amendment that defined WPA2 (AES-CCMP). This is the security standard, distinct from the physical layer amendments above.
802.11w (2009) — Management Frame Protection (MFP). Protects management frames (deauthentication, disassociation) against spoofing. Without 802.11w, attackers can forge de-auth frames to disconnect clients. Mandatory in WPA3.

4. WEP – Wired Equivalent Privacy (Broken)

📖 WEP: The original wireless security protocol from 1999. Completely broken — do not use. Passive eavesdropping can recover the WEP key in minutes from captured traffic using freely available tools (Aircrack-ng, AirSnort).

WEP Cryptographic Weaknesses

WeaknessTechnical DetailImpact
RC4 with weak IVUses RC4 stream cipher with a 24-bit Initialization Vector (IV). With only 2^24 (16.7 million) possible IVs, IV collision is inevitable on a busy network (birthday problem).When same IV+key is reused, XOR of two ciphertexts reveals plaintext. Statistical analysis of IV reuse (PTW attack, FMS attack) recovers the WEP key.
Short key lengths40-bit or 104-bit RC4 key (the "64-bit" and "128-bit" WEP options included the 24-bit IV)Even the longer key is broken via statistical attacks — key length is irrelevant when the IV mechanism is fundamentally flawed
CRC-32 integrity checkUses CRC-32 for integrity checking — CRC is not a cryptographic MAC; it is linear and can be modified predictablyBit-flipping attacks: attacker can modify encrypted frames without knowing the key and produce a valid CRC
Static keysOne shared key for all users on the network; no per-session key derivationAny user who knows the key can decrypt all other users' traffic; compromised key affects everyone
⚠️ WEP is Irreparably Broken: The IEEE deprecated WEP in 2004. Any system still using WEP has zero effective encryption. On the CISSP exam, WEP is always the "wrong" security choice. WPA3 is the current best practice; WPA2 is the acceptable minimum.

5. WPA – Wi-Fi Protected Access

📖 WPA (Wi-Fi Protected Access): Introduced in 2003 as an emergency interim fix for WEP while WPA2 (802.11i) was being finalized. Uses TKIP (Temporal Key Integrity Protocol) as an encryption improvement designed to run on existing WEP hardware.
FeatureWPA/TKIP Detail
EncryptionTKIP (Temporal Key Integrity Protocol) — uses RC4 but with per-packet key mixing and a 128-bit temporal key, extended IV (48-bit), and sequence counter to prevent replay attacks
IntegrityMIC (Message Integrity Code) — Michael algorithm — replaced CRC-32 to prevent bit-flipping attacks
Key managementPer-session key derivation (4-way handshake) — eliminates static shared key weakness of WEP
StatusDeprecated. TKIP has known vulnerabilities (TKIP MIC attacks, Beck-Tews attack on short packets). IEEE deprecated TKIP in 2012. Must not be used for new deployments.
💡 Exam Tip — WPA TKIP: The critical fact: WPA used TKIP (RC4-based). WPA2 replaced TKIP with AES-CCMP. On the exam if asked "which uses TKIP?" → WPA. "Which uses AES-CCMP?" → WPA2. TKIP is explicitly associated with backward compatibility and legacy hardware — it is the weaker option and should not be selected.

6. WPA2 – The Long-Running Standard

📖 WPA2 (802.11i): Ratified in 2004, WPA2 replaced TKIP with AES-CCMP — a true cryptographically strong encryption standard. WPA2 has been the mandatory Wi-Fi Alliance certification standard since 2006 and remains widely deployed. Comes in two modes: Personal (PSK) and Enterprise (802.1X/RADIUS).
FeatureWPA2 Personal (PSK)WPA2 Enterprise (802.1X)
EncryptionAES-CCMP (Counter Mode with CBC-MAC)AES-CCMP (same)
AuthenticationPre-Shared Key: all users share one password; 4-way handshake derives session keys from PSK + SSID + random noncesRADIUS + EAP (EAP-TLS, PEAP, EAP-TTLS); individual credentials per user; PKI certificates
Key vulnerability4-way handshake can be captured passively; offline dictionary attack against PSK (Hashcat, Wireshark + wordlist). PMKID attack allows capture without client association.No shared secret to attack; per-user credentials with mutual authentication. MitM attack surface greatly reduced.
Use caseHome, small offices, guest networksEnterprise — any environment with individual user accountability
🔑 KRACK Attack (2017): Key Reinstallation Attack — a vulnerability in the WPA2 4-way handshake. By replaying handshake messages, an attacker can force nonce reuse in AES-CCMP, breaking encryption. Patched in updated clients and APs. Illustrates that implementation flaws can undermine strong cryptography. KRACK is patched in all modern devices. WPA3's SAE prevents this class of attack by design.

7. WPA3 – Current Standard

📖 WPA3: Released in 2018, WPA3 addresses WPA2's PSK offline dictionary attack vulnerability by replacing PSK with SAE (Simultaneous Authentication of Equals), also known as the Dragonfly handshake. WPA3 also mandates PMF (Protected Management Frames) per 802.11w.
FeatureWPA3 PersonalWPA3 Enterprise
AuthenticationSAE (Simultaneous Authentication of Equals) — Dragonfly handshake. Forward secrecy: even if password is later compromised, past sessions cannot be decrypted.192-bit security mode (GCMP-256, BIP-GMAC-256) for high-security environments. Still uses 802.1X/RADIUS.
Offline attacksSAE prevents offline dictionary attacks. An attacker must interact with the AP for each password guess — online guessing only, which can be rate-limited.N/A (enterprise uses per-user certificates)
Management framesPMF (Protected Management Frames) mandatory — deauth/disassoc frames cannot be forgedSame — PMF mandatory
Forward secrecyYes — unique session keys per connection; past sessions safe even if password compromised laterYes (inherent in 802.1X with EAP-TLS ephemeral key exchange)
Open networksWPA3 Enhanced Open / OWE (Opportunistic Wireless Encryption): encrypts open network traffic with unique per-client keys, preventing passive eavesdropping even without authentication. Café and airport Wi-Fi use case.

8. Wireless Attacks

AttackDescriptionDefence
Evil Twin / Rogue APAttacker sets up a malicious AP broadcasting the same SSID as the legitimate network. Clients auto-connect, believing it's the real network. Attacker performs MitM — intercepts all traffic, captures credentials, injects malware.802.1X (mutual auth prevents MitM); WIDS (wireless IDS detects duplicate SSID); VPN over wireless
WardrivingDriving through areas with a laptop and wireless adapter scanning for open or poorly secured Wi-Fi networks. Wardialing is the telephone equivalent. Intent: identify targets for later attack.WPA2/3 Enterprise; monitor for unauthorized SSIDs; restrict signal strength
Deauthentication AttackAttacker sends spoofed 802.11 Management Deauthentication frames (forged as AP) to force client disconnection. Used to: (a) capture WPA2 4-way handshake when client reconnects; (b) simple DoS attack. Unprotected management frames (pre-802.11w) cannot be authenticated.802.11w (PMF) — makes management frames unauthenticated = rejected. WPA3 mandates PMF.
WPA2 PSK Offline Dictionary AttackCapture WPA2 4-way handshake (or use PMKID attack without client) then run offline dictionary/brute-force against the PSK hash. Tools: Hashcat, hcxdumptool + hcxtools.WPA3 SAE (prevents offline attack); use long complex PSK (20+ characters); WPA2 Enterprise (no shared secret)
WEP CrackingPassive IV collection (Airodump-ng); active IV injection to accelerate collection (Aireplay-ng); key recovery (Aircrack-ng). WEP key recoverable in minutes with sufficient IV capture (~40,000 unique IVs).Never use WEP. Migrate to WPA2/3 immediately.
Wireless EavesdroppingPassive sniffing of wireless frames in range. On open (unencrypted) networks — trivial. On WEP/WPA-TKIP — feasible. On WPA2 PSK — attacker who knows the PSK can decrypt all traffic (no forward secrecy).WPA3 (forward secrecy via SAE); VPN overlay; OWE for open networks
Jamming / RF InterferenceFlood 2.4 GHz or 5 GHz spectrum with noise to deny wireless service. Physical DoS. Can be targeted or broad (affects whole building).WIDS detects RF jamming; 802.11ax (Wi-Fi 6) better handles interference; move to 6 GHz (Wi-Fi 6E) band

9. Wireless Security Controls

ControlEffectivenessNotes
WPA3 Enterprise (AES-CCMP + RADIUS)HighBest practice; individual user authentication; mutual auth prevents rogue AP; forward secrecy
802.1X EAP-TLSHighCertificate-based mutual authentication; requires PKI; strongest EAP method; completely prevents PSK attacks
Wireless IDS (WIDS)HighDetects rogue APs, evil twins, deauth attacks, anomalous clients; commercial products: Cisco WLC WIDS, Rogue AP containment
VPN over wirelessHighEven if wireless layer is compromised, VPN provides encryption overlay — defence in depth
Network segmentation (guest VLAN)HighIsolate guest wireless from corporate LAN at L2/L3; prevent guest-to-corporate lateral movement
MAC address filteringVery LowSecurity theatre — trivially bypassed by sniffing a legitimate MAC and spoofing it. Not a CISSP-recognized security control.
SSID hidingNoneSecurity through obscurity. SSID is visible in probe frames. Any passive scanner reveals hidden SSIDs. Not a security control.
Power level managementLow-MediumReduce AP transmit power to minimize signal coverage outside building. Reduces attack surface but sophisticated attackers use high-gain antennas.
⚠️ MAC Filtering and SSID Hiding are NOT Security Controls: These are the two most common "security theatre" wireless measures. MAC addresses are transmitted unencrypted in 802.11 frames — any attacker can observe a valid MAC and spoof it within seconds. Hidden SSIDs are revealed in probe request/response frames visible to any passive scanner. The CISSP exam will present these as distractors. The correct answer will always be WPA2/3 Enterprise + 802.1X.

10. Enterprise Wireless: RADIUS & 802.1X

📖 802.1X (Port-Based Network Access Control): An IEEE standard for authentication before network access is granted. Originally designed for wired Ethernet port access; adopted by Wi-Fi for enterprise authentication. 802.1X requires three components: Supplicant (client device), Authenticator (AP or switch), and Authentication Server (RADIUS server).
802.1X / EAP Authentication Flow: [Client/Supplicant] ── 802.1X EAP ──▶ [AP/Authenticator] ── RADIUS ──▶ [RADIUS Server] │ │ ◀── EAP-Request ─────────│ │ ─── EAP-Response ─────────▶── RADIUS Access-Request ──────────▶ │ │ │◀─── RADIUS Access-Challenge ────────│ ◀── EAP-Request(challenge)│ │ ─── EAP-Response(cred) ───▶── RADIUS Access-Request ──────────▶ │ │ │◀─── RADIUS Access-Accept ───────────│ ◀── EAP-Success ──────────│ │ (Wireless access granted) Until EAP-Success: port is UNCONTROLLED (only EAP traffic allowed) After EAP-Success: port CONTROLLED (network access granted) Per-user session key (PMK) derived from successful EAP auth → WPA2/3 4-way handshake uses this key

11. Cellular & Mobile Network Security (4G/5G)

📖 Cellular Networks: Wireless communications using licensed radio spectrum divided into geographic cells. Each cell is served by a base station (eNodeB in 4G LTE, gNodeB in 5G NR). Devices authenticate via SIM to the mobile network operator (MNO).
GenerationStandardEncryptionKey Security Notes
2G (GSM)GSM/GPRSA5/1 (weak RC4 variant)One-way authentication only — network authenticates device but device does NOT authenticate network. Vulnerable to IMSI catchers (stingrays). A5/2 is completely broken.
3G (UMTS)UMTS/HSPAKASUMI (128-bit)Mutual authentication introduced. Still vulnerable to downgrade attacks forcing fallback to 2G.
4G (LTE)LTE/LTE-AAES-128 (EEA2)All-IP network; mutual authentication; separate encryption for control and user planes. SS7 signaling protocol still vulnerable in backend carrier interconnects.
5G NR5G New RadioAES-256 / SNOW 3G / ZUCSUPI encrypted as SUCI — prevents IMSI exposure to IMSI catchers. Strengthened mutual auth. Network slicing with security isolation. Reduced downgrade attack surface.

Cellular Security Threats

AttackHow It WorksMitigation
IMSI Catcher (Stingray)Fake base station forces devices to connect at 2G (no mutual auth). Captures IMSI identifier; enables location tracking and call interception.5G SUCI protects IMSI; some devices allow disabling 2G fallback; use VPN over cellular
SS7 AttacksSignaling System 7 (the global carrier interconnect protocol) has no authentication — any carrier on the SS7 network can query subscriber location, intercept calls/SMS. SMS-based 2FA is vulnerable.SS7 firewalls at carrier level; migrate from SMS OTP to TOTP authenticator apps or hardware tokens
SIM SwappingAttacker social-engineers carrier to transfer victim's number to attacker's SIM. Intercepts all SMS including 2FA codes.Carrier PIN/port-out protection; use authenticator apps instead of SMS 2FA
Downgrade AttackAttacker jams 4G/5G signal; device falls back to 2G with weaker security.Disable 2G on device where possible; use VPN over all cellular connections
💡 CISSP Key Point — SMS OTP is NOT secure 2FA: SS7 vulnerabilities and SIM swapping mean SMS-based one-time passwords can be intercepted or redirected. NIST SP 800-63B classifies SMS OTP as "RESTRICTED" (not recommended for high-assurance authentication). CISSP favors TOTP apps or hardware tokens over SMS 2FA.

RFID & NFC Security Concerns

ThreatDescriptionMitigation
RFID CloningLegacy 125 kHz RFID access cards transmit a fixed ID with no cryptography — readable and cloneable with a <$20 device in seconds.Upgrade to smart cards with cryptographic challenge-response (MIFARE DESFire, HID iCLASS SE)

💡 Key CISSP Points — Bluetooth & RFID:
• Bluejacking = annoying spam, no data stolen
• Bluesnarfing = data exfiltrated (contacts, files)
• Bluebugging = full device control
• Legacy RFID 125 kHz = no crypto = trivially cloneable in seconds
• Smart card RFID 13.56 MHz (MIFARE DESFire) = cryptographic authentication = secure
• NFC range is <10 cm — relay attacks artificially extend this range
