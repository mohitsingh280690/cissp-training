🔑 Security-Relevant 802.11 Amendments:
802.11i (2004) — the amendment that defined WPA2 (AES-CCMP). This is the security standard, distinct from the physical layer amendments above.
802.11w (2009) — Management Frame Protection (MFP). Protects management frames (deauthentication, disassociation) against spoofing. Without 802.11w, attackers can forge de-auth frames to disconnect clients. Mandatory in WPA3.

4. WEP – Wired Equivalent Privacy (Broken)

📖 WEP: The original wireless security protocol from 1999. Completely broken — do not use. Passive eavesdropping can recover the WEP key in minutes from captured traffic using freely available tools (Aircrack-ng, AirSnort).

WEP Cryptographic Weaknesses

WeaknessTechnical DetailImpact
RC4 with weak IVUses RC4 stream cipher with a 24-bit Initialization Vector (IV). With only 2^24 (16.7 million) possible IVs, IV collision is inevitable on a busy network (birthday problem).When same IV+key is reused, XOR of two ciphertexts reveals plaintext. Statistical analysis of IV reuse (PTW attack, FMS attack) recovers the WEP key.
Short key lengths40-bit or 104-bit RC4 key (the "64-bit" and "128-bit" WEP options included the 24-bit IV)Even the longer key is broken via statistical attacks — key length is irrelevant when the IV mechanism is fundamentally flawed
CRC-32 integrity checkUses CRC-32 for integrity checking — CRC is not a cryptographic MAC; it is linear and can be modified predictablyBit-flipping attacks: attacker can modify encrypted frames without knowing the key and produce a valid CRC
Static keysOne shared key for all users on the network; no per-session key derivationAny user who knows the key can decrypt all other users' traffic; compromised key affects everyone
⚠️ WEP is Irreparably Broken: The IEEE deprecated WEP in 2004. Any system still using WEP has zero effective encryption. On the CISSP exam, WEP is always the "wrong" security choice. WPA3 is the current best practice; WPA2 is the acceptable minimum.

5. WPA – Wi-Fi Protected Access

📖 WPA (Wi-Fi Protected Access): Introduced in 2003 as an emergency interim fix for WEP while WPA2 (802.11i) was being finalized. Uses TKIP (Temporal Key Integrity Protocol) as an encryption improvement designed to run on existing WEP hardware.
FeatureWPA/TKIP Detail
EncryptionTKIP (Temporal Key Integrity Protocol) — uses RC4 but with per-packet key mixing and a 128-bit temporal key, extended IV (48-bit), and sequence counter to prevent replay attacks
IntegrityMIC (Message Integrity Code) — Michael algorithm — replaced CRC-32 to prevent bit-flipping attacks
Key managementPer-session key derivation (4-way handshake) — eliminates static shared key weakness of WEP
StatusDeprecated. TKIP has known vulnerabilities (TKIP MIC attacks, Beck-Tews attack on short packets). IEEE deprecated TKIP in 2012. Must not be used for new deployments.
💡 Exam Tip — WPA TKIP: The critical fact: WPA used TKIP (RC4-based). WPA2 replaced TKIP with AES-CCMP. On the exam if asked "which uses TKIP?" → WPA. "Which uses AES-CCMP?" → WPA2. TKIP is explicitly associated with backward compatibility and legacy hardware — it is the weaker option and should not be selected.

6. WPA2 – The Long-Running Standard

📖 WPA2 (802.11i): Ratified in 2004, WPA2 replaced TKIP with AES-CCMP — a true cryptographically strong encryption standard. WPA2 has been the mandatory Wi-Fi Alliance certification standard since 2006 and remains widely deployed. Comes in two modes: Personal (PSK) and Enterprise (802.1X/RADIUS).
FeatureWPA2 Personal (PSK)WPA2 Enterprise (802.1X)
EncryptionAES-CCMP (Counter Mode with CBC-MAC)AES-CCMP (same)
AuthenticationPre-Shared Key: all users share one password; 4-way handshake derives session keys from PSK + SSID + random noncesRADIUS + EAP (EAP-TLS, PEAP, EAP-TTLS); individual credentials per user; PKI certificates
Key vulnerability4-way handshake can be captured passively; offline dictionary attack against PSK (Hashcat, Wireshark + wordlist). PMKID attack allows capture without client association.No shared secret to attack; per-user credentials with mutual authentication. MitM attack surface greatly reduced.
Use caseHome, small offices, guest networksEnterprise — any environment with individual user accountability
🔑 KRACK Attack (2017): Key Reinstallation Attack — a vulnerability in the WPA2 4-way handshake. By replaying handshake messages, an attacker can force nonce reuse in AES-CCMP, breaking encryption. Patched in updated clients and APs. Illustrates that implementation flaws can undermine strong cryptography. KRACK is patched in all modern devices. WPA3's SAE prevents this class of attack by design.

7. WPA3 – Current Standard

📖 WPA3: Released in 2018, WPA3 addresses WPA2's PSK offline dictionary attack vulnerability by replacing PSK with SAE (Simultaneous Authentication of Equals), also known as the Dragonfly handshake. WPA3 also mandates PMF (Protected Management Frames) per 802.11w.
FeatureWPA3 PersonalWPA3 Enterprise
AuthenticationSAE (Simultaneous Authentication of Equals) — Dragonfly handshake. Forward secrecy: even if password is later compromised, past sessions cannot be decrypted.192-bit security mode (GCMP-256, BIP-GMAC-256) for high-security environments. Still uses 802.1X/RADIUS.
Offline attacksSAE prevents offline dictionary attacks. An attacker must interact with the AP for each password guess — online guessing only, which can be rate-limited.N/A (enterprise uses per-user certificates)
Management framesPMF (Protected Management Frames) mandatory — deauth/disassoc frames cannot be forgedSame — PMF mandatory
Forward secrecyYes — unique session keys per connection; past sessions safe even if password compromised laterYes (inherent in 802.1X with EAP-TLS ephemeral key exchange)
Open networksWPA3 Enhanced Open / OWE (Opportunistic Wireless Encryption): encrypts open network traffic with unique per-client keys, preventing passive eavesdropping even without authentication. Café and airport Wi-Fi use case.

⚠️ MAC Filtering and SSID Hiding are NOT Security Controls: These are the two most common "security theatre" wireless measures. MAC addresses are transmitted unencrypted in 802.11 frames — any attacker can observe a valid MAC and spoof it within seconds. Hidden SSIDs are revealed in probe request/response frames visible to any passive scanner. The CISSP exam will present these as distractors. The correct answer will always be WPA2/3 Enterprise + 802.1X.

10. Enterprise Wireless: RADIUS & 802.1X

📖 802.1X (Port-Based Network Access Control): An IEEE standard for authentication before network access is granted. Originally designed for wired Ethernet port access; adopted by Wi-Fi for enterprise authentication. 802.1X requires three components: Supplicant (client device), Authenticator (AP or switch), and Authentication Server (RADIUS server).
802.1X / EAP Authentication Flow: [Client/Supplicant] ── 802.1X EAP ──▶ [AP/Authenticator] ── RADIUS ──▶ [RADIUS Server] │ │ ◀── EAP-Request ─────────│ │ ─── EAP-Response ─────────▶── RADIUS Access-Request ──────────▶ │ │ │◀─── RADIUS Access-Challenge ────────│ ◀── EAP-Request(challenge)│ │ ─── EAP-Response(cred) ───▶── RADIUS Access-Request ──────────▶ │ │ │◀─── RADIUS Access-Accept ───────────│ ◀── EAP-Success ──────────│ │ (Wireless access granted) Until EAP-Success: port is UNCONTROLLED (only EAP traffic allowed) After EAP-Success: port CONTROLLED (network access granted) Per-user session key (PMK) derived from successful EAP auth → WPA2/3 4-way handshake uses this key

11. Memory Tricks & Mnemonics

🧠 WEP → WPA → WPA2 → WPA3 — "WEP Breaks, WPA Transitions, WPA2 Advances, WPA3 Secures":
WEP = RC4 + 24-bit IV = BROKEN (aircrack recovers key in minutes)
WPA = TKIP (RC4 improved) = DEPRECATED 2012
WPA2 = AES-CCMP = strong encryption; PSK vuln. to offline dict attack (KRACK 2017)
WPA3 = AES-CCMP + SAE = forward secrecy; offline attack eliminated; PMF mandatory
🧠 SAE Advantage — "SAE = Online Only":
WPA2 PSK: attacker captures handshake → runs offline dictionary for days/months → no interaction with AP needed.
WPA3 SAE: each guess requires a live interaction with the AP → rate-limiting defeats dictionary attacks.
"SAE puts the AP as the bouncer — you have to prove yourself in person, not with a forged ID."
🧠 MAC Filtering + SSID Hiding = Security Theatre:
"A thief who can see your house number despite your blind isn't stopped by the blind."
Both are visible in wireless frames. Neither prevents a motivated attacker. Don't even mention these as controls on the CISSP exam — choose 802.1X/RADIUS instead.
🧠 Evil Twin vs. Rogue AP:
Evil Twin = same SSID as legitimate network; specifically intended to intercept traffic (active MitM).
Rogue AP = unauthorized AP on the corporate network (plugged in by employee); may or may not be malicious.
Both detected by WIDS. Both blocked by 802.1X (mutual auth refuses connections to unauthorized APs).
