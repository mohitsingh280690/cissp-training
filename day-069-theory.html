4. Firewall & Network Defence Technology Review

TechnologyLayerKey CharacteristicsLimitation
Packet-filter firewallL3–L4Static rules on IP/port; stateless; fastNo session state; no application inspection; vulnerable to IP spoofing
Stateful Packet Inspection (SPI)L3–L4Tracks TCP/UDP sessions; only permits return traffic for established sessionsCannot inspect application payload; no URL/content filtering
Application-layer firewall (proxy)L7Full packet reassembly; circuit-level proxy; deep inspectionHigh CPU overhead; must understand each application protocol
NGFW (Next-Generation Firewall)L3–L7SPI + application identification + IPS + TLS inspection + identity-based rules + URL filteringExpensive; requires TLS inspection for HTTPS traffic (man-in-the-middle to decrypt)
WAF (Web App Firewall)L7 (HTTP)SQLi, XSS, OWASP Top 10 defence; sits in front of web app; positive and negative security modelsHTTP/HTTPS only; bypassed by unknown/novel attacks not in rule set
IDS (Intrusion Detection)L3–L7Passive monitoring; alerts on matches; no traffic blocking; out-of-bandOnly detect, not prevent; IDS with high FP rates cause alert fatigue
IPS (Intrusion Prevention)L3–L7Inline; blocks matched traffic in real-time; active responseFalse positives block legitimate traffic; inline = single point of failure

5. VPN & Wireless Security Review

TopicKey Facts
IPSec AH (Protocol 51)Authentication + integrity only; no encryption; incompatible with NAT (protects IP header incl. source IP which NAT changes); rarely used alone
IPSec ESP (Protocol 50)Encryption + authentication + integrity; NAT-compatible (with NAT-T, UDP 4500 encapsulation); used in all modern IPSec VPNs
IPSec Transport modeProtects payload only; original IP header visible; used for host-to-host
IPSec Tunnel modeEncapsulates entire original packet in new IP packet; hides internal addressing; used for site-to-site VPNs
IKEv1 weaknessAggressive mode transmits identity hash in the clear — offline cracking possible; use IKEv2 (MOBIKE, EAP, simpler)
WEPBroken: RC4 + 24-bit IV reuse → FMS/PTW attacks crack key in minutes; never use
WPA/TKIPRC4 with 128-bit key per packet (better than WEP); deprecated 2012; avoid
WPA2 / AES-CCMPStrong encryption; PSK mode vulnerable to offline dictionary attack (capture 4-way handshake + PMKID); KRACK (2017) vulnerability in 4-way handshake; use Enterprise mode (802.1X)
WPA3 / SAEReplaces PSK with SAE (Simultaneous Authentication of Equals — Dragonfly) providing forward secrecy; online-only guessing; PMF mandatory; resistant to KRACK
802.1X / EAPPort-based network access control; EAP-TLS (strongest, mutual cert auth); PEAP (server cert only, MS-CHAPv2 inside TLS tunnel — most common); EAP-MD5 (deprecated)

7. Email Security Quick Reference

TechnologyWhat It ChecksWhat It Does NOT Protect
SPFSMTP MAIL FROM (envelope sender) domain vs. authorized sending IPs (DNS TXT)Does NOT check user-visible From: header; breaks on email forwarding
DKIMCryptographic signature on message headers + body; public key in DNS; verifies integrity and signing domainDoes NOT enforce policy; survives forwarding; does not check From: header alignment alone
DMARCAlignment of SPF/DKIM results with From: header domain; enforces policy (none/quarantine/reject); provides reportsOnly as good as SPF + DKIM underneath; p=none = monitoring only, no enforcement
S/MIMEEnd-to-end message encryption (recipient pubkey) and signing (sender privkey); X.509 certificatesDoes NOT protect metadata; requires certificate exchange with each recipient
PGP/GPGSame E2EE operations as S/MIME; Web of Trust (no central CA)No centralized CA — trust relies on manual key signing; harder to deploy at enterprise scale
💡 DMARC alignment rule: DMARC PASSES if: (SPF passes AND MAIL FROM domain aligns with From: domain) OR (DKIM passes AND DKIM d= domain aligns with From: domain). At least ONE must align.

8. Secure Communications Quick Reference

TopicKey Facts
TLS 1.3Forward secrecy mandatory (ECDHE/DHE only); 1-RTT handshake; RSA key exchange removed; 5 AEAD cipher suites only; most of handshake encrypted
Forward secrecyEphemeral session keys — past sessions safe even if long-term key is later compromised; TLS 1.2 with RSA key exchange does NOT provide FS
HSTSInstructs browser to refuse HTTP connections for the domain; max-age; preloading bakes into browsers; defeats SSL stripping
SSHPort 22; replaces Telnet (23); public key auth preferred; used for admin, SFTP, SCP, tunneling
SFTP vs FTPS vs SCPSFTP = SSH subsystem (port 22, full file management); SCP = SSH copy (port 22, one-way copy); FTPS = FTP+TLS (port 990 implicit / 21 explicit); FTP = plaintext never use
SRTPRTP + AES-CTR encryption + HMAC-SHA1; required to prevent VoIP eavesdropping; SIP must also use TLS (SIPS) for complete confidentiality
Signal ProtocolDouble Ratchet (per-message forward secrecy + break-in recovery) + X3DH (async key agreement); used by Signal, WhatsApp; E2EE means server cannot read messages

10. High-Value Exam Traps for Domain 4

⚠️ Common Traps — Memorize These:
TrapCorrect Answer
Hub vs. Switch securityHub broadcasts all frames to all ports — passive sniffing possible. Switch sends frames only to the destination port. Switches do NOT prevent all sniffing (SPAN ports, ARP poisoning re-enables it).
ARP is at Layer…Layer 2 (Data Link) — ARP maps IP (L3) to MAC (L2) but operates at L2. Often considered to bridge L2/L3.
IPSec AH vs ESPAH = auth only, no encryption, NAT-incompatible. ESP = encryption + auth, NAT-compatible. Use ESP in practice.
SPF checks which address?MAIL FROM (envelope sender) — NOT the From: header visible to users. Attackers can pass SPF while spoofing the visible From: header.
DMARC enforcementp=none = monitor only (no action!). Only p=reject blocks spoofed email. "Deploying DMARC" without p=reject does nothing to block attacks.
SFTP runs on which port?Port 22 (SSH). NOT port 21 (that's FTP). SFTP is NOT "FTP with SSL" — that's FTPS. SFTP is a completely different protocol that happens to have "FTP" in the name.
TLS terminates at…In CDN/reverse proxy deployments, TLS terminates at the proxy/CDN edge — NOT end-to-end. The CDN/proxy can see plaintext. End-to-end encryption (S/MIME, PGP) is different.
VLAN securityVLANs alone are NOT equivalent to firewall security zones. VLAN hopping attacks (802.1Q double-tagging) can bypass VLANs. Inter-VLAN traffic must route through a firewall.
Replay vs. session hijackReplay = captured token reused (session may be over). Session hijack = stealing an active live session. Both defeated by different mechanisms (nonce/timestamp vs. HTTPS/HttpOnly).
DoS vs. DDoS mitigationSmall DoS: firewall rate limiting works. Large volumetric DDoS: ONLY cloud scrubbing/CDN can absorb it — on-premise firewall is saturated before it can even respond.
WPA2 PSK weaknessWPA2-PSK can be cracked offline after capturing the 4-way handshake. All users share the same PSK — if cracked, all traffic can be decrypted retroactively (no forward secrecy). Use WPA3 or WPA2-Enterprise (802.1X).
IDS placementIDS in passive/monitoring mode = copy of traffic via SPAN/TAP — cannot block traffic. IPS in inline mode = traffic passes through it — can block but creates single point of failure and causes latency.

11. Domain 4 Master Mnemonics

🧠 OSI Layers — "Please Do Not Throw Sausage Pizza Away":
Physical · Data Link · Network · Transport · Session · Presentation · Application (bottom to top)
🧠 Wireless Security Evolution — "Weak WEP, Temporary WPA, Workhorse WPA2, Winning WPA3":
WEP = Completely broken (RC4 + tiny IV) → Never use
WPA = Transitional fix (TKIP) → Deprecated 2012 → Avoid
WPA2 = Enterprise standard (AES-CCMP) → PSK vulnerable to offline attack → Use Enterprise mode
WPA3 = Modern (SAE, forward secrecy) → Best available today
🧠 SPF / DKIM / DMARC — "Check the IP, Sign the Message, Make the Policy":
SPF = check authorized IP list
DKIM = sign with private key
DMARC = define policy (none/quarantine/reject) when alignment fails
All three are DNS TXT records. DMARC without enforcement (p=none) = monitoring only.
🧠 ZTNA vs VPN — "ZTNA gives you a door, VPN gives you the whole building":
VPN = network access — you're ON the network, can explore
ZTNA = app access — you get to use ONE specific app; can't see the network
